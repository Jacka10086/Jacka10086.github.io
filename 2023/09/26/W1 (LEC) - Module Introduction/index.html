COM435 - 软件产品和流程管理

Module Overview - 模块概览

  • 讲师信息:

    • Dr George Moore - 乔治·摩尔博士
      • Senior Lecturer - 高级讲师
      • Email: g.moore@ulster.ac.uk - 电子邮件：g.moore@ulster.ac.uk
    • Richard Davies - 理查德·戴维斯
      • Lecturer - 讲师
      • Email: rj.davies@ulster.ac.uk - 电子邮件：rj.davies@ulster.ac.uk

  • 软件产品和流程管理是什么:

    • 5人团队项目。
    • 使用敏捷Scrum规划一个MVP - 结构化实践。
    • 应用其他模块的学习。

  • 为什么要学这个:

    • 开发专业实践的机会。
    • 有机会做大型项目。
    • 制定和计划一个基于Web的MVP的机会。
    • 精炼、演进和审查你的工作。
    • 团队合作的空间。
    • 增强就业能力的潜力。
    • 需要辛勤工作。
    • 不断应用知识（实践）和团队合作（沟通）。

    概括：这是一个关于软件产品和流程管理的模块，涉及团队项目，应用敏捷Scrum规划和其他重要概念。

Module Logistics - 模块物流

  • 时间表:

    • Lectures - 讲座
      • 引入新概念和主题。
      • 提供示例和相关背景。
    • Tutorials - 教程
      • 处理示例。
      • 问答环节。
    • Labs - 实验
      • 处理每周任务。
      • 讨论团队进展。
      • 迭代你的MVP。
      • 向我们提问的机会。

  • 团队物流:

    • 5人团队。
    • 共50个团队。
    • 教学团队分配团队成员。
    • 在实验室中平衡各团队。

  • Microsoft Teams:

    • 模块资料。
    • 内容。
    • 一般频道。
    • 公告。
    • 实验室团队空间。

    概括：模块物流涉及时间表、团队物流和使用Microsoft Teams进行组织和沟通。

Teams Overview - 团队概览

此部分内容较为简略，无明确具体内容。

Assessment - 评估

  • 评估计划:

    • 日期：2024年1月12日。
    • 在实验室进行。
    • 评估包括：
      • 需求和风险评估。
      • MVP和冲刺计划。
      • 班级测试。

  • 工作量:

    • 有很多交付物。
    • 敏捷冲刺代表了公司的工作。
    • 没有空闲时间。
    • 每个人都必须发挥自己的作用，并做得很好。

    概括：评估部分涉及评估计划、工作量和对团队和个人的期望。

Week 1 Lab Activity - 第1周实验活动

  • 数字工具:

    • Microsoft Teams

  • 任务1:

    • 组织团队。
    • 检查Microsoft Teams找出你的团队。
    • 确定应该在哪个实验室会议。
    • 加入团队的专用频道。
    • 与团队聊天。
    • 创建一个word文档。

  • 任务2:

    • 探索MS Teams的功能。
    • 设置课外会议。

  • 任务3:

    • 下载作业包并与团队讨论。
    • 列出疑问或不确定的事情。
    • 作为团队讨论并互相帮助。

    概括：第1周的实验活动涉及使用数字工具，如Microsoft Teams，组织团队，探索MS Teams的功能，以及下载和讨论作业包。
