Learning Outcomes

在这个实验中，您将会：

  • 使用命令行操作 OpenSSL
  • 定位和访问 OpenSSL 在线文档
  • 应用密码学服务进行：
    • 文件加密
    • 文件解密
    • 验证文件的完整性

总结：在这个部分，您将学会如何通过命令行使用 OpenSSL，查找和使用相关的在线文档，以及如何使用 OpenSSL 提供的密码学服务来加密和解密文件，还有如何检验文件的完整性。

Tasks

实验任务包括：

  • 下载 OpenSSL 并运行基本命令
  • 加密文件
  • 解密文件
  • 检查文件

总结：此节介绍了本实验的具体任务，包括如何下载和安装 OpenSSL，以及如何使用 OpenSSL 执行文件加密、解密和完整性检查的基本操作。

Introduction

简介中提到，我们将安装 OpenSSL，这是一个包含多个库的命令行工具，用于执行各种密码学操作。它可以在脚本中或者执行一次性命令行任务时使用。

总结：在简介部分，阐述了 OpenSSL 是一个功能强大的命令行工具，可以执行广泛的密码学操作。它不仅可以嵌入脚本中自动化使用，也适合执行一次性的命令行任务。

OpenSSL基础

1. OpenSSL命令行工具基础

  • 如何查找OpenSSL版本：使用 version 命令可以查询到OpenSSL的版本信息。
  • 可用命令列表：通过 help 命令可以列出OpenSSL所有可用的命令。
  • 可用密码列表：使用 ciphers -v 命令可以显示所有可用的密码算法。
  • 性能基准测试：speed 命令用于测试OpenSSL的性能。

总结：本节介绍了如何在OpenSSL中查找版本信息，获取帮助信息，列出可用的密码算法，以及如何进行性能测试。

2. 使用伪随机数生成器产生密钥

  • 伪随机数生成器(PRNG)是一个利用数学公式产生随机数序列的算法。我们将使用OpenSSL的 rand 函数来生成可以作为加密算法密钥的随机数。

  • 在OpenSSL命令提示符中，用户会自动被放置在默认的家目录中，可以通过 cd 命令切换目录。

  • 为DES和AES生成密钥：在OpenSSL命令提示符中，使用以下命令为DES生成一个56位的密钥，为AES生成一个256位的密钥。

    • DES密钥生成：rand -out deskeySS 56
    • AES密钥生成：rand -out aeskeySS 128

总结：本节讲解了如何使用OpenSSL的伪随机数生成器产生用于DES和AES加密的密钥，并提供了具体的命令行操作示例。

数据加密标准（DES）

3. 数据加密标准（DES）

  • DES是一种对称密钥加密算法。由于其使用了非常短的密钥（即56位），DES被认为是不安全的，但它在密码学的发展中扮演了非常重要的角色。

图表解释

  • 图3展示了加密和解密过程，其中P代表明文，C代表密文，K代表密钥，Encryption代表加密过程，Decryption代表解密过程。

  • 使用DES加密和解密文件，加密和解密的场景如图3所示。首先创建一个文本文件（例如SystemSec.txt），然后在其中写入文本。现在，让我们加密这个文件。

    • 使用命令 des -e -kfile deskeySS -in SystemSec.txt -out SystemSec.enc
    • 分析这个命令：
      • des 是加密算法的名称。
      • -e 开关用于加密文件。
      • -kfile 用于指定密钥文件。
      • deskeySS 是密钥文件的名称。
      • -in 用于指定输入文件。
      • SystemSec.txt 是输入文件的名称。
      • -out 用于指定输出文件。
      • SystemSec.enc 是输出文件的名称。

  • 执行命令后，你会在当前工作目录中发现一个名为SystemSec.enc的文件。打开该文件，你将看到加密后的输出。根据输入文本和密钥，每个系统上的加密输出可能会有所不同。

  • 为了确保加密和解密过程是可靠的，我们应该解密我们刚刚创建的SystemSec.enc文件。解密文件后，可以在你喜欢的编辑器中打开它（本例中使用Windows记事本）。

    • 使用命令 des -d -kfile deskeySS -in SystemSec.enc -out SystemSec.dec

  • 我们期望SystemSec.dec文件与我们最初创建的SystemSec.txt文件是相同的。由于相同的文件会有相同的消息摘要，你可以通过创建和比较每个文件的消息摘要来证明文件是相同的。输入以下命令行，然后比较生成的摘要（哈希值）。

    • md5 SystemSec.txt
    • md5 SystemSec.dec

总结：本节详细解释了使用DES算法加密和解密文件的过程，包括命令行的使用和加密文件的验证方法。图表清晰展示了加密和解密的过程，帮助更好地理解DES算法的工作原理。

高级加密标准（AES）

4. 高级加密标准（AES）

  • AES是由NIST开发的对称块密码。我们可以使用AES的五种操作模式进行加密和解密，如下所示：

加密命令

  • aes-256-cbc -e -in plaintext.txt -out encrypted.enc
  • aes-256-ecb -e -in plaintext.txt -out encrypted.enc
  • aes-256-cfb -e -in plaintext.txt -out encrypted.enc
  • aes-256-ofb -e -in plaintext.txt -out encrypted.enc
  • aes-256-ctr -e -in plaintext.txt -out encrypted.enc

解密命令

  • aes-256-cbc -d -in encrypted.enc -out plaintext.txt

  • aes-256-ecb -d -in encrypted.enc -out plaintext.txt

  • aes-256-cfb -d -in encrypted.enc -out plaintext.txt

  • aes-256-ofb -d -in encrypted.enc -out plaintext.txt

  • aes-256-ctr -d -in encrypted.enc -out plaintext.txt

  • 接下来，我们将重复AES加密和解密过程，并使用其128位密钥。请记住，我们已经在“使用伪随机数生成器生成密钥”的第2部分中创建了aes_keySS。

加密与解密过程

  • 加密文件：aes256 -e -kfile aes_keySS -in SystemSec.txt -out SystemSec.aes.enc

  • 解密文件：aes256 -d -kfile aes_keySS -in SystemSec.aes.enc -out SystemSec.aes.dec

  • 通过消息摘要确认文件完整性，此过程与之前的DES解密确认过程相同。我们需要将原始文件（即SystemSec.txt）和AES解密后的文件（即SystemSec.aes.dec）进行比较。

总结：本节介绍了高级加密标准（AES）的基础知识，包括其加密和解密命令的各种操作模式。此外，还提供了使用AES进行文件加密和解密的具体命令示例，并解释了如何通过消息摘要来验证文件在加密和解密过程中是否保持不变。

DES和AES任务

在这一部分，我们将探索如何通过DES和AES加密标准来验证加密文件是否被篡改。

1. 任务描述

  • Alice在她的电脑上存储了一个加密的秘密文件（Sec.enc）。Eve不知道加密密钥，但是她想要修改文件的内容。Alice如何验证文件内容是否被修改了？为了模仿这个过程，请执行以下任务：

任务步骤

a. 生成一个256位的AES密钥。
b. 创建一个Sec.txt文件，并写入一些文本。
c. 使用AES-CTR模式加密文件，输出可能是Sec.enc，并计算md5消息摘要。
d. 从Sec.enc文件中删除一些内容，然后重新计算md5消息摘要。
e. 比较两个消息摘要。

总结：这一系列任务旨在通过模拟加密和篡改过程来教授验证加密文件完整性的方法。首先，我们需要创建密钥和待加密的文件，然后进行加密并计算其摘要。之后，模拟篡改行为，再次计算摘要，最后比较两次摘要来确定文件是否被更改。
