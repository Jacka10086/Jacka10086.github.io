Week5-Intrusion Detection & Firewalls

2023-10-23-15:20

Cyber Criminals

  1. 身份盗窃 (Identity theft): 这是指非法获取并使用另一个人的个人信息，如姓名、社会保险号或信用卡号，通常是为了经济利益。这可能导致受害者面临财务损失或信誉受损。

  2. 财务凭证盗窃 (Theft of financial credentials): 这涉及到窃取银行账户、信用卡或其他财务相关的登录凭证。这样的信息可以被用来非法访问受害者的账户并进行未授权的交易。

  3. 企业间谍 (Corporate espionage): 这是指为了获得竞争优势或经济利益而非法获取另一家公司的机密信息。这可能包括商业策略、新产品设计或其他有价值的信息。

  4. 数据盗窃 (Data theft): 这是指非法访问、复制或传输某人或某组织的数据，而不一定是为了直接的经济利益。这可能包括个人数据、商业数据或政府数据。

  5. 数据勒索 (Data ransoming): 这是指黑客入侵某人或某组织的系统，加密其数据，并要求支付赎金以解密数据。如果不支付，受害者可能无法再访问他们的数据，或者数据可能被公开。

通过何种方式攻击

  1. Website Defacement (网站篡改):

    • 描述：这是一种攻击方式，其中攻击者更改网站的外观或内容，通常是为了传达某种消息、宣传或仅仅是为了炫耀技能。
    • 目的：这种攻击可能是为了传达政治、宗教或社会信息，或者只是为了损害目标组织的声誉。
    • 示例：一个示范性的例子是，攻击者可能会更改公司主页的内容，添加他们自己的标志或消息。

  2. Denial of Service Attacks (DoS) (拒绝服务攻击):

    • 描述：这种攻击的目的是使目标系统、网络或应用程序不可用，通常是通过洪水流量或其他手段来实现的。
    • 目的：使目标服务不可用，可能是为了破坏、报复或其他恶意目的。
    • 示例：一个常见的DoS攻击是通过发送大量的请求来使目标服务器超载，从而使其无法处理合法的请求。

  3. Theft and Distribution of Data (数据盗窃和分发):

    • 描述：在这种攻击中，攻击者窃取敏感或私有数据，并可能将其发布到公共平台或用于其他恶意目的。
    • 目的：可能是为了经济利益、破坏声誉、勒索或其他目的。
    • 示例：攻击者可能会窃取公司的客户数据库，并将其发布到暗网，或者使用这些数据进行身份盗窃或欺诈活动。

Intruder Behavior

  1. Target acquisition and information gathering
    入侵者首先确定他们的目标，并收集有关目标的尽可能多的信息。这可能包括公开的信息，如公司的网站，或非公开的信息，如员工的电子邮件地址。

  2. Initial access
    一旦收集了足够的信息，入侵者会尝试首次访问目标系统。这可能是通过利用已知的漏洞，或者通过社交工程技巧，如欺骗员工提供密码。

  3. Privilege escalation
    在成功访问系统后，入侵者可能会寻找方法提高他们的权限，从而获得更多的系统控制权。这可能是通过利用系统中的漏洞或配置错误。

  4. Information gathering or system exploit
    有了更高的权限后，入侵者会进一步收集信息或利用系统资源。例如，他们可能会尝试访问敏感数据，或者使用受害者的系统作为攻击其他目标的跳板。

  5. Maintaining access
    为了确保他们可以随时访问受害者的系统，入侵者可能会安装后门或其他恶意软件。

  6. Covering tracks
    最后，为了避免被发现，入侵者会尝试删除或修改日志和其他证据，以掩盖他们的行为。

入侵技术

当然可以。这两个术语在计算机安全和密码学中都很重要，但它们涉及的是两个不同的概念。

1. One-way function

单向函数是密码学中的一个概念，指的是一种可以容易计算的函数，但其逆函数（即从输出反推输入）却很难计算。换句话说，给定一个输入值，我们可以轻易地得到输出值，但如果只给定输出值，我们则很难找到对应的输入值。

例如，哈希函数就是一种常见的单向函数。它们经常用于密码学应用，如密码存储。当用户设置密码时，系统会使用哈希函数将密码转换为哈希值，并存储该哈希值。当用户再次登录时，系统会再次对输入的密码进行哈希，然后与存储的哈希值进行比较。由于哈希函数是单向的，即使攻击者获得了哈希值，他们也很难从中恢复原始密码。

2. Access Control

访问控制是计算机安全中的一个概念，指的是限制对资源的访问。这可以基于用户的身份、角色或其他属性来实现。访问控制确保只有经过授权的用户或系统能够访问、修改或删除特定的资源。

常见的访问控制模型包括：

  1. **基于身份的访问控制 (Identity-Based Access Control, IBAC)**：基于用户的身份进行访问控制。
  2. **基于角色的访问控制 (Role-Based Access Control, RBAC)**：基于用户的角色（如管理员、普通用户等）进行访问控制。
  3. **基于属性的访问控制 (Attribute-Based Access Control, ABAC)**：基于用户或资源的属性（如年龄、部门、文件类型等）进行访问控制。

总之，单向函数主要与数据的加密和保护有关，而访问控制则与资源的权限管理有关。

Advanced Persistent Threats (APTs)

高级持续性威胁（Advanced Persistent Threats，简称APTs）是一种复杂的网络攻击形式，攻击者通常有着丰富的资源和高度的专业技能。这类攻击通常是有针对性的，攻击者会长时间地潜伏在目标网络中，悄无声息地进行数据窃取或者其他恶意活动。

特点 Characteristics

APTs的主要特点包括：

  • 持续性：攻击者会在目标网络中长时间潜伏，可能持续数月甚至数年。
  • 高度隐蔽：攻击者会使用各种手段来隐藏自己的行踪，避免被发现。
  • 有针对性：APTs通常针对特定的组织或个人，攻击者会事先进行详细的情报收集。
  • 资源丰富：进行APTs攻击的组织通常拥有丰富的资源，包括资金、人员和技术。

攻击手段 Attack Methods

APTs攻击可能包括但不限于以下手段：

  • 社会工程学：通过欺骗手段诱使目标泄露敏感信息。
  • 恶意软件：使用病毒、木马等恶意软件侵入目标网络。
  • 零日漏洞：利用尚未公开的软件漏洞进行攻击。
  • 内网渗透：在成功侵入目标网络后，进一步渗透到内网中获取更多资源。

防御策略 Defense Strategies

防御APTs需要综合运用多种安全技术和策略，包括但不限于：

  • 网络监控：实时监控网络活动，及时发现异常行为。
  • 入侵检测和防御：部署入侵检测系统和防火墙，阻止恶意活动。
  • 安全培训：对员工进行安全意识培训，提高防范社会工程学攻击的能力。
  • 定期更新和打补丁：确保所有系统和软件都是最新版本，及时修补已知漏洞。

通过这样的方式，组织可以提高自身的安全防御能力，降低APTs攻击的风险。

Intrusion Detection System (IDS)

是一个设备或软件应用，用于监视网络或系统的活动以检测恶意活动或违规操作。当检测到可疑活动时，IDS会发出警告或采取预定的行动。根据其工作方式和部署位置，IDS可以分为两大类：

  1. 网络入侵检测系统 (NIDS): 监视整个网络的流量，寻找异常或恶意的数据包。
  2. 主机入侵检测系统 (HIDS): 安装在特定设备或服务器上，监视该设备上的日志文件、系统调用或文件系统修改等，以检测恶意活动。

IDS主要是被动地检测和警告，而不是主动地阻止攻击。如果一个系统既能检测也能阻止攻击，那么它通常被称为入侵防御系统 (IPS)。

IDS组件

入侵检测系统是用于检测和警告潜在的未授权访问或对系统的恶意活动的计算机安全系统。以下是对这些组件的简要解释：

  1. **传感器 (Sensors)**：这些是收集数据的组件。它们可以监听网络流量、读取日志文件、跟踪系统调用等，以收集可能与安全事件相关的信息。

  2. **分析器 (Analyzer)**：此组件对从传感器收集的数据进行分析，以确定是否发生了入侵。它使用各种算法和签名来识别已知的恶意行为或异常活动。

  3. **用户界面 (User Interface)**：这是与IDS交互的界面。管理员可以使用它来查看系统的输出、接收警报、配置系统行为等。

这三个组件共同工作，以确保系统安全并及时警告任何潜在的威胁。

Anomaly Detection (异常检测)

异常检测是一种识别数据中不符合预期行为或模式的过程。这种技术在许多领域都有应用，如网络安全、欺诈检测、系统健康监测等。

方法 (Methods)

异常检测的方法多种多样，包括统计方法、机器学习方法和基于距离的方法等。

统计方法 (Statistical Methods)

这些方法基于统计学原理，通过计算数据的统计参数（如均值、方差等）来识别异常值。

机器学习方法 (Machine Learning Methods)

机器学习方法利用算法来学习数据的正常模式，并基于这些学习到的模式来识别异常。

基于距离的方法 (Distance-Based Methods)

这些方法通过计算数据点之间的距离来识别异常，常见的算法有K-最近邻（K-NN）等。

应用 (Applications)

异常检测在许多领域都有广泛的应用，如：

  • 网络安全：检测网络入侵或异常流量
  • 金融领域：识别信用卡欺诈或异常交易
  • 工业制造：监测设备状态，预防故障
  • 医疗领域：监测病人的健康状况，预警异常情况

通过有效的异常检测，我们可以及时发现并处理异常情况，保障系统的稳定运行和数据的安全。

Signature Detection (签名检测)

签名检测是一种传统的恶意软件检测方法，主要基于已知的恶意代码样本生成特征签名。

优点：

  1. 速度快：对于已知的恶意软件，可以迅速识别。
  2. 准确性高：对于已知样本，误报率较低。

缺点：

  1. 无法检测新型恶意软件：只能检测已有签名的恶意软件。
  2. 需要常规更新：为了检测新的恶意软件，需要不断更新签名数据库。

Heuristic Detection (启发式检测)

启发式检测是基于恶意软件的行为和属性，而不是固定的签名来检测的。

优点：

  1. 能够检测新型恶意软件：不依赖于已知的签名。
  2. 适应性强：可以应对恶意软件的快速变化。

缺点：

  1. 可能出现误报：由于是基于行为和属性，可能将正常软件误判为恶意软件。
  2. 检测速度可能较慢：相对于签名检测，启发式检测可能需要更多的计算资源。

Intrusion Prevention System (IPS) 入侵防御系统 (IPS)

定义 Definition

入侵防御系统（IPS）是一种网络安全/防护工具，设计用于检测并阻止潜在的有害活动。

功能 Features

  • 检测 Detection：IPS能够识别恶意活动，如病毒、虫、木马等。
  • 阻止 Prevention：当检测到潜在的恶意活动时，IPS会立即采取行动，阻止其进一步的传播或损害。
  • 日志 Logging：记录所有检测到的活动和所采取的行动，以供后续分析。

工作方式 How It Works

  1. 流量分析 Traffic Analysis：IPS不断地分析进出网络的数据流。
  2. 签名匹配 Signature Matching：与已知的恶意活动签名进行匹配，以识别威胁。
  3. 异常检测 Anomaly Detection：检测与正常流量模式不符的行为。

优点 Advantages

  • 实时防护 Real-time Protection：IPS提供即时的威胁检测和响应。
  • 减少风险 Risk Reduction：通过阻止恶意活动，降低网络受损的风险。

缺点 Disadvantages

  • 误报 False Positives：有时可能会误判合法的网络活动为恶意行为。
  • 性能影响 Performance Impact：高级的IPS可能会对网络性能产生一定的影响。

希望这样的格式可以满足您的需求！
