字符串比较：.equals() vs ==

在Java中，当你想比较两个字符串是否相同，你可以使用 .equals() 方法或者 == 运算符，但它们的比较方式是有区别的。

.equals() 方法

  • 描述: name1.equals(name2) 用于比较两个字符串的内容是否相等。
  • 实现: equals() 方法被 String 类重写，所以它比较的是两个字符串的内容。
  • 结果: 如果两个字符串的内容完全一样，那么它返回 true；否则返回 false。

== 运算符

  • 描述: name1 == name2 用于比较两个字符串对象的引用是否相等。
  • 实现: == 运算符比较的是对象在内存中的地址。
  • 结果: 如果两个字符串对象指向内存中的同一个地址，那么它返回 true；否则返回 false。即使两个字符串内容相同，但是在内存中的地址不同，name1 == name2 的结果也会是 false。

总结

当你想要比较两个字符串的内容是否相同，应该使用 .equals() 方法。而当你想要检查两个字符串对象是否是同一个对象时，可以使用 == 运算符。

Integer vs int

在Java中，Integer 和 int 都可以用于表示整数，但它们之间有明显的差异。

Integer

  • 类型: Integer 是 int 的包装类，属于 java.lang 包。
  • 描述: 作为一个类，Integer 可以像其他类一样拥有方法和属性。它允许我们将一个 int 值包装在一个对象中。
  • 例子: Integer num1 = 200; 声明了一个 Integer 对象，并将其初始化为 200。
  • 用法: 由于 Integer 是一个对象，你可以调用它上面的各种方法，例如 num1.toString()。

int

  • 类型: int 是一个基本数据类型。
  • 描述: 用于存储整数值，并且不是一个对象。因此，它不具有方法和属性。
  • 例子: int num2 = 200; 声明了一个 int 变量，并将其初始化为 200。
  • 用法: 由于 int 不是一个对象，你不能在它上面调用任何方法。

自动装箱 & 拆箱

Java 提供了自动装箱（autoboxing）和拆箱（unboxing）的功能，使得 Integer 和 int 之间的转换变得无缝。例如，你可以将一个 Integer 对象赋值给一个 int 变量，反之亦然。但需要注意，这种转换在某些情况下可能会影响性能。

总结

尽管 Integer 和 int 都用于表示整数，但它们在Java中的使用和特性上有所不同。选择使用哪一个取决于你的特定需求，但在大多数情况下，如果没有特殊的面向对象的需求，使用基本数据类型 int 会更高效。

“” vs null

在 Java 中，"" 和 null 都可以用于表示空值，但它们的意义和用法有所不同。

“”

  • 描述: "" 表示一个空字符串，即一个长度为 0 的字符串。
  • 特性: 它是一个有效的字符串对象，因此你可以在它上面调用字符串的方法。
  • 例子: "".length() 返回 0，因为这是一个空字符串的长度。

null

  • 描述: null 表示一个空对象引用，即它不指向任何有效的对象。
  • 特性: 不能在 null 上调用任何方法，因为它不是一个对象。尝试这样做将抛出 NullPointerException 异常。
  • 用法: null 通常用于表示一个变量没有被初始化或者一个方法没有返回值。

注意事项

  1. 空引用: 在 Java 中，尝试将一个 null 引用赋值给一个字符串变量会导致 NullPointerException。
  2. 编译错误: 如果尝试将 "" 赋值给一个对象变量，你会得到一个编译错误。
  3. 初始化: 在 Java 中，如果你声明了一个对象变量但没有初始化，其默认值是 null。
  4. 返回值: 一个没有返回值的方法在 Java 中的返回类型是 void。这意味着它不返回任何值，包括 null。

总之，尽管 "" 和 null 在某些情况下可能看起来相似，但它们有不同的用法和特性。在编写代码时，需要清楚地区分这两者。
