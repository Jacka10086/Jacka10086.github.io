Git 简介

Git 是一个分布式版本控制系统，允许开发者记录和跟踪文件的更改，而不需要中央服务器。
Git is a distributed version control system that allows developers to record and track changes to files without the need for a central server.

主要特点：

  • 分布式版本控制：每个开发者拥有完整的版本库和全部历史记录。
  • 快照而非差异：Git 将数据视为快照流，而不是文件差异。
  • 数据完整性：通过 SHA-1 哈希确保数据完整性。
  • 灵活的工作流：支持多种工作流，如特性分支、Gitflow。
  • 轻量级分支：分支创建和切换异常迅速。

基本概念：

  • **仓库 (Repository)**：存储项目的地方，包含所有文件和历史。
  • **提交 (Commit)**：保存更改的操作，每次都有唯一的 SHA-1 哈希。
  • **分支 (Branch)**：用于并行开发的轻量级指针。
  • HEAD：指向当前分支最新提交的指针。
  • **远程 (Remote)**：在其他服务器上的版本库引

常用命令：

  • git init：初始化新的 Git 仓库。
  • git clone：克隆仓库到本地。
  • git add：添加更改到暂存区。
  • git commit：提交暂存区的更改。
  • git push：推送更改到远程仓库。
  • git pull：拉取远程仓库的更改。
  • git branch：查看或操作分支。
  • git checkout：切换分支或恢复文件。

Git 可能需要一些时间去学习，但其功能和效率使其成为开发者的首选工具。
