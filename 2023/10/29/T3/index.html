Java Generics是什么？

Java Generics是Java编程语言的一个特性，它允许开发者在类、接口或方法上定义类型参数。通过使用Generics，开发者可以创建灵活的代码，而不需要牺牲类型安全性。Generics的主要目标是提供编译时的类型检查，以防止运行时的类型转换错误，同时提供一个更优雅、更强大的解决方案来解决许多与类型相关的问题。

核心概念和特点：

  1. 类型参数
    Generics允许你定义一个类、接口或方法，同时带有一个或多个类型参数。例如，List<T>中的T就是一个类型参数。

  2. 类型安全
    Generics提供了编译时的类型安全性。这意味着，如果你有一个List<String>，编译器会确保你只能向该列表添加String对象，而不是其他类型的对象。

  3. 类型擦除
    为了保持与早期Java版本的向后兼容性，Generics在字节码级别使用了类型擦除的技术。这意味着，运行时，泛型类型信息被擦除，例如List<String>和List<Integer>在运行时都被视为List。

  4. 有界类型参数
    你可以限制类型参数必须是某个类的子类或实现某个接口。例如，<T extends Number>表示T必须是Number类或其子类。

  5. 通配符
    Generics还提供了通配符（?），允许在不知道具体类型参数的情况下表示类型参数。例如，List<?>表示一个未知类型的列表。

  6. 泛型方法
    不仅类和接口可以是泛型的，方法也可以。这使得开发者可以编写灵活的方法，这些方法可以在不同的类型上工作，同时保持类型安全。

ArrayBag 构造函数解释

这段代码展示了ArrayBag类中一个构造函数的部分实现。这个构造函数允许你初始化一个特定容量的ArrayBag。以下是对每行代码的解释：

public ArrayBag(int capacity) {

这是ArrayBag类的构造函数的开始，它接受一个整数capacity作为参数，表示要创建的数组的容量。

T[] tempBag = (T[]) new Object[capacity];

这是一个使用Java的泛型创建数组的常见技巧。由于Java不允许直接创建泛型数组（例如new T[capacity]），因此首先创建一个Object类型的数组，然后将其转型为泛型类型T[]。这里的T是一个类型参数，代表可以是任何类型。

this.bag = tempBag;

将tempBag（一个新创建的泛型数组）赋值给ArrayBag类的bag成员变量。这意味着bag现在引用了一个新的数组，其容量由构造函数的参数决定。

this.numberOfEntries = 0;

初始化numberOfEntries成员变量为0。这表示ArrayBag在刚被创建时是空的，没有任何条目。

总的来说，这个构造函数允许用户创建一个指定容量的新ArrayBag，初始时没有任何条目。
