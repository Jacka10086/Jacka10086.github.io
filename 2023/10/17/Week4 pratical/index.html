LinkedBag.java

是一个实现了链式结构的 Bag（或称为 Multiset）数据结构的 Java 类。

主要特点:

  • 数据结构: Bag 是一种可以存储重复元素的数据结构，它类似于 Set，但允许元素重复出现。与 Set 不同，Bag 中的元素没有固定的顺序。
  • 链式实现: Linked Bag 使用链表来存储元素，每个链表节点包含一个元素和一个指向下一个节点的指针。

优势:

  • 动态大小: 由于使用链表来存储元素，Linked Bag 可以动态地添加和删除元素，不需要预先分配固定大小的空间，这与数组实现的 Bag 不同。
  • 操作效率: 在链表的开始或结束添加或删除元素通常很快，但在链表中间可能较慢。

应用场景:

  • 文本编辑器中的撤销和重做操作。
  • 图形界面中的事件处理器。
  • 任何需要频繁添加和删除元素，且不太关心元素访问速度的场景。

结论: LinkedBag.java 提供了一个基于链表的 Bag 数据结构实现，它适用于动态数据集合，可以方便地进行元素的添加和删除操作。

addNewEntry(Building newEntry)

addNewEntry(Building newEntry) 是 LinkedBag 类中的一个方法，它的主要功能是向 Bag 中添加一个新的 Building 对象。

工作原理:

  1. 节点创建: 方法首先创建一个新的 Node 对象，将 newEntry 对象存储为该节点的数据。
  2. 头部插入: 方法将新节点插入到链表的头部。这样的插入方法是效率最高的，因为它不需要遍历整个链表。
  3. 更新引用: 在插入新节点后，它将新节点的 next 引用指向当前的头节点，并将链表的头节点更新为新创建的节点。
  4. 数量更新: numberOfEntries 是一个变量，用于跟踪 Bag 中的元素数量。在添加新元素后，该方法会将 numberOfEntries 的值增加1。
  5. 返回值: 最后，方法返回 true，表示元素已成功添加到 Bag 中。

结论: addNewEntry(Building newEntry) 方法提供了一种高效的方式向 LinkedBag 中添加新的 Building 对象，通过在链表头部插入节点来确保操作的速度。

newNode.setNext(this.firstNode);

  1. 目的: 此语句的目的是将新节点 (newNode) 与现有链表连接起来。
  2. 操作: 将新节点的 next 指针（或引用）设置为当前链表的头节点 firstNode。这样，newNode 就成为了指向原先链表头部的节点，为插入新的头节点做好准备。

this.firstNode = newNode;

  1. 目的: 这个语句将链表的头部移动到新节点上，完成了节点的插入过程。
  2. 操作: 把链表的 firstNode 引用（头节点指针）更新为 newNode。现在，newNode 成为了链表的新头节点。

结合使用

  • 这两个语句配合使得新节点成为链表的新头节点，同时保持链表的其他部分不变。
  • 在这个操作中，原来的头节点现在成为了新头节点的下一个节点。

  • 通过替换要删除的节点的数据，并删除链表头部的节点，我们可以简化删除操作。这种方法特别适用于单链表，因为在单链表中找到一个节点的前一个节点需要遍历链表，这是一个时间复杂度为 O(n) 的操作。

public void clear() { while (!isEmpty()) remove(); }

clear() 方法是 LinkedBag 类中定义的，主要用于删除链表中的所有节点，从而清空链表。

功能：

  • 目的: 清空链表中的所有数据，使其恢复到初始状态。
  • 操作: 使用 while 循环连续调用 remove() 方法，直到链表为空。

实现逻辑：

  1. 条件判断: isEmpty() 方法用于判断链表是否为空。!isEmpty() 确保当链表不为空时，循环继续执行。
  2. 删除操作: 在循环体中，remove() 方法每次都删除链表的头节点。由于每次循环都会删除一个节点，最终所有节点都会被删除，链表将变为空。

示例：

假设链表的状态是：A -> B -> C

  • 在第一次循环中，节点 A 被删除，链表变为：B -> C
  • 在第二次循环中，节点 B 被删除，链表变为：C
  • 在第三次循环中，节点 C 被删除，链表变为空

此时，isEmpty() 返回 true，循环结束，链表已经被完全清空。

结论：

  • clear() 方法提供了一种高效的方式来清空链表。
  • 它利用了 isEmpty() 和 remove() 两个方法来逐一删除链表中的所有节点。
  • 当链表已被清空时，isEmpty() 会返回 true，从而结束 while 循环。

Node.java

Node.java 是一个通常在链表数据结构中使用的基本组件。它定义了链表中的单个元素或节点。

结构：

  1. 属性：

    • data：这是存储在该节点中的实际数据。在这个情况下，它是一个 Building 对象。
    • next：这是指向链表中下一个 Node 的引用。如果这是链表的最后一个节点，则 next 通常会设置为 null。

  2. 构造方法：

    • Node(Building dataValue)：接受一个 Building 类型的数据作为参数，并将其存储在 data 属性中。同时，它将 next 属性设置为 null，意味着初始情况下，它不指向链表中的任何其他节点。

  3. 方法：

    • getData()：返回节点中存储的数据。
    • setData(Building dataValue)：允许我们修改或设置节点中的数据。
    • getNext()：返回当前节点后面的节点。如果当前节点是链表中的最后一个节点，它会返回 null。
    • setNext(Node nextNode)：设置该节点后的节点。这允许我们在链表中重新安排或添加新的节点。
