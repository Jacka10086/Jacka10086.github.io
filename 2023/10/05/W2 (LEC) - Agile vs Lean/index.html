W2 (LEC) - Agile vs Lean

Created2023-10-05|Updated2023-10-25|Com435
|Post Views:

Introduction to Agile Scrum (敏捷Scrum简介)

敏捷Scrum是一种灵活、迭代的软件开发框架，旨在提供高质量的软件和增强团队合作。

Software Development Practice (软件开发实践)

  • Lean (精益)
    • 精益开发注重消除浪费，以最小的资源实现最大的效益。
  • Agile Scrum (敏捷Scrum)
    • 敏捷Scrum是一种灵活的开发方法，强调小团队、短迭代和客户反馈。

总结：这部分介绍了敏捷Scrum下的两种主要软件开发实践。

Software Development Life Cycle/Project Methodologies (软件开发生命周期/项目方法论)

  • Traditional (传统的)
    • 传统的方法通常是线性的，从需求分析到维护阶段。
  • Waterfall (瀑布)
    • 瀑布方法是一种线性方法，每个阶段依赖于前一个阶段的完成。
  • Incremental / Waterfall (modified) (增量式/修改过的瀑布)
    • 这是瀑布方法的变种，允许在整个开发过程中进行小的、增量的改进。
  • Prototyping (原型制作)
    • 这种方法侧重于创建软件原型以获取早期反馈。
  • Rapid Application Development (RAD) (快速应用开发)
    • RAD强调快速迭代和早期原型。
  • Spiral (螺旋)
    • 螺旋方法结合了原型制作和风险管理。
  • Agile (敏捷)
    • 敏捷是一种迭代方法，强调灵活性、客户合作和响应变化。
    • Scrum, Kanban, eXtreme Programming (XP), Lean Development, Feature Driven Development (FDD), DevOps, Test Driven Development (TDD)都是敏捷方法的子集或相关方法。

总结：这部分详细介绍了多种软件开发生命周期和项目方法论。

Agile & Lean (敏捷和精益)

Software Development Practice (软件开发实践)

  • Philosophies (哲学)
    • 敏捷和精益都有自己的开发哲学，注重客户价值和团队合作。
  • Methodologies (方法论)
    • 这些方法论提供了具体的实践指导和步骤。
  • Tools (工具)
    • Lean, Agile, Traditional等提供了各种工具来帮助开发团队。
    • Scrum, Kanban, XP, TPS等是这些工具的实例。
    • sprints, boards, tests, cohort analysis等是具体的实践工具。

总结：这部分对敏捷和精益下的软件开发实践进行了深入的探讨。

Agile Manifesto (敏捷宣言)

Agile Manifesto 描述了开发软件的更好方法，并列出了以下核心价值观：

  • Individuals and Interactions (个体和交互)
    • 更注重团队成员之间的交互和合作。
  • Working software (可工作的软件)
    • 交付可工作的软件比文档更重要。
  • Customer collaboration (与客户合作)
    • 与客户的持续合作比合同谈判更有价值。
  • Responding to change (对变化的响应)
    • 对变化的响应比遵循计划更重要。

总结：敏捷宣言强调了软件开发中的核心价值观。

12 Principles for Agile (敏捷的12条原则)

这12条原则为敏捷开发提供了指导思想，包括优先满足客户、欢迎变化等。

总结：这12条原则提供了敏捷开发的指导思想。

Lean Principles (精益原则)

精益原则包括消除浪费、增强学习、尽量晚做决策、快速交付、赋权团队、构建完整性和看到整体。

总结：这些原则为精益开发提供了明确的指导。

What is Lean? (什么是精益?)

精益的核心理念是持续提高并最大化客户价值，包括最大化客户价值、减少浪费和不断追求完美。

总结：精益的核心理念是持续提高并最大化客户价值。

Lean vs Agile (精益 vs 敏捷)

敏捷主要是为了更快地执行任务和适应变化，而精益则关注智能开发和消除不为客户带来价值的事物。

总结：这部分比较了敏捷和精益的哲学和实践方法。
