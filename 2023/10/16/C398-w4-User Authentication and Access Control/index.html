数字用户认证介绍 (Introduction to Digital User Authentication)

本部分内容介绍了NIST SP 800-63-3关于数字用户认证的定义。

数字用户认证定义 (Definition of Digital User Authentication)

  • NIST SP 800-63-3 (数字认证指南，2016年10月) 定义了数字用户认证为：
    • “建立对以电子方式呈现给信息系统的用户身份的信任的过程。”

分支解释 (Branch Explanation)

  • 用户身份：指一个个体在数字世界中的身份标识。
  • 建立信任：通过验证措施确认用户身份的真实性。
  • 信息系统：可以是任何需要用户认证的电子系统，如网站、应用程序或数据库。

总结：数字用户认证是信息安全的核心组成部分，它确保只有经过授权的用户才能访问系统。这个过程涉及到验证用户身份的真实性，是保护信息系统不受未经授权访问的重要步骤。

用户身份验证方法 (Authenticating User Identity)

用户身份验证的四种手段基于以下原则：

身份验证的四种手段 (The four means of authenticating user identity are based on)

  • 某个人知道的信息 (Something the individual knows)

    • 密码、PIN码、预先安排的问题的答案。

  • 某个人拥有的物品 (Something the individual possesses)

    • 智能卡、电子钥匙卡、物理钥匙。

  • 某个人的生物特征 (Something the individual is - static biometrics)

    • 指纹、视网膜、面部。

  • 某个人的行为特征 (Something the individual does - dynamic biometrics)

    • 语音模式、笔迹、打字节奏。

总结：用户身份验证是确保安全访问控制的重要组成部分，涉及知识、拥有物、生物特征和行为特征这四个方面。这些方法可以单独使用也可以组合使用，以增强身份验证过程的安全性。

多因素身份验证 (Multifactor Authentication)

该图展示了多因素身份验证的过程。

多因素身份验证过程 (The Process of Multifactor Authentication)

  • 第一因素验证 (First Factor Authentication)

    • 用户通过输入密码、PIN码等知识型因素进行身份验证。
    • 若验证通过，则进入第二因素验证；若失败，则认证结束。

  • 第二因素验证 (Second Factor Authentication)

    • 用户需提供拥有型因素，如智能卡、手机验证码等。
    • 若第二因素验证通过，则认证成功；若失败，则认证结束。

图表解释 (Explanation of the Diagram)

  • 客户端 (Client)
    • 指尝试进行身份验证的用户。
  • 认证逻辑 (Authentication Logic)
    • 服务器端执行的验证过程，判断提供的凭证是否有效。
  • 认证协议 (Authentication Protocol)
    • 定义客户端和服务器之间进行身份验证所需通信的规则和步骤。

总结：多因素身份验证通过要求用户提供两种或更多类型的认证因素，大幅增强了安全性。这通常包括某个人知道的信息（如密码）和某个人拥有的物品（如智能卡）。这种方法在当前的数字安全实践中被广泛采用，以防止未授权访问。

基于密码的认证 (Password-Based Authentication)

基于密码的认证是防止入侵者的常用防线。

认证流程 (Authentication Process)

  • 用户提供用户名/登录名和密码。
  • 系统将提供的密码与为该指定登录名存储的密码进行比较。

用户ID的作用 (The Role of User ID)

  • 确定用户是否被授权访问系统。
  • 决定用户的权限。
  • 用于自主访问控制。

总结：基于密码的认证是最普遍的用户身份验证方式，用户ID不仅用于确认身份，还定义了用户在系统中的权限级别。这种方法便于实施但也需要适当的安全措施，以确保密码的安全性和防范未授权访问。

基于密码的认证机制 (Password-Based Authentication Mechanism)

此图展示了UNIX系统中用于密码验证的机制。

密码验证流程 (Password Verification Process)

  • 加载新密码 (Loading a new password)

    • 用户的密码与“盐值”结合后通过慢速哈希函数处理。
    • 处理后的哈希值存储在密码文件中，与用户ID和盐值一起。

  • 验证密码 (Verifying a password)

    • 系统根据用户ID在密码文件中选择相应的盐值和哈希码。
    • 用户输入的密码与盐值结合，再次通过慢速哈希函数处理。
    • 系统比较这个新生成的哈希值与存储的哈希值是否匹配。

UNIX密码方案图解 (UNIX Password Scheme Diagram)

  • 密码文件 (Password File)
    • 存储用户ID、盐值和密码的哈希码。
  • 盐值 (Salt)
    • 用于保护密码，使得同一个密码在不同盐值下产生不同的哈希值。
  • 慢速哈希函数 (Slow Hash Function)
    • 用于增加破解密码的难度，因为它需要更多的计算资源和时间。

总结：UNIX密码方案通过使用盐值和慢速哈希函数提高了密码存储的安全性。这种机制可以有效地防止密码被破解，确保系统安全。

密码漏洞 (Password Vulnerabilities)

这张图表列出了密码系统可能面临的多种安全漏洞。

密码漏洞类型 (Types of Password Vulnerabilities)

  • 离线字典攻击 (Offline dictionary attack)

    • 攻击者在未连接到系统的情况下，尝试破解密码。

  • 针对单一用户的密码猜测 (Password guessing against single user)

    • 攻击者尝试猜测单一用户的密码，通常基于用户的个人信息。

  • 工作站劫持 (Workstation hijacking)

    • 攻击者控制了用户的工作站，进而尝试越过登录过程。

  • 电子监控 (Electronic monitoring)

    • 通过监控电子通信来拦截密码。

  • 针对特定账户的攻击 (Specific account attack)

    • 目标明确的攻击，专门针对特定账户的密码进行破解。

  • 常见密码攻击 (Popular password attack)

    • 利用用户倾向于使用常见或简单密码的习惯进行攻击。

  • 利用用户错误 (Exploiting user mistakes)

    • 利用用户可能犯的错误，如密码重用或简单密码。

  • 多重密码使用的利用 (Exploiting multiple password use)

    • 利用用户在不同服务中重用相同密码的习惯。

总结：密码漏洞是信息安全中一个常见的问题，包括了多种不同类型的攻击方法。了解这些漏洞有助于我们采取适当的安全措施来保护密码不被破解，从而保障账户安全。

密码破解技术 (Password Cracking Techniques)

这张图表详细介绍了几种常见的密码破解技术。

常见的密码破解方法 (Common Password Cracking Methods)

  • 字典攻击 (Dictionary attacks)

    • 开发一个大型的可能密码字典，并尝试每一个密码对照密码文件。
    • 每个密码必须使用每个盐值进行哈希处理，然后与存储的哈希值进行比较。

  • 彩虹表攻击 (Rainbow table attacks)

    • 预先计算所有盐值的哈希值表。
    • 使用足够大的盐值和足够长的哈希长度可以对抗此攻击。

  • 猜测 / 密码破解 (Guessing / Password crackers)

    • 利用人们倾向于选择容易被猜测的密码的事实。
    • 更短的密码长度也更容易被破解。

  • John the Ripper

    • 开源密码破解工具，于1996年首次开发。
    • 使用暴力破解和字典技术的组合。

总结：上述密码破解技术说明了密码安全的重要性以及为什么需要采取强密码策略。了解这些破解技术可以帮助我们更好地防范潜在的安全威胁，采用更安全的密码管理措施。

字典攻击 (Dictionary Attacks)

字典攻击是猜测密码的一种常见方法。

字典攻击的过程 (The Process of Dictionary Attacks)

  • 使用预选的词汇库和短语来猜测可能的密码。
  • 假设用户通常会使用简单的、常见的密码，如 “password”、”123abc” 和 “123456”。

字典攻击机制 (Dictionary Attack Mechanism)

  • Bot

    • 机器人程序从字典文件中取出猜测密码。

  • 字典文件 (Dictionary File)

    • 包含可能的密码列表。

  • 登录 (Login)

    • 针对特定账户或系统尝试密码。

  • 猜测正确? (Guess correct?)

    • 如果猜测的密码正确，则认证成功；
    • 如果不正确，尝试字典中的下一个密码。

总结：字典攻击表明密码的复杂性和唯一性对于安全性至关重要。通过理解字典攻击的工作原理，我们可以更好地采取措施来保护账户不被此类攻击手段所侵犯。

彩虹表攻击 (Rainbow Table Attacks)

彩虹表攻击是一种高效的密码破解方法。

彩虹表攻击的原理 (Principles of Rainbow Table Attacks)

  • 使用一张特别的预计算字符串或常用密码及其对应哈希值的表。
  • 针对已经通过哈希函数加密的密码进行攻击，通过哈希反查破解密码。

彩虹表链的解释 (Explanation of a Rainbow Table Chain)

  • 密码哈希过程 (Password Hashing Process)
    • 将密码通过哈希函数转换成哈希值。
  • 还原过程 (Reduce Process)
    • 将哈希值通过还原函数转换回可能的密码。
  • 彩虹表 (Rainbow Table)
    • 包含多个这样的哈希和还原序列，形成一个链。

总结：彩虹表攻击通过利用预计算的哈希值来尝试破解密码，这种方法在没有适当保护措施的系统中可能特别有效。了解彩虹表攻击的工作机制对于设计防御这类攻击的安全系统至关重要。

现代密码策略 (Modern Approaches to Password Policy)

现代密码策略强调了制定复杂的密码规则以增强安全性。

密码策略的要点 (Key Points of Password Policy)

  • 强制用户选择更强的密码，增加密码的复杂性。

密码破解技术的进步 (Advancements in Password-Cracking Techniques)

  • 密码破解的处理能力已大幅提高。
  • 使用复杂算法生成可能的密码。

实际密码的研究 (Studying Actual Passwords)

  • 通过分析实际使用中的密码的例子和结构，可以更好地了解密码的脆弱性。
  • 例如，在2009年，针对在线游戏服务RockYou.com的SQL注入攻击泄露了3200万个明文密码。

总结：现代密码策略和密码破解技术的发展表明，简单提升密码复杂性不足以防止破解，还需要其他安全措施。了解密码破解的最新技术对于制定有效的密码政策和保护用户账户至关重要。

密码文件访问控制（Password File Access Control）

可以通过拒绝访问加密密码来阻止离线猜测攻击（Can block offline guessing attacks by denying access to encrypted passwords）

  • 仅向特权用户开放（Make available only to privileged users）
    • 扩展说明：这意味着应该将访问权限严格限制在具有必要权限的用户上，避免未授权用户获取密码文件。
  • 阴影密码文件（Shadow password file）
    • 扩展说明：通常指在系统中有一个加密的密码文件，仅系统内部服务可读取，提高了安全性。

弱点（Vulnerabilities）

  • 操作系统中允许访问文件的弱点（Weakness in the OS that allows access to the file）
    • 扩展说明：这可能指操作系统的安全漏洞，可以让未授权者或恶意软件访问或修改密码文件。
  • 权限设置失误使文件可读（Accident with permissions making it readable）
    • 扩展说明：如果文件权限设置不当，可能导致原本应该受保护的密码文件被任意用户读取。
  • 其他系统上使用相同密码的用户（Users with same password on other systems）
    • 扩展说明：这种情况下，如果用户在多个系统上重用密码，一旦一个系统被破解，其他系统的安全也会受到威胁。
  • 从备份媒体访问（Access from backup media）
    • 扩展说明：备份媒体可能未加密或保护不当，导致攻击者可以从中获取密码信息。
  • 网络流量中嗅探密码（Sniff passwords in network traffic）
    • 扩展说明：在数据传输过程中，密码可能会被截获，尤其是在未加密的网络环境中。

密码选择策略（Password Selection Strategies）

用户教育（User education）

  • 告知用户选择难以猜测的密码的重要性，并提供选择强密码的指南。
    • 扩展说明：教育用户理解强密码设置的重要性，例如使用多种字符类型，避免常见词汇，以及定期更新密码，增强账户安全。

计算机生成密码（Computer generated passwords）

  • 用户常常难以记住计算机生成的密码。
    • 扩展说明：虽然这类密码通常较安全，因为其随机性和复杂性，但用户记忆负担大，可能导致写下密码或使用易记但不安全的方式存储。

反应式密码检查（Reactive password checking）

  • 系统定期运行自己的密码破解器以找到容易猜测的密码。
    • 扩展说明：这种方法通过定期检查现有密码的强度，来识别和提示用户改进弱密码，从而提高系统的整体安全性。

复杂密码政策 / 主动密码检查（Complex password policy / Proactive Password checker）

  • 用户可以选择自己的密码，但系统会检查密码是否可接受，如果不行，则拒绝。
    • 扩展说明：这种策略要求用户在创建密码时遵守特定规则，如长度、字符类型和复杂性，以此来减少可猜测密码的使用。
  • 目标是在允许用户选择好记的密码的同时，消除容易猜测的密码。
    • 扩展说明：通过教育用户创建既难以猜测又容易记忆的密码，可以提高密码的安全性和用户的使用便利性。

总结

本页内容为Ulster University的演示文稿页面，介绍了密码选择的不同策略，包括用户教育、计算机生成密码、反应式和主动密码检查等方法。主旨在于指导用户如何创建强大而易记的密码，同时通过系统检查来避免弱密码的设置，确保系统安全。

记忆卡（Memory Cards）

基本特性

  • 能够存储但不处理数据。
    • 扩展说明：记忆卡用于存储信息，但不具备处理或计算数据的能力。
  • 最常见的是磁条卡（The most common is the magnetic stripe card）。
    • 扩展说明：磁条卡广泛用于银行卡和身份验证卡，信息存储在磁条中。
  • 可包含内部电子存储器（Can include an internal electronic memory）。
    • 扩展说明：一些记忆卡内置有电子存储器，可以存储更多信息。

使用方式

  • 可单独用于物理访问。
    • 例子：
      • 酒店房间（Hotel room）：使用记忆卡作为房间钥匙。
      • 自动取款机（ATM）：使用磁条卡进行身份验证和交易。

安全性提升

  • 当与密码或PIN码结合使用时，能够显著提高安全性。
    • 扩展说明：结合密码使用，可以防止他人在无密码或PIN码的情况下使用卡片。

记忆卡的缺点

  • 需要特殊的读卡器（Requires a special reader）。
    • 扩展说明：使用记忆卡需要有相应的设备来读取卡内信息。
  • 令牌丢失的风险（Loss of token）。
    • 扩展说明：如果记忆卡丢失，可能导致存储在卡上的信息泄露。

总结

本页内容为Ulster University的演示文稿页面，介绍了记忆卡的基本特性、使用方式、安全性优势以及潜在缺点。记忆卡虽然方便，但需要特别注意保管以防丢失，并且在安全性方面，最好与密码或PIN码结合使用以提高保护等级。

智能令牌（Smart Tokens）

物理特性（Physical characteristics）

  • 包含嵌入式微处理器。
    • 扩展说明：这种微处理器赋予智能令牌一定的计算能力，可以执行安全认证等任务。
  • 外观类似银行卡（A smart token that looks like a bank card）。
    • 扩展说明：这种设计便于用户携带和使用，类似于常见的银行卡。
  • 可以是计算器、钥匙、小型便携物品的形状（Can look like calculators, keys, small portable objects）。
    • 扩展说明：多样的形态使得智能令牌可以根据不同的使用场景进行定制。

用户界面（User interface）

  • 手动界面包括键盘和显示屏，用于人与令牌之间的交互（Manual interfaces include a keypad and display for human/token interaction）。
    • 扩展说明：用户可以通过键盘输入信息，显示屏用于反馈信息或指示操作。

电子界面（Electronic interface）

  • 智能卡或其他令牌需要一个电子界面来与兼容的读/写器通信（A smart card or other token requires an electronic interface to communicate with a compatible reader/writer）。
    • 扩展说明：电子界面可能包括接触式和非接触式界面，如插卡槽或NFC。

认证协议（Authentication protocol）

  • 分为三种类别（Classified into three categories）：
    • 静态（Static）
    • 动态密码生成器（Dynamic password generator）
      • 扩展说明：生成一次性密码，增加安全性。
    • 挑战-响应（Challenge-response）
      • 扩展说明：需要令牌计算并提供对特定挑战的响应，用于验证身份。

总结

本页内容为Ulster University的演示文稿页面，介绍了智能令牌的物理特性、用户界面、电子界面和认证协议。智能令牌以其便携性、易于交互的设计以及高安全性的认证协议，成为现代安全领域的重要工具。其多样的形态和高级的认证机制为用户身份验证提供了更加灵活和安全的选择。

智能卡（Smart Cards）

最重要的智能令牌类别（Most important category of smart token）

  • 外观像信用卡（Has the appearance of a credit card）。
    • 扩展说明：方便携带和广泛使用，用户熟悉的形状。
  • 有电子界面（Has an electronic interface）。
    • 扩展说明：允许与读卡器等设备进行通信。
  • 可使用任何智能令牌协议（May use any of the smart token protocols）。
    • 扩展说明：支持多种认证和通信协议，提高兼容性和安全性。

包含内容（Contain）

  • 完整的微处理器（An entire microprocessor）。
    • 扩展说明：使智能卡能够处理数据和执行安全加密操作。
  • 存储器（Memory）。
    • 扩展说明：用于存储数据和程序。

通常包含三种类型的内存（Typically include three types of memory）

  • 只读内存（ROM）- 存储在卡片生命周期内不变的数据。

生物特征认证过程

(a) 注册（Enrollment）

  1. 用户通过用户界面输入姓名（PIN）。
  2. 生物特征传感器捕获生物特征数据。
  3. 特征提取器提取生物特征。
  4. 提取的特征被存储在生物特征数据库中。

(b) 验证（Verification）

  1. 用户通过用户界面输入姓名（PIN）。
  2. 生物特征传感器捕获生物特征数据。
  3. 特征提取器提取特征。
  4. 特征匹配器将提取的特征与数据库中的单一模板比对，返回验证结果（真/假）。

(c) 识别（Identification）

  1. 用户界面不需要用户提前输入身份信息。
  2. 生物特征传感器捕获生物特征数据。
  3. 特征提取器提取特征。
  4. 特征匹配器将提取的特征与数据库中的多个模板进行比对，确定用户身份或返回“用户未识别”。

访问控制政策分类 (Access Control Policy Categories)

属性基于访问控制 (ABAC - Attribute-based access control)

  • 定义: 依据属性（如用户、资源、环境条件）来决定访问权限的策略。
  • 优势: 灵活性高，可实现复杂的访问控制决策。
  • 适用情景: 对访问控制有细粒度需求的场合。

自主访问控制 (DAC - Discretionary access control)

  • 定义: 资源拥有者或管理员控制访问权限的模式。
  • 优势: 管理简便，用户可控性强。
  • 适用情景: 个人或小团队资源共享。

强制访问控制 (MAC - Mandatory access control)

  • 定义: 系统级的访问控制策略，不由个别用户控制。
  • 优势: 安全性高，防泄露能力强。
  • 适用情景: 需要高安全级别的环境，如政府或军事。

角色基于访问控制 (RBAC - Role-based access control)

  • 定义: 依据用户的角色分配权限的访问控制策略。
  • 优势: 易于管理，适应性强。
  • 适用情景: 角色分明的组织结构，如公司部门。

总结

这个图表整合了四种主要的访问控制政策，每种政策都有其独特的定义、优势和适用场景。ABAC的灵活性适合复杂环境，DAC适合小规模环境，MAC适用于安全要求极高的场合，而RBAC便于大型组织中的权限管理。

网络攻击 (Cyber attacks)

客户端攻击 (Clients Attacks)

  • 目标: 针对系统的客户端组件，利用软件或硬件的漏洞进行攻击。
  • 举例:
    • 网络钓鱼 (Phishing): 常见的客户端攻击形式。攻击者发送欺骗性电子邮件或消息给用户，诱导用户泄露敏感信息，如登录凭据。

重放攻击 (Replay Attacks)

  • 操作: 捕获并重新传输两方之间交换的数据，目的通常是为了获取未授权的访问或执行恶意行为。
  • 举例:
    • 针对安全访问控制系统的重放攻击: 攻击者截获一个合法用户的加密访问请求，然后重新播放这个请求以获得对安全设施的未授权进入。

总结

本节内容介绍了两种常见的网络攻击方式：客户端攻击和重放攻击。客户端攻击主要是通过利用客户端的软硬件漏洞来进行，例如通过网络钓鱼诱导用户泄露信息。重放攻击则涉及到数据的截获与重新传输，以便非法获取系统的访问权。了解这些攻击方式有助于我们更好地防范网络安全威胁。

网络攻击 (Cyber attacks)

主机攻击 (Host Attacks)

  • 目标: 针对特定主机或计算机系统的漏洞进行攻击，目的是为了未授权访问或破坏主机的完整性。
  • 举例:
    • 远程代码执行攻击: 攻击者利用主机软件中的漏洞，远程执行恶意代码，可能导致攻击者控制整个系统。

窃听攻击 (Eavesdropping Attacks)

  • 操作: 在不知情的情况下，拦截和监控两个或多个方之间的通讯，以获取敏感信息。
  • 举例:
    • 流量嗅探: 常见的窃听攻击形式。攻击者监听未加密的网络流量，在不安全的网络上捕获并分析诸如登录凭据或个人消息等敏感信息。

总结

本节内容详细介绍了两种类型的网络攻击——主机攻击和窃听攻击。主机攻击通过利用系统漏洞进行远程代码执行，可能导致攻击者获得系统控制权。窃听攻击则通过拦截通信获取敏感信息，如通过流量嗅探来监听网络数据包。了解这些攻击手段对于提高网络安全意识和防范措施至关重要。

网络攻击 (Cyber attacks)

服务拒绝攻击 (DoS - Denial of Service Attacks)

  • 目标: 中断系统或网络的正常运行，通过发送大量请求或流量使其无法为合法用户服务。
  • 举例:
    • Ping Flood: 攻击者向目标服务器或网络发送大量的ping请求，导致服务器无法处理合法请求，从而无法响应。

木马攻击 (Trojan Horse Attacks)

  • 操作: 伪装成合法或良性软件的恶意软件。一旦安装，这些软件可以在用户不知情的情况下执行恶意操作。
  • 举例:
    • 假冒杀毒软件: 攻击者可能会创建一个假的杀毒软件包并在线分发。用户下载并安装后，无意中将自己的系统感染了恶意软件，从而给攻击者提供了未授权的计算机访问权限。

总结

本节讲解了服务拒绝攻击和木马攻击两种网络攻击方式。服务拒绝攻击通过大量流量中断服务，木马攻击则通过假装是合法软件来进行系统渗透。了解这些常见的网络攻击手法，有助于我们采取预防措施，保护网络安全。

  • 包括但不限于：
    • 姓名 (Name)
    • 角色 (Role)
    • 安全级别 (Security clearance)

资源属性 (Resource attributes)

  • 定义资源的特性：
    • 创建日期 (Creation date)
    • 资源所有者 (Resource owner)
    • 数据敏感性 (Data sensitivity)

环境属性 (Environmental attributes)

  • 环境因素可能影响访问决策：
    • 访问时间 (Access time)
    • 数据位置 (Data location)
    • 威胁级别 (Threat levels)

图表解释

  • 图表显示了ABAC模型的结构：
    • 如何根据用户属性、资源属性和环境属性来决定访问权限。
    • 属性之间的关系和它们如何相互作用以形成访问控制决策。

总结

  • 基于属性的访问控制 (ABAC) 是一种动态的安全策略，它根据用户、资源和环境的多个属性来控制对系统的访问。这种方法提供了高度的灵活性和精细的访问控制，适用于需要复杂和详细访问规则的场合。

主体、客体和访问权限 (Subjects, Objects, and Access Rights)

主体 (Subject)

  • 定义：能够访问客体的实体。
  • 分类：
    • 主体 (Subject)
    • 拥有者 (Owner)
    • 组 (Group)

客体 (Object)

  • 定义：访问受控的资源。
  • 功能：
    • 用于包含和/或接收信息。

访问权限 (Access right)

  • 描述：主体访问客体的方式。
  • 可能包括的权限：
    • 读取 (Read)
    • 写入 (Write)
    • 执行 (Execute)
    • 删除 (Delete)
    • 创建 (Create)
    • 搜索 (Search)

图表解释

  • 本图表明确区分了信息系统中的访问主体、客体以及它们之间的访问权限。
  • 主体：是指可以对客体执行操作的用户或进程。
  • 客体：是指系统中的文件、数据库记录或任何其他资源。
  • 访问权限：决定主体对客体的操作类型，如读取或修改等。

总结

  • 在信息安全领域，理解主体、客体和访问权限之间的关系至关重要。主体是请求访问的实体，客体是被访问的资源，而访问权限则定义了主体对客体的访问类型。此图表清晰地描绘了这一安全模型的基本框架。

自主访问控制 (Discretionary Access Control, DAC)

  • 定义：一种安全模式，允许实体授予其它实体访问权限，甚至可能违反自身的权限设定来实现资源共享。

  • 实现方式：

    • 通常使用访问矩阵来提供。

  • 矩阵的维度：

    • 一维包括被识别的主体，它们可能尝试访问资源。
    • 另一维则列出了可能被访问的对象。

  • 矩阵条目：

    • 每个条目指示了特定主体对特定对象的访问权限。

矩阵的结构解释

  • 访问矩阵通常是一个二维表格，其中一维代表访问控制系统中的主体，另一维代表对象。
  • 表格中的每个条目对应一个特定的访问权限，比如读取、写入或执行等。

总结

  • 自主访问控制（DAC）是一种基于主体自行决定其资源访问权限的模型，其特点是灵活性高，允许用户间相互授予权限。访问矩阵是实现DAC的常用工具，它详细地列出了谁可以访问什么资源，以及允许进行的操作。

访问控制结构 (Access control structure)

访问矩阵 (Access matrix)

  • **主体 (SUBJECTS)**：

    • 用户 A (User A)
    • 用户 B (User B)
    • 用户 C (User C)

  • **客体 (OBJECTS)**：

    • 文件 1 (File 1)
    • 文件 2 (File 2)
    • 文件 3 (File 3)
    • 文件 4 (File 4)

  • 权限分配：

    • 用户 A 拥有文件 1 和文件 3 的所有权，可读写。
    • 用户 B 对文件 2 有所有权，可读写；对文件 3 有写权限；对文件 4 有读权限。
    • 用户 C 对文件 1 有读写权限；对文件 2 有读权限；对文件 4 有所有权，可读写。

图表解释

  • 访问矩阵是一个二维表格，主体（如用户）在一维，客体（如文件）在另一维。
  • 表格中的每个单元格指示主体对客体拥有的权限，例如“Own”表示所有权，“Read”表示读取权限，“Write”表示写入权限。

总结

  • 这个访问矩阵图展示了不同用户对不同文件的访问权限。它是理解自主访问控制（DAC）模型中权限分配的一个直观工具，表格的每个单元格明确了谁可以以什么方式访问哪个资源。

访问控制结构 (Access control structure)

访问控制列表 (Access control lists for files of part (a))

  • **文件 1 (File 1)**：

    • 用户 A (User A)：拥有所有权限 (Own)，可读 (R)，可写 (W)
    • 用户 B (User B)：可读 (R)
    • 用户 C (User C)：可读 (R)，可写 (W)

  • **文件 2 (File 2)**：

    • 用户 B (User B)：拥有所有权限 (Own)，可读 (R)，可写 (W)
    • 用户 C (User C)：可读 (R)

  • **文件 3 (File 3)**：

    • 用户 A (User A)：拥有所有权限 (Own)，可读 (R)，可写 (W)
    • 用户 B (User B)：可写 (W)

  • **文件 4 (File 4)**：

    • 用户 B (User B)：可读 (R)
    • 用户 C (User C)：拥有所有权限 (Own)，可读 (R)，可写 (W)

权能列表 (Capability lists for files of part (a))

  • **用户 A (User A)**：

    • 文件 1 (File 1) 和 文件 3 (File 3)：拥有所有权限 (Own)，可读 (R)，可写 (W)

  • **用户 B (User B)**：

    • 文件 1 (File 1)：可读 (R)
    • 文件 2 (File 2)：拥有所有权限 (Own)，可读 (R)，可写 (W)
    • 文件 3 (File 3)：可写 (W)
    • 文件 4 (File 4)：可读 (R)

  • **用户 C (User C)**：

    • 文件 1 (File 1)：可读 (R)，可写 (W)
    • 文件 2 (File 2)：可读 (R)
    • 文件 4 (File 4)：拥有所有权限 (Own)，可读 (R)，可写 (W)

图表解释

  • 本图表展示了访问控制列表和权能列表的两种访问控制结构。
  • 访问控制列表按文件列出每个用户的权限。
  • 权能列表则是按用户列出其对每个文件的权限。

总结

  • 访问控制列表和权能列表是实现访问控制的两种常见结构。访问控制列表针对每个文件列出所有具有访问权限的用户及其权限类型。权能列表则从用户的角度出发，列出他们对每个文件所拥有的权限。这些结构帮助系统维护权限的清晰分配和管理。
