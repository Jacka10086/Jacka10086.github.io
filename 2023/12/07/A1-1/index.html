类的结构

  • 类头（Class Header）： public class Building { 定义了一个名为Building的公共类。
  • 实例变量（Instance Variables）：
    • private String address; 私有字符串变量，用于存储地址。
    • private String owner; 私有字符串变量，用于存储所有者。
      这些实例变量仅在此类内部可访问，意味着外部不能直接访问这些变量。

构造方法

  • 构造方法（Constructor Method）： public Building() 是用来创建Building类实例的构造方法。
    • this.address = ""; 和 this.owner = ""; 分别初始化地址和所有者变量为空字符串。

访问私有变量

  • 由于实例变量是私有的，问题是“它们如何能够从类外部被访问？”
    • 答案通常是通过公共的“getter”和“setter”方法来实现，这些方法提供了读取和修改私有变量的途径。

总结

  • 本部分展示了如何在Java中定义一个类，包括其类头、实例变量和构造方法。它还提出了一个问题关于如何访问私有变量，通常的解决方案是使用公共方法来间接访问它们。

访问器和修改器方法

  • 访问器（”Getter”）和修改器（”Setter”）方法用于返回和更新实例变量的值。

Getter 方法

  • public String getOwner() { return this.owner; }
    • 返回数据类型（The data type returned by the method）： String，这表明getOwner方法将返回一个字符串类型的数据。
    • 返回实例变量值（Return the current value of the instance variable）： 方法通过return this.owner;返回owner属性的当前值。

Setter 方法

  • public void setOwner(String newOwner) { this.owner = newOwner; }
    • 更新实例变量值（Update the current value of the instance variable）： 方法通过this.owner = newOwner;将owner属性更新为新值。
    • 无返回数据（void methods return no data）： void表示这个方法不返回任何数据。

提问

  • “我们可以在创建Building时提供值吗？”
    • 是的，通过在构造方法中添加参数，然后使用这些参数来初始化实例变量，我们可以在创建对象时提供值。

总结

  • 本部分介绍了在Java类中定义的访问器和修改器方法，这些方法允许我们安全地访问和更新类的私有属性。访问器方法返回属性值，而修改器方法则更新属性值。此外，还提出了关于在对象创建时提供属性值的问题，这通常是通过定义带参数的构造方法来实现的。

构造方法重载

  • 可以提供多个版本的构造方法（或其他方法），这些方法通过它们的参数列表区分开来。

默认构造方法

  • public Building() { this.address = ""; this.owner = ""; }
    • 默认构造方法（Default constructor）： 不带参数的构造方法，用于创建类的实例时初始化所有属性为默认值。

重载的构造方法

  • public Building(String add, String own) { this.address = add; this.owner = own; }
    • 重载构造方法（Overloaded constructor）： 带有参数的构造方法，允许在创建类的实例时直接设置address和owner属性的值。

说明

  • 通过构造方法的重载，可以为同一个类提供多种创建对象的方式。这样可以在创建对象时根据需要提供不同的初始值。

总结

  • 本部分展示了在Java中如何通过参数列表的不同来重载构造方法。这是面向对象编程中常用的一种技术，允许开发者在创建对象时提供不同的初始化数据。

toString() 方法

  • 通常用来表示对象状态的方法。

方法定义

  • public String toString() { return this.address + " occupied by " + this.owner; }
    • 这个toString()方法被定义在Building类中。
    • 方法返回一个字符串，该字符串包含了对象的address和owner属性，以及” occupied by “的文字。

说明

  • toString()方法在Java中是一个常见的方法，它被用于返回对象的字符串表示形式。
  • 当打印对象时，例如使用System.out.println()，toString()方法将被自动调用。

总结

  • 本部分展示了如何在Java类中覆盖toString()方法，以便提供更有意义的对象状态信息。这对于调试和日志记录很有帮助，因为它允许对象在被打印时展示其关键信息。

关键OOP概念

  • 到目前为止开发的小型应用程序展示了面向对象编程中的一些关键概念。

类和对象

  • 类和对象 - 类是创建对象的模板。每个对象都有一系列的状态（属性）以及行为集合。状态和行为在Java中分别用实例变量和方法实现。

抽象化

  • 抽象化 - 用户（客户端）只知道对象上提供的操作，而不知道这些操作是如何实现的。

封装

  • 封装 - 变量和方法被包装在一个单一的单元中。变量被保持为私有的，只能通过设置器和修改器方法从外部访问。

总结

  • 本部分描述了面向对象编程的三个基本概念：类与对象、抽象化和封装。这些概念是构建一个结构良好和易于管理的代码基础的核心。

对象层次结构（Hierarchy of Objects）

  • 在面向对象编程（OOP）中，对象可以通过“类”来形成层次结构。例如，商店和房屋都是建筑物的类型。

建筑物类（Building）

  • 属性：所有建筑物都有地址和所有者这些共同属性。
  • 行为：建筑物的行为可能包括获取地址和所有者的操作。

商店子类（Shop）

  • 特定属性：商店可能有特有的属性，如员工数和年营业额。
  • 特定行为：商店特有的行为可能包括计算年营业额。

房屋子类（House）

  • 特定属性：房屋可能有特有的属性，如卧室数和是否有车库。
  • 特定行为：房屋的行为可能包括查询卧室数和车库情况。

类层次结构总结

  • 类的层次结构允许我们在适当的层级指定属性。例如，所有建筑物都有地址和所有者，但是商店和房屋有它们特有的属性和行为。这种方式提高了代码的可重用性和可维护性。

创建子类（Creating the subclass）

House 类的定义

  • public class House extends Building { ... }

调用父类构造方法

  • super(add, own);

  • 还应该包括访问器(getter)和修改器(setter)方法，以获取和设置私有属性。

总结

  • House类展示了如何在Java中创建一个子类并继承父类的属性。它也通过其构造方法展示了如何利用super关键字调用父类的构造方法，以及如何设置子类特有的属性。这是面向对象编程中继承和封装概念的典型应用。

关键概念（Key Concepts）

  • 本示例介绍了面向对象编程中更多的关键概念：

继承（Inheritance）

  • 属性和方法的共享 - 继承允许在层次结构中排列的类之间共享属性和方法。
  • 子类继承 - 子类将从其超类继承所有属性和方法。
  • 属性和方法覆写 - 如果需要，可以在子类中覆写属性和方法。

Shop和House类

  • 独立属性 - 它们还拥有自己的个别属性。

多态性（Polymorphism）

  • 多态性 - 是指类层次结构产生的“是一个（is-a）”关系。
    • 例如：一个Shop“是一个（is a）”Building，Shop对象可以访问Building类的所有方法。

总结

  • 继承和多态性是面向对象编程的核心概念。通过继承，子类可以重用超类的代码，而多态性允许我们使用通用的代码来处理不同类型的对象，从而使代码更加灵活和可扩展。

静态变量和方法的概念

  • 到目前为止，我们使用的变量和方法都是实例变量和实例方法。

    • 实例变量：类的每个实例都有变量的一个副本。
    • 实例方法：这些方法适用于类的实例。

  • 静态变量和静态方法属于类本身，而不是属于特定的实例。

    • 它们通常是“辅助”方法和相关数据，为应用程序提供通用服务。
    • 例如，在我们的Anytown项目中，我们可能希望跟踪可以生成的建筑物总数。

总结

  • 在面向对象编程中，静态变量和方法为整个应用程序提供了共享的数据和服务，而实例变量和方法则为每个对象提供了独立的状态和行为。理解静态和实例之间的区别对于设计有效的类结构和编写可维护的代码非常重要。

静态变量和方法（Static Variables and Methods）

  • 在Java类中定义了静态变量和静态方法。

Building 类

  • public class Building { ... }
    • 代表一个名为Building的类。

静态变量

  • private static int numBuildings = 0;
    • 这是一个私有的静态变量，代表建筑物的数量，初始值为0。
    • 作为类变量，numBuildings在所有Building对象间共享。

静态方法

  • public static int getNumBuildings() { return numBuildings; }
    • 这是一个公共的静态方法，用于返回建筑物的数量。
    • 作为类方法，getNumBuildings可以不通过对象实例而直接被类名调用。

总结

  • 静态变量和方法是类级别的成员，它们与类的任何实例无关，而是属于类本身。在这个例子中，numBuildings变量可以用来追踪创建的Building对象的总数，而getNumBuildings方法提供了获取这个总数的途径。

  • 这些步骤为我们提供了一个坚实的基础，了解了OOP和UML的基本概念，并在Java中实现了一个简单的应用程序。下一步，将涉及更高级的Java编程概念，以进一步提高我们的技能和应用程序的复杂性。
