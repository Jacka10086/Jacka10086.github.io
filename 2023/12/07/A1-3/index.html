A1.3 Static Variables and Methods (A1.3 静态变量与方法)

  • 到目前为止，我们使用的变量和方法一直是实例变量和实例方法。

    • 这意味着每个类的实例都有变量的副本。
    • 方法被应用到类的实例上。

  • 静态变量和静态方法属于类本身，而不是特定的实例。

    • 它们通常是“助手”方法和提供一般服务的相关数据。
    • 例如，在我们的Anytown项目中，我们可能希望跟踪可以生成的建筑物的总数。

总结：本节介绍了静态变量和静态方法的概念。不同于实例变量和方法，静态变量和方法属于类本身，为所有实例共享。这些通常用于执行跨多个实例共享的任务，如维护全局状态或提供工具函数。

Definition of a Building (建筑的定义)

  • 在UML（统一建模语言）中，Building类的定义如下：

    • address : String
    • owner : String
    • numBuildings : integer (静态变量)
    • + Building() (构造函数)
    • + Building(owner : String, address : String) (带参数的构造函数)
    • + getOwner(newOwner : String) : String (获取所有者的方法)
    • + setOwner() : String (设置所有者的方法)
    • + getNumBuildings() : integer (静态方法)

  • 按照惯例，在UML中，静态变量和方法是通过下划线表示的。

  • 构造函数也已添加到UML图中，以完成类的定义。

总结：这部分内容展示了如何在UML中定义一个具有实例变量和静态变量的类。静态变量在类所有实例之间共享，通常用来表示类级别的状态或属性，如此例中的numBuildings。构造函数和方法用于创建对象实例和访问对象的属性和行为。

Static Variables and Methods (静态变量和方法)

  • Java中的Building 类展示了静态成员的使用：
    • private static int numBuildings = 0; 表明 numBuildings 是一个静态变量，它属于类而不是类的任何特定实例。它通常用于跟踪所有实例共有的数据，如所有建筑物的计数。
    • public static int getNumBuildings() { return numBuildings; } 定义了一个静态方法，它允许外部代码获取numBuildings 的值而无需创建类的实例。

总结：在这个例子中，静态变量和方法是类级别的组件，意味着它们可以在没有类实例的情况下被访问。这对于维护全局状态或提供工具功能很有帮助，如在本例中维护所有建筑物的总数。

Scenario (情景)

  • 对Anytown项目进行以下修改：
    1. 在Building类中添加一个静态整型变量numBuildings，并提供一个静态方法getNumBuildings()来返回其值。同时，更新两个构造函数方法，使得每次创建Building实例时，numBuildings的值都会增加。
    2. 在AnytownTest类中添加代码，以便在最终输出中显示创建的建筑物数量。
    3. 为House和Shop类也提供类似的静态变量和方法，并适当修改它们的构造器。
    4. 更新AnytownTest类，修改最终输出消息，使其显示类似于以下形式的信息：”There are a total of 6 buildings of which 4 are houses and 2 are shops”。

总结：本情景要求更新和扩展Java类以包含静态变量和静态方法，用于跟踪和显示创建的建筑物总数。这要求修改现有的构造函数，并在测试类中添加新的逻辑来展示累计信息。

Next Steps (下一步)

  • 到目前为止，我们已经：

    • 确定了面向对象编程（OOP）的主要概念。
    • 引入了UML作为一种建模技术来帮助指定对象及其属性和操作。
    • 在Java中实现了我们的初始对象层次结构。
    • 使用IntelliJ IDEA构建并运行了应用程序。

  • 为了进一步学习，我们需要更深入了解Java，特别是：

    • 循环结构，条件语句。
    • 变量，数据类型，数组和字符串。
    • 输入，输出和文件操作。
    • 生成随机值（对测试很有用）。

总结：这一部分概述了学习面向对象编程接下来的步骤。我们已经建立了基础，接下来需要深入了解Java的更多细节，这将帮助我们编写更复杂的程序并进行有效的测试。
