COM410 编程实践 (COM410 Programming in Practice)

T8. 异常处理 (Exception Handling)

  • 异常的定义: 在Java中，异常是程序执行过程中发生的不期望或意外的事件，它会打断程序正常的指令流。

    • 中文总结: 异常是Java程序运行中的意外事件，需要被妥善处理以维持程序的正常运行。

  • 异常和错误的区别:

    • 异常 (Exception): 一个合理的应用可能会尝试捕获的条件。
      • 异常通常是由程序的错误导致的，可以通过程序代码捕获和处理。
      • 创建了包含异常名称和描述的Exception对象。
      • 中文总结: 异常可以通过程序代码捕获和处理，通常不会导致程序终止。
    • 错误 (Error): 程序员控制范围之外的严重问题，合理的应用不应该尝试捕获。
      • 错误通常指出了更严重的问题，如系统崩溃或资源不足，这些情况下程序通常无法恢复。
      • 中文总结: 错误是严重的问题，通常不应该或无法通过程序代码捕获和处理。

总结 (Summary)

  • 本节介绍了在Java编程中处理异常的重要性和方法。
  • 区分了异常和错误，指出了它们的不同处理方式。
  • 了解异常和错误有助于编写更稳健和可靠的Java程序。
  • 中文总结: 异常处理是Java编程实践中的一个关键部分，它帮助程序在面对意外事件时保持稳定和安全。

异常的成因 (What causes exceptions?)

用户输入错误 (Invalid user input)

  • 用户可能会输入不符合程序预期的数据，如错误的数据格式或范围。

设备故障 (Device failure)

  • 硬件故障如内存损坏、存储器读写错误等。

网络连接丢失 (Loss of network connection)

  • 网络不稳定或中断会导致程序无法继续执行网络相关的操作。

打开不可用文件 (Opening an unavailable file)

  • 尝试打开不存在或者无权限访问的文件。

代码错误 (Code errors)

  • 程序中的bug或逻辑错误导致程序无法正常运行。

总结：
在软件开发和使用过程中，异常可以由多种原因引起。无效的用户输入、设备故障、网络连接丢失、尝试打开不可用的文件以及代码错误都是常见的异常成因。了解这些成因有助于开发者在设计程序时考虑更全面的错误处理和异常捕捉机制，以提升软件的稳定性和用户体验。

Java异常层次结构 (Java Exception Hierarchy)

基类对象 (Object)

  • Java中所有类的根基类。

可抛出基类 (Throwable)

  • 表示所有错误和异常的超类。

异常 (Exceptions)

  • 表示程序可以处理的条件。
受检查异常 (Checked Exceptions)
  • 需要显式捕获或声明的异常，例如IO异常或编译时异常。
未受检查异常 (Unchecked Exceptions)
  • 程序运行时可能出现的异常，如运行时异常或空指针异常。

错误 (Error)

  • 表示不期望程序处理的严重问题。
虚拟机错误 (Virtual Machine Error)
  • 表示JVM遇到的问题，如系统崩溃或资源不足。
断言错误 (Assertion Error etc)
  • 表示当断言失败时抛出的错误。

总结：
Java异常层次结构是Java语言定义的错误和异常的分类方式。在顶层是Throwable类，它分为两个分支：Error和Exceptions。Error类代表了系统级别的严重问题，通常不由程序控制；而Exceptions则分为受检查异常和未受检查异常，前者需要在编译时处理，后者通常是运行时出现的问题。了解这个层次结构对于Java程序员来说是至关重要的，它能帮助他们更好地设计和处理程序中可能出现的各种问题。

受检异常与非受检异常 (Checked and unchecked exceptions)

受检异常 (Checked exceptions)

  • 编译时发生的异常。
  • 必须在代码中处理，否则会生成编译错误。

非受检异常 (Unchecked exceptions)

  • 不会生成编译错误的异常。

异常类型 (Types of Exceptions)

用户定义异常 (Use-Defined Exception)
  • 程序员自定义的异常类型。
内建异常 (Built-in Exception)
受检异常 (Checked Exceptions)
  • ClassNotFoundException：找不到类异常。
  • InterruptedException：线程中断异常。
  • IOException：输入输出异常。
  • InstantiationException：实例化异常。
  • SQLException：SQL操作异常。
  • FileNotFoundException：文件未找到异常。
非受检异常 (Unchecked Exceptions)
  • ArithmeticException：算术运算异常。
  • ClassCastException：类转换异常。
  • NullPointerException：空指针异常。
  • ArrayIndexOutOfBoundsException：数组索引越界异常。
  • ArrayStoreException：数组存储异常。
  • IllegalThreadStateException：线程状态非法异常。

总结：
在Java编程中，异常分为受检异常和非受检异常。受检异常必须在编写代码时进行处理，否则会在编译时报错；而非受检异常则不需要显式处理，它们通常表明存在程序运行时错误。Java也允许用户自定义异常，以满足特定的业务需求。这些异常类型的正确处理是编写健壮程序的关键。

异常处理 (Exception Handling)

try…catch 语句块 (The try…catch block)

  • try...catch语句块是Java语言中处理异常的基本结构。当程序执行到try块内的代码时，如果发生异常，程序会跳转到catch块，执行异常处理代码。

代码结构 (Code Structure)

  • try { ... }：这里的...表示可能抛出异常的代码区块。在实际编程中，应当只将有可能抛出异常的代码放入try块中。
  • catch (ExceptionType e) { ... }：catch块紧跟在try块后面，用来捕获并处理异常。ExceptionType是要捕获的异常类型，而e是该异常的实例。
  • 在try块中的任何代码一旦抛出异常，执行流将直接跳转到catch块，catch块中的代码用来处理异常。

总结 (Summary)

  • 屏幕截图介绍了Java中处理异常的基础结构try...catch语句块。try块包含可能引发异常的代码，而catch块包含处理这些异常的代码。这是Java编程中异常控制流程的核心，旨在为程序提供更健壮的错误处理机制。
