COM410 Programming in Practice (COM410 编程实践)

T6. Developing Secure Code (T6. 开发安全代码)

  • 程序员必须在他们的代码中包含故障安全措施，以确保程序对用户的安全。
  • Java为你管理内存，检查数组索引的有效性，并且是类型安全的。
  • 然而，一个错误可能会使代码变得脆弱。
  • 通过包括预期错误的检查，你可以练习故障安全编程。

Secure Implementation (安全实施)

  • 在实现抽象数据类型（ADT）时，应该提出以下问题：
    1. 如果构造函数没有完全执行会发生什么？
    2. 如果客户端尝试在创建集合类型时超过存储限制会发生什么？

总结：本部分强调了在实践编程时开发安全代码的重要性。它提出了确保代码安全的关键问题，并指导程序员如何通过预防措施来避免潜在的安全隐患。

Securing the ArrayBag (保护 ArrayBag)

  • 在ArrayBag类中，我们希望通过以下方式提高代码的安全性：

    • 为类添加两个额外的数据字段：
      • private boolean initialised = false; 为新的实例变量，表示ArrayBag是否已初始化。
      • private static final int MAX_CAPACITY = 10000; 为新的类变量，定义了ArrayBag的最大容量限制。

  • 为了确保客户端不能创建一个过大的ArrayBag，构造函数将检查期望容量（作为参数传递）是否超过了MAX_CAPACITY。

  • 如果请求的容量过大，构造函数可以抛出异常。

总结：在实现数据结构时考虑安全性非常重要。通过在ArrayBag类中引入初始化标志和最大容量限制，我们可以防止错误的使用方式，从而增强代码的健壮性。

Securing the Implementation (确保实施的安全)

Securing the Implementation (确保实现的安全性)

Check for Initialisation (检查初始化)

  • 由于initialised会在多个方法中检查，可以通过定义一个私有方法来避免重复代码。
    • 私有方法 checkInitialisation() 会检查变量是否已初始化，如果没有，则抛出一个 SecurityException 异常。
  • 这个方法可以在类中的其他方法执行操作前被调用。
    • 例如，addNewEntry 方法在执行其操作之前会先调用 checkInitialisation()。

Exceptions Handling (异常处理)

  • 注意：SecurityException 和 IllegalStateException 是标准运行时异常，它们位于 java.lang 包中。

总结：本页内容介绍了如何通过定义一个私有方法来避免在实现中多次检查初始化的重复代码，并且解释了如何通过在其他方法中调用这个私有方法来确保对象在执行操作前已正确初始化。同时，还强调了 SecurityException 和 IllegalStateException 是标准的运行时异常，它们用于处理程序运行中的错误状态。

Securing the Implementation (确保实现的安全性)

Writing Secure Java Code (编写安全的Java代码)

  • 为了编写更安全的Java代码，有一些常见的指导原则：
    1. 将一个类的大多数（如果不是全部）数据字段声明为私有（private），任何公共数据字段都应该是静态（static）和最终的（final），且具有恒定的值。
    2. 不要过分追求巧妙（Don’t be too clever!）。
    3. 避免重复代码；相反，应将这类代码封装到私有方法中，以便其他方法可以调用。
    4. 当构造函数调用一个方法时，确保它不能被覆盖（overridden）。

Class Declaration Revision (类声明修订)

  • 因此，我们修订了 ArrayBag 类的声明：
    • public final class ArrayBag<T> implements BagInterface<T> {
  • 声明为 final 的类，没有其他类可以继承它。
  • 一个 final 类因为不能被继承来改变其行为，因此更加安全。

总结：本节内容强调了编写安全Java代码的重要性，并提出了一些基本指导原则，比如使用私有字段，避免过度巧妙的代码，避免代码重复，并保证构造函数调用的方法不可被覆盖。同时，通过修订类声明为 final，增加了代码的安全性，因为这样的类不能被继承，从而避免了潜在的安全风险。

Scenario (情景)

Secure ArrayBag Implementation (确保ArrayBag实现的安全)

  • 确保 ArrayBag 类实现的安全性，通过添加以下内容：

    1. 添加 initialised 和 MAX_CAPACITY 变量。
    2. 更新构造函数以检查所请求的容量是否被允许，并在不允许的情况下抛出异常。如果创建了Bag，应将 initialised 类变量设置为 true。
    3. 添加新的私有方法 checkInitialisation() 以测试 initialised 的值，并在其为 false 时抛出异常。
    4. 在 addNewEntry() 方法的第一个动作中调用新的 checkInitialisation() 方法。
    5. 更新类头部，将 ArrayBag 类定义为 final。

  • 检查在进行这些更改后你的实现是否仍然像之前一样工作。

总结：本节内容描述了如何通过添加初始化检查和最大容量变量，更新构造函数，添加新的私有方法进行初始化检查，以及将类声明为 final 来加强 ArrayBag 类的安全性。这些步骤旨在确保在进行必要的安全更新后，类的功能依然保持原有的工作方式。
