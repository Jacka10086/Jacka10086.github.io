COM410 Programming in Practice (COM410 实践编程)

T7. Recursion (T7. 递归)

递归是编程中的一种重要概念，涉及到自我引用的过程。

Recursion (递归)

What is recursion? (什么是递归？)

  • 递归是指在定义中自我引用的过程。

Why learn recursion? (为什么学习递归？)

  • 它代表了一种新的思考方式。
  • 提供了一个强大的编程范式。
  • 能够洞察计算的本质。

Naturally self-referential computational artifacts (自然的自引用计算产物)

  • 包含文件夹的文件系统中的文件夹。
  • 分形图形模式。
  • 分而治之的算法。

总结：递归是一种在程序设计中自我引用的技术，它不仅代表一种新的思维模式，而且是一种强大的编程范式，能够帮助我们深入理解计算的本质。递归在很多自然和计算过程中广泛存在，如文件系统、分形图案和分而治之的算法等。

What is Recursion (in programming)? (什么是编程中的递归？)

递归发生在方法在其自己的定义中调用自己的情况。

  • 一个调用自身的方法是一个递归方法。
  • 递归方法有两个主要部分：
    1. Base Case (基础情况): 非常重要，因为如果没有基础情况，方法将永远不会停止（最小的相同问题，已知解决方案）。
    2. Recursive Call (递归调用): 一个递归方法的递归调用（解决更小的相同问题）。

图表解释

  • 图表展示了递归方法的结构：开始于一个初始调用，然后进行工作，其中包括递归调用和检查基础情况。一旦达到基础情况，方法就会返回其结果。

总结：在编程中，递归是一种方法自引用自己以解决问题的技术。一个递归方法必须包含基础情况以避免无限循环，以及递归调用自身以解决问题的较小部分。理解递归的工作原理对于编写高效和简洁的代码至关重要。

递归方法的示例：倒计时（Recursive Method Example: The Countdown）

  • 递归方法用于打印从给定开始值递减到1的所有整数
    • 基本情况（Base Case）
      • 当 start > 1 时，方法会继续调用自身，直到 start 为1
    • 递归调用（Recursive Call）
      • countDown(start - 1); 表示方法每次调用自己时都会将 start 的值减少1

代码解析（Code Analysis）

  • public static void countDown(int start) 是方法声明
    • public 表明该方法是公开的，可以被任何对象或方法调用
    • static 意味着不需要创建类的实例就可以调用该方法
    • void 表明该方法不返回任何值
    • (int start) 是方法的参数，表示从 start 开始倒数
  • System.out.println(start); 用于打印当前的 start 值
  • if (start > 1) 判断是否满足基本情况
    • 如果 start 大于1，则方法会递归调用自己，每次调用时 start 减1
    • 当 start 等于1时，递归结束，因为不再满足 start > 1 的条件

图表解释（Chart Explanation）

  • 代码块被高亮显示，突出显示了基本情况和递归调用的部分
  • 红色箭头标注了代码的关键部分，方便观众理解代码的流程

总结（Summary）

本幻灯片展示了一个简单的递归方法，用于从任意给定的整数 start 倒计时到1。通过递归调用自身并在每次调用中将数值减少1，直到基本情况满足为止。这是理解递归概念的一个典型例子。

追踪递归方法的执行过程（Tracing a Recursive Method Execution）

  • 追踪一个对 countDown(3) 的调用
    • 参数3被复制到参数 start，接下来执行方法内的语句：
      • System.out.println(3); 打印出数字3
      • if (3 > 1) 判断条件为真，执行递归调用 countDown(3 - 1);

递归调用流程（Recursive Call Process）

  • (a) 输出包含“3”的行，并进行递归调用 countDown(2)
  • (b) 接着，countDown(2) 会输出2，并调用 countDown(1)
  • (c) 最后，countDown(1) 会输出1，并因为不满足 if (start > 1) 的条件而停止递归

图表详细解释（Detailed Chart Explanation）

  • 图表中展示了递归调用的三个阶段：
    • (a) 显示数字3并调用 countDown(2)
    • (b) 显示数字2并调用 countDown(1)
    • (c) 显示数字1并结束递归
  • 红色框内是递归方法的代码，旁边的白色框展示了递归调用的具体过程

总结（Summary）

本幻灯片通过追踪 countDown 方法的执行流程，详细解释了递归调用的步骤。从 countDown(3) 开始，递归地调用自身，每次递减参数的值，直到参数为1时停止调用。这个过程展示了递归如何逐步解决问题，直至达到基本情况，结束递归。

继续追踪递归方法的执行（Continuing to Trace a Recursive Method Execution）

  • 原始调用 countDown(3) 的执行被挂起，直到 countDown(2) 的结果已知为止
    • 尽管在递归调用之后没有任何操作， countDown(3) 依然等待 countDown(2) 完成

追踪 countDown(2) 的执行（Tracing the Execution of countDown(2)）

  • countDown(2) 执行以下语句：
    • System.out.println(2); 打印出数字2
    • if (2 > 1) 条件判断为真，执行递归调用 countDown(2 - 1);

递归调用细节（Recursive Call Details）

  • b) 输出包含“2”的行，并进行递归调用 countDown(1)
    • 这一步是 countDown(3) 调用链中的第二步
    • countDown(1) 将会是接下来的递归调用，这是最后一次调用，因为它将不满足条件 if (start > 1)

图表解释（Chart Explanation）

  • 图表中的 (b) 显示了递归调用 countDown(2) 的过程，包括输出和下一步的递归调用
  • 红色框标注了正在执行的代码块，展示了从 countDown(3) 到 countDown(1) 的递归调用序列

总结（Summary）

本幻灯片继续展示了 countDown 方法的递归调用过程。从 countDown(3) 开始，递归调用被挂起直到 countDown(2) 执行结束。通过这种方式，递归方法能够保持执行的顺序和层次，直至完成所有调用。

完成递归方法的追踪（Completing the Trace of a Recursive Method）

  • 继续追踪，countDown(1) 执行以下语句：
    • System.out.println(1); 打印出数字1
    • 由于 if (1 > 1) 的条件不成立，没有更多的递归调用发生

countDown(1) 的执行（Execution of countDown(1)）

  • c) 输出包含“1”的行，并且没有其他递归调用发生
    • 这表示递归调用已经到达基底情况，并停止进一步调用

图表解释（Chart Explanation）

  • 图表中的 (c) 展示了递归调用 countDown(1) 的过程，只有输出没有递归调用
  • 红色框标注了执行的最后阶段，显示了递归方法最终如何结束调用序列

总结（Summary）

本幻灯片展示了递归方法 countDown 的最后一个调用 countDown(1) 的执行过程。当递归到达基底情况，即 if (start > 1) 不再成立时，递归调用停止，这标志着递归过程的完成。

使用递归 (Using Recursion)

  • 计算和: 对于任意正整数 n，计算 1 + 2 + ... + n 的和。

    • 中文总结: 使用递归方法计算数列的和。

  • 问题的输入: 问题的输入是整数 n。

    • 中文总结: 输入 n 是计算数列和的起点。

  • 基础情况: 小值 n 的情况下，和是容易知道的。

    • 基础情况是递归结束的条件，例如 n = 1 时，和为 1。
    • 中文总结: 基础情况是递归停止的信号。

  • 更小版本的任务: 计算 1 + ... + (n-1) 的和，这样逐步逼近原问题的解。

    • 中文总结: 通过解决规模较小的问题来逐步解决原问题。

  • 递归定义: 根据以上逻辑，可以定义 sumOf() 函数。

    • 中文总结: sumOf() 函数定义了如何递归计算数列的和。

图表解释 (Explanation of the Diagram)

  • 数学符号: Σ 表示求和，即从 i=1 到 n 的连续整数和。
    • 中文总结: Σ 符号用于表达连续整数的求和。

总结 (Summary)

  • 介绍了使用递归来解决数列求和问题。
  • 通过递归，每次处理一个更小的问题，直至基础情况。
  • 递归是数据结构和算法中常用的技术。
  • 中文总结: 详细阐述了递归的概念及其在计算数列和中的应用。

追踪 sumOf() 函数 (Tracing sumOf() Function)

  • 假设我们调用方法: 使用语句 sumOf(3) 调用方法。

    • 中文总结: 调用 sumOf() 函数计算 1 + 2 + 3 的和。

  • 计算过程: 如下所示：

    • a) sumOf(3) 是 3 + sumOf(2)，此时 sumOf(2) 开始执行，sumOf(3) 暂停执行。
    • b) sumOf(2) 是 2 + sumOf(1)，此时 sumOf(1) 开始执行，sumOf(2) 暂停执行。
    • c) sumOf(1) 返回 1，因为达到了基础情况，暂停的执行恢复。
    • d) sumOf(2) 接着返回 2 + 1 = 3。
    • e) sumOf(3) 最终返回 3 + 3 = 6。
    • 中文总结: 这个过程演示了递归如何通过分步计算小问题的和，最终得到总和。

总结 (Summary)

  • 本页内容追踪了 sumOf() 函数的执行过程。
  • 通过递归调用，函数逐层计算直到基础情况，然后逐层返回结果。
  • 这个过程是理解递归函数如何工作的一个很好的例子。
  • 中文总结: 通过追踪 sumOf() 函数的执行，我们可以清晰地看到递归如何一步步解决问题并计算出总和。
