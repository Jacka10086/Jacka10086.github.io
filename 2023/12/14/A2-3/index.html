COM410 编程实践

A2.3 输入、输出和文件（Input, Output and Files）

概览

  • 这是Ulster大学COM410课程的幻灯片，讲解了编程中的输入、输出以及文件处理。

文本输出（Text Output）

Java输出概述

  • 在Java中，输出是通过 System.out 对象提供的，该对象具有两个用于输出的方法：println() 和 print()。

输出方法说明

  • System.out.println()：在输出的文本后添加一个换行符，使得每次输出都从新的一行开始。
  • System.out.print()：在输出文本后不添加换行符，下一次输出将紧接着上一次的输出在同一行显示。

格式化文本输出（Formatted Text Output）

  • Java中的格式化输出：Java的格式化输出是通过System.out.printf()方法提供的。

    • 这个方法允许程序员以格式化的方式打印输出，更加美观且易于阅读。

  • 常见形式：最常用的形式是采用格式字符串和一系列参数。

    • System.out.printf("格式化字符串", 参数列表);
    • 这允许程序员指定输出的格式，如数字的小数位数、字符串的对齐方式等。

  • 示例代码：

    • public static void main(String[] args) {
  String module = "COM410";
  int classSize = 180;
  double averageMark = 61.25;

  System.out.printf("Welcome to %s%n", module);
  System.out.printf("We have %d students and the average mark is %f%%", classSize, averageMark);
}
    • 以上代码展示了如何使用不同的格式化选项来控制输出的宽度和精度。

  • 格式化输出：

    • The average mark is 61.250000%
    • Average mark to 2 decimal places is 61.25
    • In 6 characters with one decimal place is 61.3
    • Padded with leading zeroes is 0061.3
    • 这些输出示例表明了如何控制数字的显示宽度以及如何用前导零填充。

中文总结

在Java中，字段宽度是格式化输出时一个重要的概念，它允许程序员定义输出字符串的总长度。通过指定宽度，我们可以控制数字或文本的显示，使输出看起来更加规整。例如，可以指定小数点后的位数，或者为数字添加前导零。在给出的代码示例中，我们看到了如何使用字段宽度来格式化数字，并根据需要调整输出格式。

对齐设置（Field Width）

  • 字段宽度的应用：字段宽度可用于任何输出类型。

    • 字段宽度不仅可以指定宽度，还可以控制文本的对齐方式。

  • 默认对齐方式：输出默认在字段宽度内右对齐，使用-标志可以实现左对齐。

    • 这意味着如果不指定对齐标志，文本将默认填充至字段的右侧。
    • 在左对齐时，-标志将文本推向字段的左侧。

  • 示例代码：

    • public static void main(String[] args) {
  System.out.print("*");
  System.out.printf("%50s", "Right justified to 50 character places");
  System.out.print("*\n");
  System.out.printf("%-50s", "Left justified to 50 character places");
  System.out.println("*");
}
    • 代码示例展示了如何使用Scanner对象读取用户的名字和年龄。

  • 输出示例：

    • 用户输入名字Adrian和年龄21，程序输出：
      • What is your name? > Adrian
      • You entered Adrian
      • What is your age? > 21
      • You entered 21
    • 这显示了程序是如何交互式地询问并获取用户的输入信息。

中文总结

在Java程序中，当需要获取用户通过键盘输入的信息时，我们会使用Scanner类。这个类提供了多种方法来读取不同类型的数据。示例代码清楚地说明了如何创建一个Scanner对象，并使用nextLine()和nextInt()方法来分别获取用户的字符串输入和整数输入。这种方法在编写需要用户交互的程序时非常有用。

键盘输入（Keyboard Input）

  • 引入Scanner类：为了从键盘读取输入，需要导入Scanner类并实例化它。

    • 导入Scanner类使得程序能够处理来自标准输入（如键盘）的数据。

  • 实例化Scanner对象：

    • 创建Scanner对象的语法为Scanner 变量名 = new Scanner(System.in);。
    • 这个对象可以调用多种方法来读取不同类型的输入数据。

  • 使用Scanner方法：利用nextLine()或nextInt()方法从用户处收集数据。

    • nextLine()读取一行文本，直到遇到换行符。
    • nextInt()读取下一个整数值。

  • 示例代码及输出：

    • import java.util.Scanner;

public class Demo {
  public static void main(String[] args) {
    Scanner keyboard = new Scanner(System.in);

    System.out.print("What is your name? > ");
    String yourName = keyboard.nextLine();
    System.out.printf("You entered %s\n", yourName);

    System.out.print("What is your age? > ");
    int yourAge = keyboard.nextInt();
    System.out.printf("You entered %d\n", yourAge);
  }
}
    • 以上代码片段包含了获取用户输入的部分以及格式化输出赛事结果的示例。

中文总结

本场景要求修改TownChallenge项目，使其可以接受用户输入的城镇名称，并且能够以格式化的表格形式输出赛事结果。这需要使用Java的Scanner类来收集用户输入，以及使用System.out.printf()方法和字段宽度来创建整洁的表格。示例代码展示了如何实现这一要求，包括如何收集用户输入和如何格式化输出赛事结果。

使用文本文件输入（Using Text Files for Input）

  • 导入必需的类：使用文本文件作为输入时，需要导入Scanner和File类。

    • Scanner类用于读取文本文件中的数据。
    • File类用于表示文件和文件路径。

  • 创建文件实例：

    • 使用new File("文件路径");创建一个文件实例。
    • 示例中的文件路径是"src/textfile.txt"。

  • 异常处理：注意在使用文件时可能需要处理异常。

    • 在方法签名中添加throws Exception来声明可能抛出的异常。

  • 示例代码：

    • import java.util.Scanner;
import java.io.File;

public class Demo {
  public static void main(String[] args) throws Exception {
    File file = new File("src/textfile.txt");
    Scanner fileInput = new Scanner(file);
    
    while (fileInput.hasNextLine()) {
      System.out.println("Reading... " + fileInput.nextLine());
    }
  }
}
    • 这段代码示例演示了如何向文件中写入数据，并且在文件末尾追加内容，而不是覆盖原有内容。

    • 如果代码运行两次，则每次运行都会在文件的末尾追加相同的内容。

中文总结

使用FileWriter类可以将字符串数据写入文本文件。在Java中，通过实例化FileWriter并指定文件路径及是否追加模式，可以实现文件的写入。示例代码清楚地说明了如何打开文件、写入字符串以及关闭文件流。注意，如图所示，如果设置为追加模式，同样的数据可以被多次写入到文件的末尾。

挑战（Challenge）

  • TownChallenge项目的修改：在你的TownChallenge项目中，需要对TownChallenge类进行修改，以实现以下功能：
    1. 读取城镇名称：
      • 应用程序应从名为towns.txt的输入文件中读取八个城镇的名称，并将它们存储在towns数组中。
    2. 询问结果集数量：
      • 读取城镇集合后，应用程序应询问用户需要多少组结果，并从键盘接受这个数值。
    3. 生成随机结果集：
      • 程序应生成与用户请求的数量相等的随机结果集。注意所有结果集都应包含相同的比赛。
    4. 结果写入文件：
      • 所有结果（包括主场/平局/客场摘要）都应写入名为results.txt的文件，每次程序运行时，新输出都添加到文件末尾。结果文件中的信息不需要格式化（即不需要表格形式）。

中文总结

TownChallenge项目的这项挑战要求开发者对程序进行扩展，以便能够从文件中读取数据，并根据用户的需求生成结果数据。这涉及到文件的读写操作，数组的使用，以及与用户的交互。特别需要注意的是，输出到结果文件的数据不需要特别格式化，只需按顺序追加即可。
