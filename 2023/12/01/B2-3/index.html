B2.3 排序在Java中的应用

  • Ulster University的COM410课程专注于实践中的编程技能。
  • B2.3单元专门讲解Java中的排序算法和技术。

Java Provision for Sorting

  • Java为排序提供了全面的支持，包括内置的方法和算法。
  • 排序算法的时间复杂度通常是O(n log n)，这意味着它们的效率随着处理的数据量增长而缓慢增加。
  • Java使用的是合并排序、快速排序和TimSort（取决于JDK的版本和使用情况）的混合体。
    • 这些算法各有优势，适用于不同的数据结构和排序需求。
  • 需要考虑四种情况：
    • 原始数据类型的数组：简单数据类型（如int、char等）的数组排序。
    • 集合：Java集合框架中的对象，如List和Set。
    • 基于字段的自定义对象排序：涉及到比较对象的特定属性。
    • 灵活排序：根据需求定制排序逻辑，如使用Comparator接口。

总结：本单元概述了Java在排序算法方面的支持，包括不同场景下的应用和考虑事项。介绍了Java内置的排序方法和算法，以及如何根据不同的数据类型和需求选择合适的排序策略。

原始数据类型数组的排序（Arrays of Primitives）

使用Arrays.sort()方法

  • Arrays.sort()方法可以对数值型原始数据类型（例如int）和字符串（String）数组进行排序。
  • 对于数值型数组，该方法默认按照升序排列。
  • 对于字符串数组，排序依据字符串的自然顺序，通常是按照字典序。

示例代码

  • 对于整数数组：int[] intArray = { 3, 6, 2, 9, 1 }; Arrays.sort(intArray);
    • 这行代码将会把intArray数组中的元素按照从小到大的顺序排序。
  • 对于字符串数组：String[] strArray = { "one", "two", "three", "four", "five" }; Arrays.sort(strArray);
    • 这行代码将会把strArray数组中的字符串按照字典序排序。

总结：本节内容介绍了如何使用Arrays.sort()方法对原始数据类型的数组进行排序。通过示例代码，我们看到了如何对整数数组和字符串数组进行默认的升序排序。这是Java编程中常用的一个数组排序方法，简单且易于使用。

原始数据类型数组的降序排序（Arrays of Primitives）

提供比较器实现降序

  • 使用Arrays.sort()方法时，可以通过提供一个Comparator作为第二个参数来实现数组的降序排序。
  • 重要的是要注意，只有对象类型的数组（比如Integer或String对象数组）才能使用比较器进行排序。

示例代码

  • 对于整数对象数组：Integer[] integerArray = { 3, 6, 2, 9, 1 }; Arrays.sort(integerArray, Comparator.reverseOrder());
    • 代码中Comparator.reverseOrder()使整数对象数组按照降序排列。
  • 对于字符串数组：String[] strArray = { "one", "two", "three", "four", "five" }; Arrays.sort(strArray, Comparator.reverseOrder());
    • 同样的，字符串数组也会按照反向字典序即降序排列。

总结：这一部分说明了如何利用Comparator来改变数组排序的顺序。通过提供Comparator.reverseOrder()比较器，我们可以使得对象类型的数组按照降序排序。这在需要逆序处理数组元素时非常有用。

集合的排序（Collections）

使用Collections.sort()方法

  • Collections.sort()方法用于对如ArrayList和LinkedList这样的集合进行排序。
  • 此方法只适用于对象类型的集合，例如Integer或String的集合。

示例代码

  • 创建并添加元素到ArrayList：ArrayList<Integer> integerList = new ArrayList<Integer>(); integerList.add(40); integerList.add(20); integerList.add(30);
    • 这段代码初始化了一个ArrayList，并添加了几个整数元素。
  • 升序排序：Collections.sort(integerList);
    • 默认情况下，Collections.sort()会将集合中的元素按照升序排列。
  • 降序排序：Collections.sort(integerList, Comparator.reverseOrder());
    • 提供Comparator.reverseOrder()作为参数，可以将集合中的元素按照降序排列。

总结：这一部分介绍了如何使用Java的Collections.sort()方法对集合进行排序。首先，我们了解了如何添加元素到集合，并且默认实现升序排序。接着，通过使用比较器，示例代码展示了如何对集合元素进行降序排序。这些操作对集合的排序提供了灵活的控制。

自定义对象的排序（Custom Objects）

实现Comparable接口

  • 为了能够使用Collections.sort()方法排序自定义对象，这些对象的类必须实现Comparable接口并重写compareTo()方法。

示例代码

  • 定义一个Person类，实现Comparable接口：public class Person implements Comparable<Person> { private String name; private int age; ... public int compareTo(Person anotherPerson) { return this.age - anotherPerson.getAge(); } }
    • 这个Person类包含了姓名和年龄，并定义了根据年龄排序的比较逻辑。
  • 创建并排序Person对象的ArrayList：ArrayList<Person> people = new ArrayList<Person>(); ... Collections.sort(people);
    • 创建了Person对象的ArrayList，并通过调用Collections.sort()方法来进行排序。

总结：本节内容讲解了如何对自定义对象进行排序。首先介绍了自定义类需要实现Comparable接口并重写compareTo()方法，然后展示了如何创建自定义对象的集合并使用Collections.sort()方法进行排序。这为Java中的对象排序提供了一种标准的方法。

灵活的排序（Flexible Sorting）

提供多种比较器

  • 为了在不同的对象字段上进行排序，可以提供不同的比较器。

示例代码

  • Person类实现Comparable接口，并提供一个静态比较器：public class Person implements Comparable<Person> { public static Comparator<Person> ageComparator = new Comparator<Person>() { public int compare(Person p1, Person p2) { return p1.getAge() - p2.getAge(); } }; }
    • 这里定义了一个名为ageComparator的静态比较器，它根据人的年龄进行比较。
  • 创建Person对象的ArrayList并使用比较器排序：ArrayList<Person> people = new ArrayList<Person>(); ... Collections.sort(people, Person.ageComparator);
    • 这段代码创建了一个Person对象的ArrayList，并且通过Collections.sort()方法和ageComparator比较器来进行排序。

总结：本节内容介绍了如何为自定义对象提供灵活的排序机制。通过在类内定义静态比较器，我们可以指定排序的具体逻辑，这样就能在需要时对集合中的对象按照不同的属性进行排序。这种方法增加了排序过程的灵活性和可定制性。

挑战：开发一个模拟体育比赛的游戏（Challenge: Develop a Sports Simulation Game）

游戏简介

  • Cardball是一种体育模拟卡牌游戏，每位玩家从洗好的牌堆中抽取五张牌。
  • 玩家需要将手中的牌按照降序排列。

游戏规则

  • 每位玩家出牌后，出最高点数牌的玩家得分。
  • 如果有多位玩家出的是同样高的牌，则那一轮不计分。

游戏开发指南

  • 在DataStructures项目中开发一个名为Cardball的类，用于自动播放2人游戏（不需要用户输入）。
    • 游戏逻辑应在main()方法中运行。
    • Cardball类可以包含任何所需的辅助变量或方法。

总结：本节内容是一个编程挑战，要求开发一个名为Cardball的模拟体育卡牌游戏。游戏规则要求玩家从洗好的牌堆中拿牌，并按照牌的点数降序排列后进行比赛。在每轮中，出最高点数牌的玩家将得分，若出牌点数相同则不计分。指南部分提供了开发这个游戏的基本框架和要求。
