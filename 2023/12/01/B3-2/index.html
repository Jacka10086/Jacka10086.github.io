B3.2 二分搜索(Binary Search)

二分搜索的定义(Definition of Binary Search)

  • 二分搜索是一种在有序数组中查找特定元素的算法，它通过每次比较排除数组的一半元素。

二分搜索的步骤(Steps of Binary Search)

  • 针对排序数组进行二分搜索，目标项(target)是 8。
  • 反复找到数组的中点，并确定目标项是否在每一半中。
  • 在本例中，搜索找到了目标。

二分搜索的图解(Illustration of Binary Search)

  • 查看数组中间的项，例如 10。
  • 如果目标项小于 10，搜索数组的左半部分。
  • 接下来，查看左半部分的中间项，例如 5。
  • 如果目标项大于 5，搜索右半部分。
  • 最后，查看右半部分的中间项，例如 8。
  • 如果目标项等于 8，搜索结束，8 在数组中。

总结(Summary)

二分搜索是一种高效的搜索算法，适用于有序数组。通过不断地将搜索范围缩小一半，直到找到目标项或确定目标项不在数组中。图解清晰地展示了二分搜索在一个具体例子中的应用步骤，便于理解和学习。

扩展解释(Extended Explanation)

  • 与顺序搜索相比，二分搜索的效率更高，因为它每次比较后都减少了搜索必须考虑的元素数量。
  • 在最坏的情况下，二分搜索的时间复杂度是 O(log n)，这意味着它在处理大型数据集时表现良好。
  • 二分搜索算法的实现要求数据事先排序，这是使用它的前提条件。

二分搜索另一个排序数组(Binary Search of Another Sorted Array)

二分搜索过程(Steps of the Binary Search)

  • 这是对另一个排序数组进行二分搜索的例子，目标项(target)是 16。
  • 查看数组中间的项，比如说 10，因为 16 大于 10，所以搜索数组的右半部分。
  • 接下来，查看右半部分的中间项，比如说 18，因为 16 小于 18，搜索左半部分。
  • 继续这个过程，查看 12，因为 16 大于 12，再次搜索右半部分。
  • 最后查看 15，因为 16 大于 15，尝试搜索右半部分，但是没有更多的项可以考虑。

搜索结果(Search Results)

  • 因为没有更多的项可以考虑，所以搜索结束。
  • 搜索没有找到目标，16 不在数组中。

总结(Summary)

此例子展示了当二分搜索在有序数组中找不到目标项时的情况。通过反复将搜索范围缩小一半，最终确认目标项不在数组中。这个过程清晰地展示了二分搜索是如何在没有找到目标项时终止搜索的。

扩展解释(Extended Explanation)

  • 二分搜索算法在每个步骤中都会尽可能地排除最大数量的候选项，这使得算法非常高效。
  • 当数组中不存在目标项时，二分搜索能够快速确定这一点，因为搜索范围会逐渐缩小至没有剩余的元素可供搜索。
  • 了解二分搜索失败的情况对于掌握算法的边界条件和正确实现非常重要。

成功的二分搜索实例(Example of a Successful Binary Search)

目标值为 33(Target Value: 33)

搜索过程(Search Process)

  • 从数组的中间开始，目标值 33 大于中间值，搜索右半部分。
  • 更新中间值，继续搜索左半部分。
  • 找到目标值 33，搜索成功。

总结(Summary)

这个例子展示了一个成功的二分搜索过程，目标值 33 在数组中被成功找到。这个过程有效地展示了二分搜索如何快速定位到目标值。

失败的二分搜索实例(Example of an Unsuccessful Binary Search)

目标值为 34(Target Value: 34)

搜索过程(Search Process)

  • 从数组的中间开始，目标值 34 大于中间值，搜索右半部分。
  • 更新中间值，目标值 34 小于新的中间值，搜索左半部分。
  • 目标值不在数组中，搜索失败。

总结(Summary)

这个例子展示了当目标值不在数组中时二分搜索如何操作。搜索通过几次比较后，最终确定目标值不在数组中，展示了算法的高效性和其失败时的操作过程。

递归二分搜索排序数组(Recursive Binary Search of Sorted Array)

递归搜索过程(Recursive Search Process)

  • 要搜索数组中从 a[0] 到 a[n-1] 的元素，可以递归地搜索 a[0] 到 a[mid-1] 或 a[mid+1] 到 a[n-1]。
  • 这样，每次递归的搜索范围都会缩小，每一步都是问题更小的版本。

递归二分搜索伪代码(Pseudocode for Recursive Binary Search)

  • 参数 first 和 last 用于指定数组要搜索的子范围的起始和结束索引。
  • 如果 first 大于 last，表示范围无效，返回 false。
  • 如果找到目标元素，返回 true。
  • 否则，根据 entry 与中间元素的比较结果，递归搜索数组的左半部分或右半部分。

总结(Summary)

递归二分搜索是二分搜索的一个变体，它通过递归地将问题划分为更小的问题来解决。这个过程不断缩小搜索范围，直到找到目标元素或确定元素不在数组中。

扩展解释(Extended Explanation)

  • 递归方法是二分搜索的一种优雅实现方式，它可以简化代码结构，避免使用循环和多余的状态变量。
  • 由于递归可能会导致较大的调用栈，对于非常大的数组，需要注意栈溢出的可能性。
  • 理解递归搜索的基础对于理解更复杂的数据结构和算法，如搜索树和图搜索算法，是非常重要的。

二分搜索的数组效率(Efficiency of a Binary Search of an Array)

效率分析(Efficiency Analysis)

  • 搜索会在每一步排除大约一半的数组元素。
  • 最佳情况(Best Case): 目标元素在首次检查的元素中，搜索效率为 O(1)。
  • 最差情况(Worst Case): 搜索继续进行直到只剩一个元素，需要 k 次比较使得 2^k = n，效率为 O(log n)。
  • 平均情况(Average Case): 搜索将进行一半的递归调用，效率也为 O(log n)。

总结(Summary)

二分搜索通过每次比较排除数组中的一半元素，因此它的效率非常高。在最佳情况下，它可以立即找到元素；在最差情况和平均情况下，其效率都是 O(log n)，这意味着它适合大型数据集的搜索。

扩展解释(Extended Explanation)

  • 二分搜索的高效率使其成为处理大型有序数组的理想选择。
  • O(log n) 的时间复杂度意味着即使是非常大的数组，所需的比较次数也相对较少。
  • 理解这三种情况对于评估二分搜索在不同数据集上的性能至关重要。

对排序链进行二分搜索(Binary Search of a Sorted Chain?)

二分搜索链的问题(Problems with Binary Searching a Chain)

  • 如何找到链的中点？（为了找到链的中点，必须遍历整个链到那个点）
  • 然后你必须遍历半个链来找到那一半的中点——工作量太大！

改进的方法(Improved Approach)

  • 更好的方法是遍历一次列表，构建一个数组，然后在这个数组上使用二分搜索。

总结(Summary)

对于排序链结构，使用传统的二分搜索方法效率低下，因为找到中间元素需要顺序遍历。一个更有效的方法是先将链转换为数组，然后在数组上应用二分搜索。

扩展解释(Extended Explanation)

  • 链结构不同于数组，它不允许随机访问，这意味着不能直接访问中间元素。
  • 将链结构转换为数组可以利用数组的随机访问特性，使二分搜索变得可行和高效。
  • 这种方法的缺点是需要额外的时间和空间来创建数组，这可能在数据集非常大时成为问题。

顺序搜索与二分搜索(Sequential Search vs. Binary Search)

何时使用顺序搜索(When to Use Sequential Search)

  • 当需要搜索链表时，应使用顺序搜索。

何时选择搜索技术(When to Choose a Search Technique)

  • 如果要搜索对象数组，需要选择合适的搜索技术。

搜索效率对比(Search Efficiency Comparison)

  • 搜索类型(Search Type) 最佳情况(Best Case) 平均情况(Average Case) 最坏情况(Worst Case)
    顺序搜索(未排序数据)(Sequential search (unsorted data)) O(1) O(n) O(n)
    顺序搜索(排序数据)(Sequential search (sorted data)) O(1) O(n) O(n)
    二分搜索(排序数组)(Binary search (sorted array)) O(1) O(log n) O(log n)

数组大小对搜索影响(Array Size Impact on Search)

  • 如果数组较小，顺序搜索可能更快（计算量少）。
  • 如果数组大且已经排序，通常二分搜索更快。

总结(Summary)

如果要搜索的数据是未排序的或是链表形式的，顺序搜索可能是更好的选择。对于大型且已排序的数组，二分搜索在效率上通常更胜一筹。

扩展解释(Extended Explanation)

  • 在最佳情况下，两种搜索方法都可能立即找到目标元素，但在平均和最坏情况下，二分搜索的效率显著高于顺序搜索。
  • 在选择搜索策略时，考虑数据的结构和大小是非常重要的，它将直接影响搜索效率。

挑战(Challenge)

在一个名为Searching的新项目中，创建一个类SearchTimeTest来测量排序数组的搜索算法时间。

任务要求(Task Requirements)

  • 生成随机的升序整数数组，大小为 1000, 2000, 4000, 8000, 16000, 和 32000 元素。对于每个数组，整数范围应该是 1 到数组大小 * 10（例如，1K 整数在 1 到 10K 范围内，2K 整数在 1 到 20K 范围内，等等）。
  • 对于每个数组，生成一个包含 1000 个搜索值的数组，这些搜索值的范围与数组元素的范围相同。
  • 测量迭代顺序搜索和递归二分搜索完成所有 1000 个搜索值的搜索所需的时间。记住要为排序数组优化顺序搜索。
  • 按照演示视频所示报告结果。

总结(Summary)

此挑战要求开发者创建一个性能测试类，用于比较在不同大小的排序数组上执行顺序搜索和二分搜索的性能，并将结果按照某种格式进行报告。

扩展解释(Extended Explanation)

  • 此性能测试可以帮助理解不同搜索算法在处理不同大小数据集时的效率差异。
  • 优化搜索算法是确保算法性能的关键步骤，特别是在处理大型数据集时。
  • 报告结果的方式应该能够直观地展示不同算法的性能，便于比较和分析。
