B3.1 顺序搜索

  • Ulster University提供的COM410课程着重于编程实践，B3.1单元专注于顺序搜索算法。

人们总在寻找某些东西…

搜索的重要性

  • 搜索是计算机执行的最常见任务之一。

基本搜索策略

  • 有两种基本的搜索策略：
    1. 顺序搜索（Sequential Search）
      • 顺序搜索是一种简单的搜索方法，通过遍历数据集中的每个元素来查找目标元素。
    2. 二分搜索（Binary Search）
      • 二分搜索则是在已排序的数据集中使用，通过重复分割数据集的方式来减少搜索范围，从而提高搜索效率。

总结：这两张幻灯片提供了关于搜索方法的介绍。在编程实践中，了解并实现有效的搜索算法是基本技能之一。顺序搜索适合于未排序的数据，而二分搜索适用于已排序的数据集，可以大幅度提高搜索的效率。

顺序搜索数组的效率（Efficiency of a Sequential Search of an Array）

不同情况下的时间复杂度

  • 最佳情况（Best case）: 所需元素是数组中的第一个元素，此时搜索的时间复杂度为O(1)。
  • 最坏情况（Worst case）: 必须搜索整个数组（不管是否成功找到），对于有n个元素的数组，需要进行n次比较，时间复杂度为O(n)。
  • 平均情况（Average case）: 通常搜索会查看数组一半的元素，因此时间复杂度为O(n/2)，但这在大O表示法中仍然简化为O(n)。

总结：这张幻灯片讲解了顺序搜索数组时的时间效率。在最佳情况下，搜索可以很快完成；但在最坏情况下，可能需要查看所有元素。平均来看，搜索操作的效率是线性的，这意味着所需时间与数组大小成正比。

未排序链表的顺序搜索（Sequential Search of an Unsorted Chain）

在链表中搜索

  • 在抽象数据类型（ADT）列表的链表实现中，contains()方法会在链表的节点中搜索目标元素。

顺序搜索的过程

  • 顺序搜索将从列表的第一个节点开始，连续检查每个节点的数据，直到找到数据等同于目标的节点，或者所有节点都被检查过。

  • 搜索将沿着链表移动，访问每个节点，直到找到所需的数据或达到链表的末尾。

总结：本幻灯片解释了在未排序的链表结构中如何进行顺序搜索。这种搜索方法是线性的，可能需要遍历整个链表才能找到目标元素，或者确定目标元素不在列表中。

链表顺序搜索的效率（Efficiency of a Sequential Search of a Chain）

不同情况下的时间复杂度

  • 最佳情况（Best case）: 所需元素位于链表的第一个节点，此时搜索的时间复杂度为O(1)。
  • 最坏情况（Worst case）: 必须搜索整个链表（不论是否成功找到），对于有n个节点的链表，需要进行n次比较，时间复杂度为O(n)。
  • 平均情况（Average case）: 通常搜索会查看链表一半的节点，因此时间复杂度为O(n/2)，但这在大O表示法中简化为O(n)。

总结：这张幻灯片讲解了在链表中进行顺序搜索的效率。顺序搜索可能需要遍历整个链表才能找到目标元素或确定目标元素不在链表中，因此它的效率是线性的，与链表长度成正比。

顺序搜索排序集合(Sequentially Searching a Sorted Collection)

  • 遍历集合直到找到包含或大于所需项目的节点，或者在没有成功的情况下检查集合中的所有项目。
  • 如果找到了较大的项目，则所搜寻的元素不存在。

算法解释(Algorithm Explanation)

  • inArray(array, entry) 算法用于判断数组是否包含某个元素，如果包含则返回真，否则返回假。
  • 设置索引为0（set index to 0）。
  • 当索引小于数组长度并且数组在该索引的值小于或等于条目时，继续循环（while index < array length and array[index] <= entry）。
  • 如果在该索引的数组值等于条目，则返回真（if array[index] equals entry return true）。
  • 如果没有找到匹配项，则索引增加1，继续循环（add 1 to index）。
  • 如果循环结束还没有返回真，则返回假（return false）。

总结(Summary)

顺序搜索是一种简单的搜索算法，它通过逐个检查数组的每个元素来查找特定的值。在排序的集合中，一旦找到一个大于所寻找值的元素，搜索可以停止，因为所有后续的元素都会更大，这是排序集合的特性。这里提供的inArray算法伪代码正是基于这样的原理进行搜索。

扩展解释(Extended Explanation)

  • 顺序搜索尽管简单，但效率不高，特别是在处理大数据集时。因为它可能需要遍历整个集合来找到一个元素，或确定该元素不存在。
  • 对于排序的集合，顺序搜索比随机集合中的搜索要快，因为它利用了集合的顺序属性。
  • 如果需要更高效的搜索算法，可以考虑使用二分搜索，它的搜索速度要快得多，特别是对于大型的排序集合。

场景概述(Scenario Overview)

  • 手中的牌(Card in hand)
    • 返回到你的DataStructures项目，并添加一个新类Hand，它有一个实例变量，用于存储按升序排列的Card对象数组。
    • 该类应包含一个inHand()方法，该方法接受一张牌作为参数，并使用顺序搜索来返回true，如果手中有相同等级的牌，否则返回false。

main() 方法(Main Method)

  • main()方法应该创建一个由5张随机牌组成的手牌，这些牌按升序排列。
  • 然后，应用程序应该随机生成另外10张牌，对每张牌检查手牌中是否包含相同等级值的牌。

输出(Output)

  • 输出手牌，然后依次输出每张随机牌，同时附带信息说明手牌中是否包含等级相同的牌。

总结(Summary)

场景描述了一个关于扑克牌的数据结构的编程任务，需要添加一个能够存储和搜索特定牌的Hand类。这个类的inHand()方法使用顺序搜索来判断手牌中是否存在特定等级的牌。main()方法的任务是生成手牌并检查新牌是否与手牌中的牌等级匹配，并输出结果。

扩展解释(Extended Explanation)

  • 这个场景是数据结构和算法教学中的一个实际例子，它展示了如何将数据结构（如数组）与算法（如顺序搜索）结合起来解决实际问题。
  • 通过实际的扑克牌例子，可以帮助学生更好地理解顺序搜索算法如何在一个有序集合中工作，以及它在实际应用中的效率如何。
  • 类的设计和方法的实现需要考虑到效率和扩展性，因为这将影响程序处理大量牌时的性能。
