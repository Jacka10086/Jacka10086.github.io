COM410 编程实践——队列（COM410 Programming in Practice - Queues）

队列在数据组织中的例子（Recall Examples of Data Organisation in Queues）

  • 队列（Queue）- 先进先出，是日常生活中常见的数据组织技术。
    • 例如，在商店排队、计算机科学中的数据结构、生活中的待办事项列表等。

队列的定义和应用（Definition and Application of Queues）

  • 队列是等候线的另一个称呼。
  • 在操作系统内部及模拟现实世界事件中使用。
  • 当过程或事件需要等待时就会用到队列。
  • 队列的条目是按照先进先出（按时间顺序）组织的。

队列的灵活性（Flexibility in Queues）

  • 有时候需要更多的灵活性：
    • 双端队列（deque）允许在最老和最新的条目上进行操作。
    • 当一个对象的重要性不仅仅依赖于其到达时间时，可以分配优先级（称为优先队列）。

总结：这一系列的幻灯片提供了队列在数据组织中的应用示例，定义了队列，讨论了它们在操作系统和模拟现实世界事件中的使用，并介绍了队列的灵活形式，如双端队列和优先队列。

队列操作（Queue Operations）

  • 队列抽象数据类型（ADT）像栈一样，按照它们被添加的顺序组织条目。
  • 队列的行为被描述为 FIFO（先进先出）。

具体操作（Specific Operations）

  • 所有新增条目都在队列的后端添加，队列后端的条目是最新添加到队列的。
  • 最早添加的条目位于队列的前端。
  • 添加条目到队列的操作通常称为入队（enqueue）。
  • 从队列中移除条目的操作通常称为出队（dequeue）。

图解说明（Diagram Explanation）

  • 示意图展示了队列中条目的排列，以及如何进行入队和出队操作。
    • 例如，如果队列中依次添加了 A、B、C 和 D，那么 A 将是第一个出队的，D 将是最后一个出队的。

总结：这段Markdown内容详细说明了队列的基本操作和其FIFO的特性。入队和出队操作是队列最基本的两个操作，它们决定了队列中元素的流动方向和处理顺序。

队列操作补充（Additional Queue Operations）

  • 与栈一样，队列也限制了对其条目的访问（只能查看或检索最早的条目）。
  • 除了入队（enqueue）和出队（dequeue）操作外，不移除条目而检索队列前端的操作称为 getFront。

队列的查看限制（Viewing Restrictions in Queues）

  • 通常情况下，你不能在队列中搜索特定的条目。
  • 查看不在队列前端的条目的唯一方法是重复移除队列中的条目，直到所需的项到达前端。

对队列操作的影响（Implications for Queue Operations）

  • 这意味着队列不适合需要频繁搜索的应用场景。
  • getFront 操作允许检查队列的第一个元素，但不会改变队列的状态。

总结：本Markdown内容介绍了队列的额外操作及其访问限制。队列的结构和操作保证了数据的有序处理，但也限制了数据的访问灵活性。

队列ADT的接口（Interface for the Queue ADT）

队列ADT（Abstract Data Type，抽象数据类型）的接口定义了队列这种数据结构的基本操作。以下是各个方法的详细解释：

  • public void enqueue(T newEntry);
    • 入队操作：将一个新元素添加到队列的尾部。
  • public T dequeue();
    • 出队操作：从队列的头部移除一个元素，并将其返回。
  • public T getFront();
    • 获取队首元素：返回队列头部的元素，但不从队列中移除它。
  • public boolean empty();
    • 检查队列是否为空：如果队列中没有任何元素，则返回true。
  • public void clear();
    • 清空队列：移除队列中的所有元素。

入队操作（Enqueue Operation）

入队是队列的基本操作之一，它允许我们将一个新元素添加到队列的末尾。

出队操作（Dequeue Operation）

出队是另一个基本操作，它移除队列头部的元素，这符合队列先进先出（FIFO）的原则。

获取队首元素（Get Front Element）

获取队首元素允许我们查看队列中的第一个元素，这常用于检查队列的当前状态而不改变其内容。

检查队列是否为空（Check if Queue is Empty）

这个操作用于判断队列中是否还有元素，它对于防止在空队列上执行出队等操作非常重要。

清空队列（Clear Queue）

清空队列的操作可以移除所有元素，这在重置队列状态或者在不再需要队列内容时非常有用。

总结：以上内容详细说明了队列ADT的接口，包括入队、出队、查看队首元素、检查队列是否为空和清空队列等操作。每个操作都有其特定的用途和实现的必要性，它们共同定义了队列这种数据结构的行为和特点。

Java队列ADT接口（Java Interface for the Queue ADT）

Java队列ADT接口规定了队列数据结构的基本操作方法。这些方法包括：

  • public void add(T newEntry);
    • 添加操作：在队列尾部添加一个新的元素。
  • public T remove();
    • 移除操作：从队列前端移除一个元素，并将其返回。
  • public T peek();
    • 检视操作：返回队列前端的元素，但不从队列中移除它。
  • public boolean isEmpty();
    • 判断队列是否为空：检查队列中是否有元素，若无则返回true。
  • public void clear();
    • 清空队列：移除队列中的所有元素。

添加操作（Add Operation）

添加操作允许用户在队列的后端插入一个新元素，这是队列的基本特性之一。

移除操作（Remove Operation）

移除操作从队列的前端删除一个元素，并将其返回，这体现了队列的先入先出（FIFO）原则。

检视操作（Peek Operation）

检视操作允许用户查看队列前端的元素，这个操作通常用于在不修改队列的情况下，查看即将被处理的元素。

判断队列是否为空（IsEmpty Operation）

这个操作用于检查队列中是否还剩余元素，它对于避免在空队列上执行某些操作是非常重要的。

清空队列（Clear Operation）

清空操作用于删除队列中的所有元素，这在用户需要重置队列或清除所有元素时非常有用。

总结：这段内容详细地解释了Java队列ADT接口中定义的各种操作方法，每个方法都有其独特的功能和用途，它们共同构成了队列这种数据结构的基本操作集合。

Java队列类（Java Queue Class）

Java提供了内置的队列类，可用于实现队列数据结构。以下是使用Java队列类的详细指南：

  • Java的队列实现

    • Java使用java.util.Queue接口提供队列的抽象实现。
    • LinkedList是Queue接口的一个具体实现类，它在java.util包中。

  • 必要的导入

    • 使用队列之前，必须导入Queue接口和LinkedList类。
    • import java.util.LinkedList;
    • import java.util.Queue;

  • 创建队列实例

    • 实例化一个队列时，通常使用LinkedList类来创建一个Queue接口的实例。
    • 示例代码：Queue<String> myQueue = new LinkedList<String>();

  • 使用注意

    • 在类的顶部导入Queue对象是必须的，以便在类中使用队列。
    • 由于Queue是一个接口，因此我们需要一个具体的类来实现它，LinkedList是这样一个类。

导入语句（Import Statements）

  • 导入语句是Java程序中的一部分，它告诉编译器从Java API中查找并加载指定的类或接口。

队列的实例化（Instantiation of Queue）

  • 在Java中，我们通过实例化LinkedList类来创建Queue的实例。
  • 这是因为LinkedList类实现了Queue接口，并提供了队列操作的方法。

类的结构（Structure of the Class）

Java Queue 类（Java Queue Class）

  • 使用 isEmpty() 方法来检测一个队列是否为空，以及 size() 方法来报告队列中包含的元素数量。
    • isEmpty() 方法是用来判断队列是否没有任何元素，如果返回 true 则队列为空。
    • size() 方法返回队列中的元素数量，用于快速获取队列的大小。
  • 新创建的队列初始状态是空的。
    • 当我们创建一个队列对象后，它不包含任何元素，因此 isEmpty() 方法会返回 true。

代码解释（Code Explanation）

  • 展示初始队列的代码示例：
    • 首先，使用 System.out.println() 方法打印一条消息表明接下来将显示队列的初始状态。
    • 然后，使用 System.out.println(myQueue) 来打印队列对象，此时队列为空。
    • 接着是一个 if 判断语句，检查 myQueue.isEmpty() 的返回值。
      • 如果队列为空，就打印 “Queue is empty”。
      • 否则，打印 “Queue contains %d elements”，并且使用 myQueue.size() 来获取队列中的元素数量。

输出结果（Output Result）

  • 输出展示了初始队列为空的信息：
    • [] 表示队列中没有元素。
    • 紧接着的消息 “Queue is empty” 确认了队列目前是空的状态。

总结：这部分讲述了Java中队列类的基本用法，包括如何检查队列是否为空以及如何获取队列中的元素数量。代码示例展示了如何在Java中声明一个队列，以及如何使用 isEmpty() 和 size() 方法。最后，输出结果证实了新创建的队列默认是空的。

Java 队列类（Java Queue Class）

  • 使用 add() 方法（注意 - 不是 enqueue）来向队列中添加元素。
    • add() 方法用于在队列的尾部添加一个新元素。
  • 新元素被添加到队列的后端。
    • 这意味着队列遵循先进先出（FIFO）的原则，最新添加的元素位于队列的末尾。

代码解释（Code Explanation）

  • 代码示例中展示了向队列添加元素的过程：
    • 通过 System.out.println() 方法首先打印出正在添加的元素名称。
    • 使用 myQueue.add("Adrian") 向队列中添加名为 Adrian 的元素。
    • 紧接着，使用同样的方法添加名为 Belle 和 Charles 的元素。
    • 打印队列对象 myQueue 会显示队列中的所有元素。
    • 使用 if 语句来判断队列是否为空，如果不为空，则打印出队列包含的元素数量。

输出结果（Output Result）

  • 输出结果显示了添加元素后的队列状态：
    • System.out.println() 输出了 “Adding Adrian, Belle and Charles”，表明了这些元素被添加进队列。
    • [Adrian, Belle, Charles] 显示了队列中元素的当前顺序。
    • “Queue contains 3 elements” 确认了队列中现在有三个元素。

总结：这部分解释了如何使用 add() 方法在Java队列类中添加新元素。示例代码显示了添加元素的具体步骤，并且通过输出结果验证了元素已成功添加到队列中。

Java 队列类（Java Queue Class）

  • 使用 remove() 方法（注意 - 不是 dequeue）来从队列中移除元素。
    • remove() 方法用于从队列的前端移除元素，遵循先进先出的原则。
  • 元素从队列的前端被移除，队列中停留时间最长的元素将是第一个被移除的。
    • 这符合队列的基本操作原理，即最先添加进队列的元素将最先被移除。

代码解释（Code Explanation）

  • 移除队列前端元素的代码示例：
    • 使用 System.out.println() 方法打印消息提示将要从队列前端移除元素。
    • 执行 System.out.println(myQueue.remove())，这将移除队列中的第一个元素，并打印被移除的元素。
    • 打印队列对象 myQueue 将展示移除元素后的队列内容。
    • 如果队列现在为空，则使用 if 语句检查并打印 “Queue is empty”。
    • 如果队列不为空，则打印 “Queue contains %d elements”，显示剩余元素的数量。

输出结果（Output Result）

  • 输出结果显示了从队列前端移除元素后的状态：
    • “Removing from the front of the queue” 表明了操作的开始。
    • 移除的元素 ‘Adrian’ 被打印出来，表明它是被移除的元素。
    • [Belle, Charles] 显示了移除元素后队列中剩余的元素。
    • “Queue contains 2 elements” 说明在移除一个元素之后，队列中还剩两个元素。

总结：本节内容介绍了如何使用 remove() 方法来从Java的队列类中移除元素。通过示例代码和输出结果的解释，我们可以理解队列操作的先进先出原则，并看到实际代码执行后队列状态的变化。

  • peek() 方法用于返回队列中的元素
  • 此方法返回队列前端的元素，但不会移除该元素
  • 使用 peek() 操作不会改变队列内容

示例代码

  • Java的Queue类通过peek()方法允许我们查看队列的首个元素，但不从队列中移除它。示例代码展示了如何使用peek()方法和如何检测队列是否为空，同时也展示了如何计算队列中的元素数量。输出结果确认了peek()方法的功能，并展示了队列中元素的状态。

Java 类库说明（Note on the Java Class Libraries）

Queue 接口

  • Java 类库中的 Queue 被定义为一个接口
  • 由于接口不能直接实例化，我们需要导入一个实现了 Queue 的类，比如 LinkedList

Queue 的方法

  • 传统的队列方法如 add()、remove() 和 peek() 是添加、访问和移除数据的唯一手段
  • Java 的 Queue 接口从其父类继承了很多方法，这些方法包括了非传统的操作，例如搜索、从中间检索等

继承关系

  • Queue 接口继承自 Collection 接口，而 Collection 接口又继承自 Iterable 接口
  • Queue 接口被 LinkedList 类、PriorityQueue 类和 ArrayDeque 类实现

本节不使用的方法

  • 在本节内容中，不会使用 Queue 接口继承来的那些方法

  • 图表展示了 Queue 接口与其他接口和类的关系
  • 显示了 Queue 是如何从 Collection 接口继承，并且被不同的类实现

  • 这部分内容说明了 Java 类库中 Queue 接口的定义和使用。它强调了 Queue 接口不能直接实例化，而是通过类如 LinkedList 来实现其功能。本节内容还提到了 Queue 接口继承了许多方法，但本节不会讨论这些继承的方法。图表清晰地展示了 Queue 接口在 Java 类库中的位置以及它与其他接口和类的关系。

场景描述（Scenario）

新增类实现

  • 在 DataStructures 项目中添加一个名为 Queues 的新类
  • 该类用于实现查找字符串中的第一个非重复字符的应用程序

功能要求

  • 用户应当通过键盘输入一个字符串
  • 应当逐个字符检索，从字符串的第一个字符开始，然后将字符入队列
  • 在字符入队列的同时，应当维护一个计数器记录每个字符出现的次数
  • 逐个从队列中移除字符，当发现有一个字符的计数器为1时停止，并报告这是第一个非重复字符
  • 如果没有找到非重复字符，应当向用户报告这一情况

中文总结

  • 这个场景描述了在 DataStructures 项目中添加一个新类 Queues 的要求，用于识别用户输入字符串中的第一个非重复字符。它详细说明了实现的步骤，包括用户输入、字符检索、入队列、计数以及如何处理找到或未找到非重复字符的情况。
