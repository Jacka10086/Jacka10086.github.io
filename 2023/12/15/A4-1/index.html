COM410 编程实践

A4.1 定义和使用指针

在COM410课程中，学生将学习指针的定义和使用方法，这是C或C++编程语言中的一个高级特性。

数组实现的问题 (Problems with Array Implementation)

  • 我们之前使用固定大小的数组来实现Bag ADT（抽象数据类型）。
  • 这样的实现可能存在的问题包括：
    • 数组有固定的大小。
    • 数组可能会填满。
    • 可能会有未使用的空间。
    • 调整大小是可能的，但需要时间开销。

总结

这两张幻灯片为Ulster University的COM410编程课程提供了教学大纲和实现细节。第一张幻灯片介绍了定义和使用编程中的指针，而第二张幻灯片讨论了在抽象数据类型实现中使用数组所带来的一些潜在问题，以及与这些问题相关的时间复杂性考量。

链接数据组织 (Linked Data Organisation)

  • 本节介绍了一种实施方法，该方法仅在需要时（如新增条目时）使用内存，并在条目被移除后将不需要的内存返回给系统。
  • 通过使用链接数据组织来实现Bag ADT（抽象数据类型），我们可以避免在添加或移除包条目时移动数据。
  • 链表（链式链）是一种线性数据结构，常用于实现其他数据结构。

总结

链式数据组织是计算机科学中常用的一种数据管理方式，特别是在动态数据结构如链表中。链表通过节点的链接方式有效管理数据元素，从而优化内存使用，并简化了数据元素的添加和移除操作。这种方法避免了在数组和其他连续内存数据结构中常见的数据移动问题，提高了操作效率。

链接数据组织 (Linked Data Organisation)

  • 链表（链式链）由一系列节点组成。
  • 每个节点通常存储一个指向数据（包中的一个条目）的引用（指针）和一个指向另一个节点（链中下一个节点的地址）的引用。
  • 在链表中，最后一个节点指向null，表示链的结束。

总结

链表是一种基本的数据结构，通过一系列相互链接的节点来存储和管理数据。每个节点包含数据和指向链中下一个节点的引用，从而形成了一个链式结构。链表的最后一个节点指向null，这表明了链的终点。这种组织方式在计算机科学中非常普遍，因为它允许动态的数据管理，容易地添加和移除元素，而无需像在数组中那样重新分配整个数据结构。

链表与数组的对比 (Linked List vs. Array)

  • 链表在序列和顺序的处理上与数组相似。

  • 与数组不同，链表：

    • 不受固定大小元素数量的限制。
    • 不是在内存中连续存储的。
    • 节点可以在不重新分配内存的情况下插入和移除。

  • 数组在访问元素方面更快。

  • 数组在插入或移除元素方面较慢。

  • 链表可以在运行时动态地增长和缩小。

总结

链表和数组都用于存储数据元素的序列，但它们在管理数据方面有本质的不同。链表提供了更大的灵活性，允许在不需要连续内存空间的情况下动态地添加和删除节点。相比之下，数组提供了快速的元素访问，但在插入和删除操作方面可能效率较低，尤其是当涉及到大量数据时。链表的动态大小特性使其在需要频繁修改数据时更加有用。

节点类 (The Node Class)

  • 为了维护一个链接集合，我们首先需要定义一个Node节点类。
  • 节点类包含两个数据字段、构造函数以及访问器（accessor）和变异器（mutator）方法。
  • 数据字段（例如 data）包含对包中某个对象的引用。
  • 指针字段（例如 next）包含对序列中下一个节点的引用。
  • 构造函数创建一个新节点，设置提供的data字段，并初始化next字段为null。
  • 访问器方法返回data和next字段的值。
  • 变异器方法设置或更新data和next字段的值。

总结

节点类是构建链表的基本组成部分，它通过存储数据和指向列表中下一个节点的指针来工作。通过构造器、访问器和变异器方法，我们可以创建、检索和修改节点的数据，这是实现动态数据结构如链表的关键。

节点类 (The Node Class)

  • Node类是链表数据结构中的一个基本组件。
  • 它包含两个私有属性：data和next。
    • data属性用于存储节点的数据。
    • next属性存储了对链表中下一个节点的引用。
  • Node类包含一个构造函数和四个公共方法。
    • 构造函数Node(String dataValue)用于创建新节点，初始化data属性，并将next设置为null。
    • getData()方法用于获取节点的数据。
    • setData(String dataValue)方法用于设置节点的数据。
    • getNext()方法用于获取指向下一个节点的引用。
    • setNext(Node nextNode)方法用于设置指向下一个节点的引用。

类结构解释

  • 实例变量包括一个数据载荷和一个指针，分别用于存储数据和指向链表中的下一个节点。
  • 构造器用于初始化新创建的节点实例。
  • 公共方法提供了对节点数据和指针的访问和修改功能。

总结

Node类是实现链表结构的基础。它通过data属性存储信息，并通过next属性链接到下一个节点。这种结构允许链表在运行时动态地增加或减少节点，使数据结构具有高度的灵活性和扩展性。

场景 (Scenario)

  • 在你的Anytown项目中，创建一个新文件Node.java，并实现Node类定义，其中载荷类型为Building对象。
  • 通过创建一个新文件NodeTest.java来测试Node的定义，该文件包含一个类NodeTest，在其中main()方法定义了三个节点，名为node1、node2和node3，每个节点用新的Building对象作为载荷，每个建筑都有一个独特的地址字段。
  • 设置节点的next字段，使node1指向node2，node2指向node3。
  • 现在，不直接引用node2或node3，编写代码打印所有3个节点的值。

总结

此场景要求开发者在一个Java项目中实现一个链表结构，节点类中包含Building类型的数据载荷。开发者需要在NodeTest类的main方法中实例化并连接三个节点，并编写代码来遍历并打印链表中每个节点的数据。这个练习旨在帮助开发者理解如何在Java中操作链表数据结构。
