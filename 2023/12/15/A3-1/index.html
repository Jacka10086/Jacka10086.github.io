抽象数据类型（Abstract Data Types）

数据类型的定义

  • 数据类型是一组值和这些值上的操作的集合，这些都是在特定编程语言内定义的。
  • 中文总结：数据类型是编程语言中定义的值和操作的集合。

基本类型（Primitive types）

  • 基本类型的值映射到机器表示，操作映射到机器指令。
  • 中文总结：基本类型是最简单的数据类型，直接对应到计算机的存储结构。

我们的目标

  • 我们希望编写可以处理其他数据类型的程序。
  • 中文总结：我们的编程目标是处理不同种类的数据类型。

抽象数据类型（Abstract Data Type）

  • 抽象数据类型是一种数据类型，其实现细节对客户端是隐藏的。
  • 中文总结：抽象数据类型隐藏了实现细节，只暴露接口给用户。

数据类型的分类（图表解释）

  • 数据类型可以分为字符型（char）、字符串型（String）、整型（int）、浮点型（double）和布尔型（boolean）。
  • 每种数据类型有它的值集合、值的例子以及操作的例子。
    • 字符型的值是字符，例如 ‘A’ 和 ‘@’，操作是比较。
    • 字符串型的值是字符序列，例如 “Hello World”，操作是连接。
    • 整型的值是整数，例如 17 和 12345，操作包括加、减、乘、除。
    • 浮点型的值是浮点数，例如 3.1415 和 6.022e23，操作包括加、减、乘、除。
    • 布尔型的值是真值，例如 true 和 false，操作包括与、或、非。
  • 中文总结：图表详细列出了不同基本数据类型的值集合、示例值以及可能的操作，帮助我们理解和使用它们。

抽象数据类型（Abstract Data Types）

抽象数据类型的具体定义

  • 抽象数据类型（ADT）是一组值及其操作的规范，这种规范在概念上定义，且独立于任何编程语言。
  • 数据结构是在某个编程语言内实现的抽象数据类型。
  • 中文总结：抽象数据类型定义了数据的模型及其操作，不依赖于具体的编程语言。数据结构是这种类型在特定语言中的具体实现。

抽象数据类型（Abstract Data Types）

Java中的字符串数据类型

  • Java的String数据类型是一个抽象数据类型（ADT），允许程序操纵字符串值。
  • 字符串是Unicode字符序列。
  • String的API提供了ADT的接口。
  • 使用ADT时，我们不需要知道底层实现的细节。
  • 中文总结：Java中的String类型是抽象数据类型的一个例子，它提供了操作字符串的一组方法，而我们无需了解其背后的实现细节。

已设计和使用的其他ADT

  • 我们已经设计并使用了其他的ADT，比如Building、House、Shop、Pet、Cat、Dog等。
  • 中文总结：在实际编程中，我们创建和使用了许多不同的抽象数据类型，以模拟现实世界中的各种对象和概念。

字符串的API方法（图表解释）

  • 公共类String包括创建字符串、获取字符串长度、字符操作、子串判断、前缀/后缀判断、字符串查找、字符串连接、字符串比较、字符串替换、分割字符串以及字符串比较等方法。
  • 中文总结：图表列出了Java字符串API提供的方法及其用途，这些方法是操作和管理字符串的工具。

抽象数据类型（Abstract Data Types）

使用数据类型时你需要知道：

  • 名称（在Java中首字母大写）
  • 如何构造新对象（实例化）
  • 如何对给定对象应用操作（调用方法）

如何构造新对象：

  • 使用关键词 new 来调用构造器
  • 使用数据类型名称（类名）来指定对象的类型

如何应用操作：

  • 使用对象名称来指定哪个对象
  • 使用点操作符来指示操作应该被应用到的对象

示例代码解释：

  • Building b1; 声明了一个类型为Building的变量b1
  • b1 = new Building("101 Main Street", "Homer"); 使用构造器创建了一个Building对象，并初始化地址和所有者
  • b1.setOwner("Marge"); 调用了Building对象的setOwner方法来设置新的所有者
  • System.out.println(b1); 打印出对象b1的信息

总结：

在使用Java等面向对象的编程语言时，了解如何创建和操作对象是编程的基础。这包括了解如何使用构造函数来实例化对象，以及如何通过调用对象的方法来执行操作。示例代码展示了这些概念的基本用法，包括声明对象、使用构造函数创建对象以及调用对象的方法来更改其状态。

使用ADT就像使用自动售货机（Using an ADT is like using a vending machine）

自动售货机的使用：

  • 只能执行机器界面所展示的任务
  • 必须理解这些任务
  • 不能访问机器的内部
  • 即使你不知道内部发生了什么，也可以使用机器
  • 即使内部结构改变了，依然可以使用

总结：

使用抽象数据类型（ADT）的过程可以类比为使用自动售货机：用户通过界面与其交互，而无需了解或访问内部的复杂机制。这强调了抽象的重要性，即提供了一种方式，让用户能够利用系统的功能，同时又无需了解其内部是如何运作的。类似地，在编程中，我们可以使用ADT提供的接口来实现功能，而无需关心其背后的实现细节。

数据组织的例子（Examples of Data Organization）

使用ADT（抽象数据类型）能让我们概括性地理解如何将对象作为集合进行分组。这里给出了几个数据组织的例子：

  • 人群中的个体被组织成队列
  • 地图上的城市和道路构成网络
  • 购物袋中的商品按类别放置
  • 书架上的书籍按照一定的顺序排列
  • 文件夹系统中的文件和文件夹按层次结构组织
  • 待办事项列表将任务按照优先级或类别排序

总结：

以上图片展示了生活中各种数据组织的实际例子。无论是现实生活中的队列和分类，还是数字世界中的文件系统和待办事项，都可以看作是抽象数据类型的实例。ADT作为一种编程中的概念，它帮助我们在软件开发中以类似的方式组织和管理数据结构。

场景（Scenario）

Anytown项目修改要求：

  • 重新审视你的Anytown项目，替换AnytownTest类中的main()方法，使其具备以下功能：
    • 定义一个数组来存储11个Building对象
    • 将数据文件buildings.txt添加到项目的src文件夹中
    • 编写代码从数据文件中读取11栋建筑的描述，每行数据一个建筑，然后将它们添加到数组中。数据文件的每个字段由|字符分隔。每行数据的第一个字段是建筑类型，其中H代表一个House对象，S代表一个Shop对象，B代表一个既不是房屋也不是商店的一般Building。所有建筑都有地址和所有者字段，而House对象还有卧室数量和一个表示有无车库的真/假字段，Shop对象还有员工数量和平均年营业额（以千欧为单位）的字段。
    • 当所有建筑都创建好后，打印它们的描述到控制台。

总结：

这个场景要求学生修改一个现有的Java项目，具体是通过操作数组和文件I/O来增加一个新功能。这个功能涉及到处理不同类型的数据，并将其存储在合适的数据结构中，最后输出结果。这是对面向对象编程和数据处理能力的一个实际练习，要求学生理解如何读取和处理结构化数据，以及如何在Java中表示和使用数据。
