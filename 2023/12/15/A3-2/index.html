COM410 编程实践（COM410 Programming in Practice）

A3.2 背包ADT（The Bag ADT）

背包ADT的定义和行为：

  • 集合：是将其他对象分组并提供各种服务（如添加、移除、检索、查询）的对象。
  • 不同的集合存在于不同的目的，其行为抽象地规定，根据集合的类型会有所不同。
背包的定义：
  • 是一个有限的、无特定顺序的对象集合。
  • 对象具有相同或相关的数据类型。
  • 可以包含重复的项目。
可能的行为：
  • 获取项目数量。
  • 检查是否为空。
  • 添加、移除对象。

总结：

背包ADT是编程中用来管理集合数据的一种抽象数据类型。通过背包ADT，我们可以操作集合中的元素，如添加或移除元素。这种ADT适用于那些不需要特定顺序，且可能包含重复元素的场景。例如，购物车的功能就可以用背包ADT来实现，顾客可以多次添加相同的商品，同时也可以轻松地移除它们。

背包行为（Bag Behaviours）

背包的基本行为包括：

  • 获取背包内的物品数量。
  • 相关行为：检查背包是否为空。

添加和移除对象：

  • 向背包中添加一个对象。
  • 从背包中移除一个对象。
  • 移除背包中某个特定对象的一个实例。
  • 移除背包中的所有对象。

内容相关的行为：

  • 测试背包中是否包含某个特定对象。
  • 计算某个特定对象在背包中出现的次数。
  • 列出背包中的所有对象。

总结：

背包ADT模拟现实生活中背包的行为，允许我们添加和移除物品。编程中的背包ADT也提供了查询和管理其内容的方法，如检查是否包含特定物品，以及数出某个物品的数量。这些行为让背包ADT成为一个灵活的数据结构，适用于需要收集和管理群体对象的场景。

定义背包（Specifying a Bag）

在Java中实现背包之前，我们需要描述其数据并详细指定方法（对应背包的行为）：

  • 为方法命名，选择它们的参数，决定它们的返回类型，并编写注释来描述它们对背包数据的影响。

已识别的行为及其方法签名：

  • 获取背包当前的物品数量（Get the number of items currently in the bag）:
    • 方法签名：int getCurrentSize()
  • 检查背包是否为空（See whether the bag is empty）:
    • 方法签名：boolean isEmpty()
  • 向背包中添加一个给定对象（Add a given object to the bag）可能涉及到一些设计决策。

总结：

定义背包的过程涉及到为每一个行为创建相应的方法，并为这些方法提供清晰的签名和文档说明。这样做可以确保背包的实现既符合预期的功能，也便于其他开发者理解和使用。例如，getCurrentSize() 方法将返回背包中的项目数量，而 isEmpty() 方法将返回一个布尔值，指示背包是否为空。添加对象的方法需要进一步设计，以决定如何处理重复项等问题。

定义背包（Specifying a Bag）

当我们想要在背包中添加一个给定对象时，我们可以命名方法并为其提供一个参数来代表新项：

  • 方法命名为 addNewEntry(newItem)。

但是，如果背包已满，我们试图添加一个新条目时，我们可能会考虑让这个方法是 void 类型，但我们应如何处理呢？

我们可以采取的一些选项：

  • 什么都不做（忽略它并保持背包不变）。
  • 保留背包不变并向客户端发出信号（我们如何通知客户端？）。

我们可以返回一个布尔值来表示成功与否：

  • boolean addNewEntry(T newItem)（其中 T 表示 newItem 的类型 - 更多细节稍后说明）。

总结：

在背包ADT的实现中，定义添加元素的方法时需要考虑背包容量已满的情况。addNewEntry() 方法的设计决策包括是否进行操作、如何向用户反馈操作结果等。返回一个布尔值是一种通用的做法，可以明确地告诉调用者操作是否成功。这种设计选择有助于防止错误的发生，并确保程序的健壮性。

定义背包（Specifying a Bag）

接下来的行为都与从背包中移除条目相关：

  • 从背包中移除一个未指定的对象（Remove an unspecified object from the bag）
  • 从背包中移除一个特定对象的一个实例（Remove an occurrence of a particular object from the bag）
  • 从背包中移除所有对象（Remove all objects from the bag）

我们首先需要确定这些操作的返回类型：

  • 从背包中移除所有对象的方法具有简单、明确的功能，且不需要返回任何内容，因此可以定义为：
    • void clear()
  • 从背包中移除一个未指定的对象可能会返回一个对象，或者如果背包为空，则返回null：
    • T remove()

总结：

在定义背包ADT时，我们需要为移除对象的不同情况提供方法。clear() 方法用于清空背包中的所有内容，而 remove() 方法用于移除并返回一个未指定的对象。在实现这些方法时，应考虑它们的返回类型和可能的边界条件，如背包为空时的行为。

定义背包（Specifying a Bag）

当定义从背包中移除一个特定对象的一个实例的方法时，我们面临以下设计决策：

  • 方法可以返回一个布尔值，以指示成功与否：
    • boolean remove(T anEntry)
  • 方法也可以返回被移除的对象，或者在无法移除时返回null：
    • T remove(T anEntry)

尽管这两种选择都是可行的，但在这种情况下，第一个选项通常是首选。即使它不返回被移除的条目，调用者已经有了作为方法参数的条目。

总结：

在设计背包ADT的接口时，决定方法的返回类型是一个重要的考虑点。返回布尔值的方法清晰地告知调用者操作是否成功，而不需要再返回对象本身，因为调用者在调用方法时已经提供了该对象。这种设计可以简化方法的使用，避免冗余，并且在逻辑上更加直观。

定义背包（Specifying a Bag）

剩余的行为不会改变背包的内容：

  • 计算某个特定对象出现的次数（Count the number of times a certain object occurs）：
    • 将频率作为整数返回：int getFrequencyOf(T anEntry)
  • 测试背包是否包含特定对象（Test whether the bag contains a particular object）：
    • 返回真或假：boolean contains(T anEntry)
  • 查看背包中的所有对象（Look at all objects that are in the bag）：
    • 返回背包内容的字符串表示：String toString()

总结：

在定义背包ADT的方法时，不仅需要考虑添加和移除元素，还需要考虑查询功能。getFrequencyOf 方法用于计数特定元素的出现次数，contains 用于检查元素是否存在于背包中。此外，toString 方法提供了一种查看背包所有内容的便捷方式。这些方法共同构成了背包ADT的完整功能集，使得背包不仅能存储对象，还能进行有效的信息检索。

定义背包（总结）（Specifying a Bag (summary)）

完整的方法签名列表如下：

  • int getCurrentSize()
  • boolean isEmpty()
  • boolean addNewEntry(T newEntry)
  • T remove()
  • boolean remove(T anEntry)
  • void clear()
  • int getFrequencyOf(T anEntry)
  • boolean contains(T anEntry)
  • String toString()

请注意，我们尚未决定数据将如何存储——但这是实现问题（而非设计问题）。

总结：

以上方法签名总结了背包ADT的基本操作。这些方法包括获取当前大小、检查是否为空、添加和移除条目、清空背包、获取条目频率、检查是否包含条目以及获取背包的字符串表示。这些操作定义了背包的接口，而背包内部数据的存储方式则留给具体实现时决定，强调了设计与实现的区别。

场景（Scenario）

  • 实现背包抽象数据类型的规格，通过创建一个Java接口类。
  • 在你现有的Anytown项目中，提供一个名为BagInterface.java的接口文件。假设这个数据类型将为前一个场景中的Building对象提供集合存储。
  • 接口类指定了我们ADT的前端 - 即将为外部实体与我们的Bag类实例互动提供的方法。
  • 我们将很快提供这个类的实现，描述这些活动将如何进行。

总结：

这个场景要求开发者在现有的项目中添加一个新的接口，用于定义与背包数据结构交互的方法。这个接口将作为背包类的蓝图，确定其他代码或程序可以如何利用这个背包类。这强调了接口在软件设计中的作用，作为隔离实现细节和提供清晰的功能访问点的一种方式。
