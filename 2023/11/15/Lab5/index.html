实验7 练习1：配置交换机端口安全性 (Lab 7 Ex 1: Configuring Switch Port Security)

拓扑结构 (Topology)

  • 描述了实验中使用的网络拓扑结构，包括一个交换机S1，两台PC和一个未知设备（流氓笔记本）。

地址表 (Addressing Table)

  • 列出了交换机、PC和流氓笔记本的接口、IP地址和子网掩码。

目标 (Objective)

  • 第一部分：配置端口安全性。
  • 第二部分：验证端口安全性。

背景 (Background)

  • 本活动中，您将配置并验证交换机上的端口安全性，通过限制允许发送流量到端口的MAC地址来限制端口的入站流量。

第一部分：配置端口安全性 (Part 1: Configure Port Security)

  • 配置交换机S1的Fast Ethernet端口0/1和0/2的端口安全性。
  • 设置最大MAC地址数，确保只有一个设备可以访问Fast Ethernet端口。
  • 配置端口以动态学习和添加设备的MAC地址到运行配置。
  • 设置安全违规时，Fast Ethernet端口不会被禁用，而是生成安全违规通知并丢弃未知源的数据包。

第二部分：验证端口安全性 (Part 2: Verify Port Security)

  • 从PC1 ping PC2，验证端口安全性是否启用，以及PC1和PC2的MAC地址是否已添加到运行配置中。
  • 将流氓笔记本连接到任何未使用的交换机端口，并注意链路灯变红。
  • 启用端口并验证流氓笔记本是否可以ping通PC1和PC2。
  • 断开PC2并连接流氓笔记本到PC2的端口，验证流氓笔记本无法ping通PC1。

总结：本实验指导通过详细步骤介绍了如何在网络交换机上配置并验证端口安全性，以增强网络安全措施，防止未授权设备访问网络资源。

实验7 练习2：配置SSH (Lab 7 Ex 2 - Configuring SSH)

拓扑结构 (Topology)

  • 描述了实验中使用的网络拓扑结构，包括一个交换机S1和一台PC1。

地址表 (Addressing Table)

  • 列出了交换机S1和PC1的接口、IP地址和子网掩码。

目标 (Objectives)

  • 第一部分：安全密码设置。
  • 第二部分：加密通信。
  • 第三部分：验证SSH实现。

背景信息 (Background)

  • 介绍了SSH应替代Telnet进行管理连接，因为SSH提供了强加密，确保了设备间传输数据的安全。

第一部分：安全密码 (Part 1: Secure Passwords)

  • 通过PC1使用Telnet登录到S1，设置用户EXEC和特权EXEC密码。
  • 保存当前配置，并验证密码是否已加密。

第二部分：加密通信 (Part 2: Encrypt Communications)

  • 设置IP域名并生成安全密钥。
  • 创建SSH用户并重新配置VTY线路，以便仅通过SSH访问。

第三部分：验证SSH实现 (Part 3: Verify SSH Implementation)

  • 退出Telnet会话并尝试使用SSH登录，验证是否只能通过SSH进行远程登录。

总结：本实验指导介绍了如何在网络设备上配置SSH服务，以提高远程管理的安全性。这包括密码加密、生成加密密钥、创建SSH用户，以及验证SSH设置是否正确阻止了不安全的Telnet访问。

拓扑结构 (Topology)

  • PC1 通过绿色连线连接到交换机 S1。

总结

这部分展示了网络的基础布局，包括一台个人计算机（PC1）和一台交换机（S1）的连接方式。

地址表 (Addressing Table)

  • 设备 S1 的 VLAN 1 接口的 IP 地址是 10.10.10.2，子网掩码为 255.255.255.0。
  • 设备 PC1 的网络接口卡（NIC）的 IP 地址是 10.10.10.10，子网掩码为 255.255.255.0。

总结

地址表详细列出了网络设备的接口、IP 地址及子网掩码，用于网络配置和管理。

目标 (Objectives)

  • 第一部分：安全密码 (Secure Passwords)
  • 第二部分：加密通信 (Encrypt Communications)
  • 第三部分：验证 SSH 实施 (Verify SSH Implementation)

总结

本节明确了实验的三个主要目标，涉及提高密码安全性、通信加密和验证 SSH 配置的正确性。

背景 (Background)

  • SSH 应替代 Telnet 用于管理连接，因为它为远程连接提供了更强的加密。
  • 在此活动中，将实施密码加密和配置 SSH。

总结

此部分说明了使用 SSH 替代 Telnet 的原因和重要性，为接下来的配置活动提供了背景信息。

第一部分：安全密码 (Part 1: Secure Passwords)

  1. 使用 PC1 的命令提示符，通过 Telnet 连接到 S1。使用 EXEC 模式的密码是 cisco。
  2. 保存当前配置，以便任何错误都可以通过切换 S1 的电源来撤销。
  3. 显示当前配置，并注意密码是以明文形式存在。输入命令加密明文密码。
  4. 验证密码是否已经被加密。

总结

在这一部分，我们将学习如何使用 Telnet 进行连接，查看和修改配置以加密密码。

第二部分：加密通信 (Part 2: Encrypt Communications)

  1. 设置 IP 域名并生成安全密钥。
  2. 创建一个 SSH 用户并重新配置 VTY 行，以便只允许 SSH 访问。

总结

此节介绍了如何增强网络通信的安全性，包括设置用于 SSH 的域名、生成密钥、创建用户及配置远程访问。

第三部分：验证 SSH 实施 (Part 3: Verify SSH Implementation)

  1. 尝试使用 Telnet 登录，登录应该会失败。
  2. 尝试使用 SSH 登录，验证登录是否成功。
  3. 如果能够成功使用 SSH 登录，保存配置；如果失败，重启第一部分的步骤。

总结

验证部分确保了 SSH 的实施正确性，并确保了远程访问的安全性。
