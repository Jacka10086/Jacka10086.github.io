1.Authentication Protocol Consideration

让我们考虑一个简单的身份验证协议，在此协议中客户端尝试向远程服务器验证其身份。在认证方案中，使用了哈希算法H，并且为了合并两个二进制词，使用了一个函数M。在注册时，客户端可能需要提供其身份（即用户名）ID和密码P。在我们的符号表示中，H(P)是密码P的哈希值。类似地，M(H(P), R)是合并了两个词H(P)和R的结果。

该协议的工作流程如下：

  1. 客户端发送其ID。
  2. 服务器回复并生成一个随机数R，然后发送给客户端。
  3. 客户端发送M(H(P), R)，这是将R与密码P的哈希值H(P)混合和合并的结果。
  4. 服务器计算M(H(P), R)，如果计算结果与接收的值相等，则认证成功。
  5. 这种认证会定期重复，并且每次使用不同的R值。
  6. 如果任何认证操作失败或超时，客户端就会被登出。

以下哪项陈述是错误的：

  • a. 服务器不必存储任何针对身份ID的密码P。相反，系统只需要存储哈希值H(P)和身份（或用户名）ID。
  • b. 如果使用合适的方法M，密码的哈希值在传输中不能被捕获。因此，即使M(H(P), R)被截获，密码的哈希值也会保持隐藏。
  • c. 所提出的协议可以被看作是通过密码进行的挑战-响应认证协议，其中随机数R是挑战。
  • d. 系统必须合并并混合密码P与随机数R，然后对结果进行哈希处理；因此密码P不会以明文形式存储。

答案解析

错误的陈述是：

  • d. 系统必须合并并混合密码P与随机数R，然后对结果进行哈希处理；因此密码P不会以明文形式存储。

选项d的说法与协议的描述不符。根据题干，客户端发送的是M(H(P), R)，这是密码P的哈希值H(P)和服务器生成的随机数R经过某种特定函数M处理的结果。题干中并没有提及再次对M(H(P), R)的结果进行哈希处理。所以，d选项中的“然后对结果进行哈希处理”是不正确的。

2.题目翻译

将四种经典的认证方法与部署的潜在缺点或可能限制其部署的场景相匹配：

  • Something the individual knows（个人知道的事物）
  • Something the individual is（个人的生物特征）
  • Something the individual possesses（个人拥有的物品）
  • Something the individual does（个人的行为特征）

A. 丢失记忆卡和智能卡的风险以及部署读卡器的成本可能很高
B. 生物特征收集的硬件和传感器成本可能很高
C. 容易因年龄相关的记忆力丧失而受到影响的老龄用户群
D. 由于医疗条件的变化导致的行为变化

答案解析

  1. 个人知道的事物：通常指的是基于知识的认证方式，如密码或PIN码。相关的部署缺点可能是：

    • C. 容易因年龄相关的记忆力丧失而受到影响的老龄用户群。

    这是因为随着人口老龄化，像记忆力丧失这样的认知问题可能会变得更加普遍，使得个体难以记住密码或PIN码。

  2. 个人的生物特征：指的是生物认证，如指纹或视网膜扫描。相关的部署缺点可能是：

    • B. 生物特征收集的硬件和传感器成本可能很高。

    生物特征系统需要专门的硬件和传感器，在大规模部署时成本可能会很高。

  3. 个人拥有的物品：通常涉及到令牌，如智能卡或安全令牌。对应的缺点是：

    • A. 丢失记忆卡和智能卡的风险以及部署读卡器的成本可能很高。

    智能卡等令牌可能会丢失或被盗，而读取这些令牌所需的读卡器也可能部署成本高昂。

  4. 个人的行为特征：涉及行为生物特征，如击键动态或签名分析。相关的缺点可能是：

    • D. 由于医疗条件的变化导致的行为变化。

    由于各种医疗条件的变化，行为模式可能会发生变化，这会影响这种认证方法的可靠性。

以上对应关系基于安全领域和认证系统部署中已知的常见挑战。

3.题目翻译

以下关于ACL（访问控制列表）的说法哪个是正确的：

  • ACL在收到数据包时会按顺序检查所有条目；如果没有找到匹配项，数据包将被丢弃。
  • ACL在收到数据包时会一直按顺序检查条目，直到找到第一个匹配项；或者到达隐式的列表末尾‘拒绝所有’声明。
  • ACL至少要有一个拒绝语句；否则所有流量都将被允许。
  • ACL可以配置并用于在任何扩展的ACL语法中控制L3和L4流量，为了避免阻止所有流量，应该允许流量被引导到一个虚拟端口。

答案解析

正确的陈述是：

  • ACL在收到数据包时会一直按顺序检查条目，直到找到第一个匹配项；或者到达隐式的列表末尾‘拒绝所有’声明。

这是因为ACL的工作原理是按顺序检查每个条目，一旦找到与数据包匹配的规则，它就会执行相应的动作（允许或拒绝）并停止进一步检查。如果ACL中没有规则与数据包匹配，则会执行列表末尾的隐式‘拒绝所有’动作，导致数据包被丢弃。这是网络安全策略中常见的“默认拒绝”原则。

4.题目翻译

使用下方提供的可能选项中的适当词语完成下面的ACL声明。可能的选项是：L3, L4, deny, ports, permit, filter, inbound, outbound, incoming, upcoming, firewall, sniffer, switch。

扩展访问控制列表（ACLs）可以允许或拒绝L3和_______流量；然而，标准的ACL只控制_______流量。扩展的ACL可以非常精细，因为它可以根据源和目的IP地址、_______以及诸如IPv4、IPv6、ICMP、UDP和TCP等类型的流量进行目标化流量控制。

答案解析

正确的句子填空应该是：

扩展访问控制列表（ACLs）可以允许或拒绝L3和L4流量；然而，标准的ACL只控制inbound或outbound流量。扩展的ACL可以非常精细，因为它可以根据源和目的IP地址、ports以及诸如IPv4、IPv6、ICMP、UDP和TCP等类型的流量进行目标化流量控制。

解析如下：

  • L3和L4分别指的是OSI模型中的网络层和传输层，ACL可以根据这两层的信息来允许或拒绝流量。
  • 标准ACL通常控制入站（inbound）或出站（outbound）流量，这取决于ACL应用于网络设备的哪个接口。
  • 扩展ACL除了可以控制基于IP地址的流量之外，还可以控制基于端口号的流量，这使得它能够管理特定的服务或应用程序（例如，根据TCP或UDP端口）。

5.题目翻译

以下哪个命令可以在ACL中用来阻止10.10.1.0/24网络访问10.10.30.0/24网络：

  • Router (config)#access-list 101 deny 10.10.0.0 0.0.0.255
  • Router (config)#access-list 101 permit ip host 10.10.1.1 host 10.10.30.2
  • Router (config)#access-list 101 permit ip host 10.10.1.1 10.10.30.0 0.0.0.255
  • Router (config)#access-list 101 deny ip 10.10.1.0 0.0.0.255 10.10.30.0 0.0.0.255

答案解析

正确的命令是：

  • Router (config)#access-list 101 deny ip 10.10.1.0 0.0.0.255 10.10.30.0 0.0.0.255

解析如下：

该命令使用了标准的访问控制列表（ACL）语法，其中包含了源IP地址、源通配符掩码、目的IP地址和目的通配符掩码。这条命令表示拒绝从10.10.1.0/24网络到10.10.30.0/24网络的所有IP流量。其中，10.10.1.0是源网络地址，0.0.0.255是对应的通配符掩码，用于表示/24子网；同样，10.10.30.0是目的网络地址，其通配符掩码也是0.0.0.255。这表示ACL将匹配所有源地址在10.10.1.0到10.10.1.255之间，且目的地址在10.10.30.0到10.10.30.255之间的IP包，并将它们拒绝。

6.题目翻译

网络管理员负责使用ACLs来保护路由器。管理员什么时候会使用 any 配置选项或命令？

  • 为了添加文本条目以作注释
  • 为了生成并发送一个信息性
  • 为了路由数据
  • 为了识别任何IP地址

答案解析

当网络管理员需要在ACL中指定规则以匹配任何IP地址时，他们会使用 any 配置选项或命令。在ACL规则中，any 关键字通常用于不限定特定源或目的IP地址的场合，表示匹配所有的IP地址。因此，正确答案是：

  • 为了识别任何IP地址

这个选项表明 any 关键字在ACL中的使用，是为了创建一个通用规则，该规则适用于所有的IP地址。例如，在拒绝或允许所有出入流量的情况下，管理员可能会选择使用 any 关键字来简化配置。

7.题目翻译

一名网络工程师正在审查他/她在一个（虚构的）路由器接口中找到的ACLs，这些接口禁用了IPv6流量（只允许IPv4流量）。该网络工程师找到了15个标准ACLs。以下哪个接口数量可能是该路由器在此场景下活动接口的数量：

  • 5个接口
  • 6个接口
  • 7个接口
  • 8个接口

答案解析

在路由器中，每个接口可以应用一个入站和一个出站的ACL。因此，如果一个网络工程师找到15个标准ACLs，最多可以有7个接口使用这些ACLs（每个接口一个入站和一个出站，加起来是14个ACLs），并且还有一个剩余的ACL。如果有8个接口，那么需要16个ACLs（每个接口两个，分别对应入站和出站）。

因此，正确答案是：

  • 7个接口

这个选项意味着在这15个ACLs中，有一个ACL没有被使用，或者是某个接口只配置了入站或出站的ACL之一。

9.题目翻译

将每个通配符掩码（每个掩码应该只用一次）与部分显示的ACL条目匹配，应该使用在哪里：

  • 这个ACL条目拒绝来自10.10.0.0/16网络的所有主机
  • 这个ACL条目仅拒绝主机10.1.10.1
  • 这个ACL条目允许来自10.10.10.0/29网络的所有主机
  • 这个ACL条目允许来自10.0.0.0/8网络的所有主机

a. 0.0.0.0
b. 0.0.0.7
c. 0.255.255.255
d. 0.0.255.255

答案解析

正确的匹配应该是：

  • 这个ACL条目拒绝来自10.10.0.0/16网络的所有主机 - d. 0.0.255.255
  • 这个ACL条目仅拒绝主机10.1.10.1 - a. 0.0.0.0
  • 这个ACL条目允许来自10.10.10.0/29网络的所有主机 - b. 0.0.0.7
  • 这个ACL条目允许来自10.0.0.0/8网络的所有主机 - c. 0.255.255.255

解析如下：

  • 通配符掩码是子网掩码的反码。在ACL中，0意味着对应位必须完全匹配，255意味着对应位可以是任何值。
  • 对于/16子网掩码（255.255.0.0），通配符掩码是0.0.255.255。
  • 单个主机的子网掩码是/32（255.255.255.255），因此通配符掩码是0.0.0.0，表示没有其他位可以变化。
  • 对于/29子网掩码（255.255.255.248），通配符掩码是0.0.0.7，表示最后3位可以变化（允许的主机数为8个）。
  • 对于/8子网掩码（255.0.0.0），通配符掩码是0.255.255.255。

10.题目翻译

参考下图。编写并完成一个扩展ACL条目的缺失字段，以便禁止来自10.1.1.0/24网络的PC（10.1.1.3，图中未显示）向10.1.3.0/24网络上的web服务器（10.1.3.8）发送ping消息。

这个ACL被应用于R2 Fa0/0入站。可能的选项列表是：150, host, any, 10.1.3.8, eq, 80, 10.1.3.0, 10.1.2.0, access-list, 50, deny, any, 10.1.1.3, permit, 10.1.1.0, 0.0.0.255, eq 21。你不能在这个列表中多次使用同一个词。

你应该完成的ACL声明是：access-list 101 ________ icmp ________ ________ echo

答案解析

正确的ACL声明应该是：

access-list 101 deny icmp host 10.1.1.3 host 10.1.3.8 echo

解析如下：

  • deny 用于禁止流量。
  • icmp 是因为ping使用ICMP协议。
  • host 10.1.1.3 表示特定的源IP地址。
  • host 10.1.3.8 表示特定的目的IP地址。
  • echo 表示ICMP echo请求，也就是ping命令。

所以，这条ACL规则意味着任何来自IP地址为10.1.1.3的主机的ICMP echo请求（ping）都不允许到达IP地址为10.1.3.8的主机。
