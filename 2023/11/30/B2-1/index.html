B2.1 排序介绍

  • 定义排序

    • 排序是将一组项目基于特定标准进行排列的过程。
    • 我们生活中处处可以看到排序的例子，如按字母顺序排列的联系人列表、按数值排序的扑克牌、按日期或标题排序的书籍等。

    总结：本部分介绍了排序的定义和生活中的排序例子，强调了排序在组织信息时的普遍性和重要性。

B2.2 排序的实际应用

  • 排序的例子

    • 一个手机上的联系人列表，通常按照字母顺序排列。
    • 一副扑克牌，可能按照数字或者花色来排序。
    • 一排书籍，可以按照出版日期或者书脊上的标题进行排序。

    总结：这一部分通过具体的例子说明了排序在不同场景下的应用，帮助学生理解排序的多样性和实用性。

The Java Comparable Interface (Java 可比较接口)

基本概念

  • 必须能够比较一个对象与另一个对象，以定义能够对任意对象集合进行排序的算法。

  • 虽然我们的大多数例子将会对整数进行排序，但Java实现将能对任何实现了Comparable接口的对象进行排序。

  • Comparable接口包含一个compareTo()方法，它被设计用来返回一个整数，指定两个对象之间的关系。

    总结：本节介绍了Java中Comparable接口的重要性，它允许我们定义如何对对象集合进行排序。compareTo()方法是这一过程的核心，它通过返回一个整数来表达两个对象之间的关系。

compareTo() 方法的实现

  • compareTo() 方法的实现应当：

    • 确立对象间的完全排序。
    • 如果obj1小于、等于或大于obj2，则分别返回一个负整数（通常为-1）、零或正整数（通常为1）。

    • 图中展示了三种可能的比较结果：obj1小于obj2时返回-1，obj1等于obj2时返回0，obj1大于obj2时返回1。

    总结：这部分详细解释了compareTo()方法如何实现，以及它如何通过返回不同的整数值来定义两个对象之间的大小关系。图示帮助理解这一点，展现了不同的比较结果对应的返回值。

Sorting (排序)

排序的概念

  • 比较两个对象的方式取决于对象的性质和它们的属性（如标题、作者、大小、高度、颜色、名称等）。
  • 目前，我们寻找的算法能够将集合中的n个条目进行排序，使得：entry 1 ≤ entry 2 ≤ ... ≤ entry n。

数组排序

  • 通常情况下，排序一个数组比排序一个链表更为简单。

  • 我们的算法将重新排列数组中的n个值，使得：a[0] ≤ a[1] ≤ ... ≤ a[n - 1]。

    总结：这部分讲述了排序的基本原理，说明了排序取决于对象的属性。它还指出，对于算法来说，数组排序通常比链表排序来得更简单，因为数组的连续内存布局使得它们更容易被访问和重新排列。这是计算机科学中数据结构和算法基本概念的核心部分。

Swapping Elements (交换元素)

数组元素交换概念

  • 我们有一个包含5个元素的数组 char[] letters = { "A", "B", "C", "D", "E" };。
  • 直接复制一个数组单元格的值到另一个单元格将导致数据丢失。

交换过程

  • 我们希望交换元素1（”B”）和元素3（”D”）的内容。
  • 解决方案是使用一个临时变量来完成交换，以避免数据丢失。

交换步骤

  1. 将元素1的值复制到临时变量中：char temp = letters[1];
  2. 将元素3的值复制到元素1的位置：letters[1] = letters[3];
  3. 将临时变量中的值（原始元素1）复制到元素3的位置：letters[3] = temp;

总结：本节说明了在数组中交换两个元素的正确方法。通过使用临时变量，我们可以确保在交换过程中不会丢失任何数据。这是计算机科学中基础而重要的数组操作技巧。

Bubble Sort (冒泡排序)

排序过程

  • 冒泡排序通过重复比较相邻元素并在必要时交换它们的位置来对值列表进行排序。
    • 扫描列表，如果相邻元素的顺序不正确就交换它们（这样做会将最大的值“冒泡”到顶部）。
    • 再次扫描列表，将第二大的值“冒泡”上来。
    • 重复这个过程，直到所有元素都在列表中的正确位置上。

交换操作

  • 每次交换操作都会将一个元素移动到其最终正确的位置。

    总结：冒泡排序是一种简单的排序算法，它重复遍历要排序的数列，一次比较两个元素，如果它们的顺序错误就把它们交换过来。遍历数列的工作是重复进行的，直到没有再需要交换的元素，这意味着数列已经排序完成。

Bubble Sort (冒泡排序)

第一次遍历

  • 考虑数组的第一次遍历，目的是将最大的值移动到数组的最后一个元素。
    • 从数组的第一个元素开始，比较相邻的元素并进行交换，以确保较大的数值“冒泡”到数组的顶端。
    • 第一次遍历结束时，最大的元素会被放置在数组的最后一个位置。

剩余的遍历

  • 在随后的每次遍历中，重复上述过程，但每次都可以忽略已经排序好的最后一个元素。
    • 每次遍历都会找到未排序部分的最大元素，并将其“冒泡”到正确的位置。
    • 这个过程会重复进行，直到所有的元素都正确排序。

冒泡排序 - 第一版本（Bubble Sort - version 1）

  • 算法伪代码（Pseudocode of algorithm）

    • 冒泡排序算法对数组a中的条目进行排序。
    • 设置最后位置为数组长度减1。
    • 设置内部最后位置为最后位置。
    • 从0循环到最后位置减1。
    • 内部从0循环到内部最后减1。
    • 如果条目a[j]和a[j+1]顺序错误，则交换位置。
    • 将内部最后位置减1设置为内部最后。

    伪代码提供了冒泡排序的基本步骤，通过比较相邻元素的顺序来进行排序，如果顺序错误则交换它们的位置。

  • 效果评价（Works OK）

    • 这种方法可以工作，但可能会在数组已经排序后仍然继续检查值。

    评价指出虽然该算法能够完成排序任务，但其效率不是最优的，因为它没有在数组已经排序好之后停止检查。

  • 改进方案（A better approach）

    • 一个更好的方法是跟踪已经进行的交换，并在没有更多交换时停止。

    改进方案建议通过跟踪交换操作来优化算法，从而在数组已经排序完成时停止算法运行，以提高效率。

总结

本段内容描述了冒泡排序算法的一个版本，包括其伪代码、效果评价以及改进方案。伪代码详细列出了算法的步骤，效果评价指出了其潜在的效率问题，改进方案则提出了一种优化方法。

冒泡排序 - 改进版（Bubble Sort - improved）

  • 算法改进点（Keep track of the last swap made and exit when none is required）

    • 跟踪最后一次交换的位置，如果不需要交换则退出算法。

    通过记录最后一次交换的位置，算法可以在确定数组已经排序完成后立即终止，从而提高效率。

  • 改进后的伪代码（Algorithm bubbleSort(a)）

    • 设置第一个位置为0。
    • 设置最后位置为数组长度减1。
    • 当第一个位置小于最后位置时，进行以下操作：
      • 将最后交换位置设置为第一个位置。
      • 从第一个位置循环到最后位置减1。
      • 如果条目a[i]和a[i+1]顺序错误，则交换位置，并将最后交换位置设置为i。
      • 将最后位置设置为最后交换的位置。

    改进后的伪代码展示了一个更高效的冒泡排序算法，它通过减少不必要的比较来缩短排序时间。

总结

改进版的冒泡排序通过跟踪交换操作，减少了排序过程中的无用比较，提高了算法的效率。伪代码清晰地展示了改进后的排序步骤，使得算法在达到排序目的后可以更快地完成。

冒泡排序的效率（Efficiency of Bubble Sort）

  • 改进后的冒泡排序（The improved Bubble Sort）

    • 改进后的冒泡排序保持对最后一次交换的记录，从而避免了主循环中的冗余步骤。

    该点强调了改进后冒泡排序算法的效率优势，即通过记录交换来避免不必要的循环迭代。

  • 最佳情况（Best case）

    • 如果数组已经是升序排列的，冒泡排序只需进行N-1次比较和0次交换，时间复杂度为O(n)。

    最佳情况说明了在数组已经有序的情况下，算法的运行时间与数组大小呈线性关系，效率较高。

  • 最坏情况（Worst case）

    • 如果数组是降序排列的，处理过程需要N-1次遍历；在第i次遍历中，会有N-i次比较和N-i次交换，因此外循环和内循环的时间复杂度都是O(n)，整体为O(n^2)。

    最坏情况描述了在数组完全逆序的情况下，算法的运行时间与数组大小的平方成正比，效率较低。

  • 平均情况（Average case）

    • 平均情况更为复杂，因为可能会跳过一些遍历，但平均的遍历次数和交换次数仍然是O(n)，因此总的时间复杂度为O(n^2)。

    平均情况讨论了在一般随机数组中，算法的平均效率仍然受到N^2级别的影响，但由于可能跳过某些遍历，实际性能可能会有所提高。

总结

本段内容讨论了冒泡排序算法的效率，包括改进后的冒泡排序避免冗余步骤的优点，以及在最佳、最坏和平均三种情况下的时间复杂度。改进后的冒泡排序在最佳情况下效率显著，但在平均和最坏情况下，时间复杂度仍然是O(n^2)，这表明尽管进行了优化，冒泡排序在处理大型数据集时可能仍然不是最优选择。

O(n²) 方法的局限性 (Limitations of O(n²) methods)

这部分详细解释了时间复杂度为O(n²)的排序算法，如冒泡排序、选择排序和插入排序的局限性。

O(n²)算法简介

  • 这些算法在小数据集上表现良好。
  • 随着数据量的增加，它们的效率会急剧下降。

O(n²)算法的具体局限

  • 冒泡排序：实现简单，但效率低下。
  • 选择排序和插入排序：与冒泡排序类似，但在不同情况下可能更有效。
  • 所有这些算法都有执行时间与要排序元素数量的平方成正比，随着元素数量的增加，排序时间急剧增长。
  • 这些方法不适合大数据集的排序。

在这个部分中，我们总结了O(n²)算法在处理大规模数据集时的不足，这也是为什么在现代应用中，更倾向于使用时间复杂度更低的算法，如快速排序或归并排序。
