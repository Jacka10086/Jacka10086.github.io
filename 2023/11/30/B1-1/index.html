Ulster University（阿尔斯特大学）

COM410 Programming in Practice（COM410 编程实践）

B1.1 Measuring Algorithm Performance（B1.1 测量算法性能）

  • 在计算机科学的早期，Charles Babbage就提出了一个关于算法性能的问题，他问道：“一旦分析引擎存在，它必将指导科学的未来进程。每当需要它的帮助来寻求结果时，一个问题就会出现 — 通过计算的什么过程可以使这些结果在最短的时间内被机器得出？”这个问题突显了计算效率在算法设计中的重要性。

  • 现代的挑战是如何利用程序解决实际的大规模问题，以及如何理解并改进程序的性能特征。这涉及到算法分析，优化以及性能测试等方面，是计算机科学中非常关键的技能。

总结：阿尔斯特大学的COM410编程实践课程中，学习测量算法性能的重要性，从Charles Babbage的时代就开始关注算法效率，到现代如何应用这些原理来解决大规模问题，并持续优化算法性能。

The Challenge (earliest days …)（挑战（最早的日子…））

  • 图片中展示了Charles Babbage的引语，和他设计的分析引擎的图片，这代表了计算机科学的黎明。他的引言强调了效率在计算过程中的重要性，这直接关联到现代计算机科学中的性能测试。

  • “你需要转动多少次手柄？” 这句话可能象征性地表达了在早期计算机科学中，执行一个算法所需的物理努力，也隐喻了优化算法以减少计算步骤的重要性。

总结：早期的挑战在于算法的效率和执行所需的物理努力，Charles Babbage的分析引擎和他的话语体现了计算机早期的发展和性能优化的重要性。

The Challenge (modern version …)（挑战（现代版本…））

  • 现代版本的挑战转变为如何利用编程来解决实际的大问题，以及如何理解和提高程序的性能特征。这包括了解算法的时间复杂度、空间复杂度和其他性能指标。

  • 图片展示了一个早期的大型计算机系统，可能是用于科学计算或数据处理的，这样的系统在当时可能代表了计算能力的极限。现代的挑战在于不仅要维持程序的可靠性和准确性，还要在速度和效率方面进行优化。

总结：现代的挑战涉及到利用程序解决大规模问题，以及对程序性能的深入理解和优化。这反映了从早期的物理劳动转向了更加抽象的思维挑战，强调了算法性能分析在现代编程实践中的核心地位。

为什么要学习程序性能？（Why Study Program Performance?）

  • 预测程序行为（To predict program behaviour）
    • 我的程序会结束吗？（Will my program finish?）
    • 我的程序何时结束？（When will my program finish?）
  • 比较算法和实现（To compare algorithms and implementations）
    • 这个变化会使我的程序运行得更快吗？（Will this change make my program faster?）
    • 我怎样才能使程序运行得更快？（How can I make my program faster?）
  • 为理解问题和开发新算法打下基础（To develop a basis for understanding the problem and developing new algorithms）
    • 使新技术成为可能（Enables new technology）

总结：学习程序性能可以帮助我们预测和改进程序的运行结果，选择更有效的算法，并促进新技术的发展。

示例 - 计算正整数的和（Example - Compute the Sum of Positive Integers）

  • 计算正整数 n 大于 0 的 1 到 n 的和（Compute the sum 1 + 2 + … + n for any positive integer n > 0）
  • 解决这个问题的三种可能算法（Three possible algorithms for solving this problem）
    • 算法 A（Algorithm A）
      • 使用 for 循环逐个加和（Uses a for loop to add each number successively）
    • 算法 B（Algorithm B）
      • 使用两个嵌套的 for 循环（Uses two nested for loops）
    • 算法 C（Algorithm C）
      • 使用数学公式直接计算（Uses a direct mathematical formula）

总结：介绍了三种计算正整数和的算法，包括逐个相加、嵌套循环和直接数学公式计算。

情景分析 - 比较算法性能（Scenario - Compare Algorithm Performance）

  • 创建一个名为 Analysis 的新 Java 项目，实现 SumIntegers 类（Create a new Java project called Analysis and implement the class SumIntegers）
    • 方法 sumA()，sumB() 和 sumC() 作为前一幻灯片上算法的实现（methods sumA(), sumB() and sumC() as implementations of the algorithms presented on the previous slide）
    • 每个方法接受一个 int 参数 n 并返回所有小于等于 n 的整数的 long 型和（Each method takes an int parameter n and returns the long sum of all integers <= n）
  • 用参数值 10000 调用每个方法并验证它们返回相同的结果（Call each method with parameter value 10000 and verify that they return the same result）
  • 现在，添加代码来计时每种方法的执行，并以纳秒为单位打印出执行时间（Now, add code to time the execution of each method and print out the execution time in nanoseconds）
  • 验证算法执行速度的明确顺序（Verify that there is a clear order in the execution speed of the algorithms）

总结：通过创建 Java 项目并实现不同算法来比较其性能，包括实现细节、执行时间的计时和结果的验证。

算法分析（Analysis of Algorithms）

  • 如何衡量效率以比较解决问题的不同方法？（How can we measure efficiency so that we can compare various approaches to solving a problem?）
    • 实现多个想法后再选择一个不是实际可行的（Implementing several ideas before choosing one isn’t practical/feasible）
    • 程序的执行时间取决于计算机和使用的语言（A program’s execution time is dependent on computer and language used）
  • 算法同时具有时间和空间约束（复杂性）（An algorithm has both time and space constraints (complexity)）
    • 时间复杂度（执行所需的时间）（Time complexity (time it takes to execute)）
    • 空间复杂度（执行所需的内存）（Space complexity (memory needed to execute)）

总结：算法分析旨在衡量算法的效率，需要考虑时间和空间复杂度。理解这些概念有助于选择或设计更优的算法。

算法分析（Analysis of Algorithms）

  • 通常最佳解决方案要平衡多种标准（时间、空间、通用性、编程工作等）（Usually the best solution to a problem balances out various criteria (time, space, generality, programming effort, etc.)）
  • 重点将放在时间复杂度上（Focus will be on time complexity）
  • 时间复杂度和空间复杂度之间通常存在反比关系（An inverse relationship often exists between an algorithm’s time complexity and its space complexity）
    • 如果你修改算法以节省执行时间，通常需要更多空间（If you revise an algorithm to save execution time, you will usually need more space）
    • 如果你减少算法的空间需求，它很可能需要更多的执行时间（If you reduce an algorithm’s space requirements, it will likely require more time to execute）

总结：在算法分析中，最优解通常涉及在执行速度与占用空间之间找到平衡点。了解时间与空间复杂度之间的关系对于优化算法至关重要。

算法分析（经验性）（Analysis of Algorithms (Empirical)）

  • 可以使用经验分析来计算算法的运行时间（Empirical analysis may be used to compute the running time of an algorithm）
  • 运行实验（Run experiments）
    • 从适中的输入大小 N 开始（Start with a moderate input size N）
    • 测量并记录运行时间（Measure and record running time）
    • 输入大小 N 翻倍（Double input size N）
    • 重复上述步骤（Repeat）
    • 制表并绘制结果（Tabulate and plot results）
  • 表格和图表解释（Tabulate and plot results）
    • 表格显示了随着问题大小增加，算法运行时间的增长（The table shows the growth of algorithm running time as the problem size increases）
    • 对数-对数图展示了问题大小和运行时间之间的对数关系（The log-log plot shows the logarithmic relationship between problem size and running time）

总结：经验性算法分析涉及通过实验来测量算法随输入大小变化的执行时间，并通过表格和图表形式进行结果可视化，有助于理解算法性能随问题规模扩大的变化趋势。
