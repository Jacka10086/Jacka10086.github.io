大O记号介绍 (COM410 Programming in Practice - B1.3 The Big O Notation)

  • 大O记号是计算机科学中用来表示算法复杂度的一种表示方法。

大O记号定义 (Big O Notation)

  • 计算机科学家用一种记号来表示算法的复杂度。
    • 算法A的时间需求与n成正比。
    • 算法B的时间需求与n^2成正比。
    • 算法C的时间需求是常数。

大O记号举例 (Big O Notation Examples)

  • 图表解释
    • 图表显示了三种算法的操作数量与输入大小的关系。
    • 算法A呈线性增长，算法B呈二次方增长，算法C操作数量不变。

大O记号的运用 (Application of Big O Notation)

  • 大O记号用于：
    • 表达应用程序的运行时间，
    • 描述运行时间相对于输入的增长速度，
    • 以及在输入大小任意增大的情况下的表现。

总结

大O记号是一种评估算法效率的工具，它可以帮助我们了解算法处理数据的速度以及当数据规模增大时算法性能的变化。通过大O记号，我们可以很清楚地看到不同算法在处理大量数据时可能遇到的性能瓶颈。

常量时间 (Constant Time)

定义与代码示例

  • 常量时间是指无论输入数据的大小如何变化，算法的执行时间都保持不变。
  • Java代码示例：
    • public static void printFirstItem(int[] items) {
    • System.out.println(items[0]);
    • }

执行时间分析

  • 无论数组items的大小如何，printFirstItem函数都只会打印数组的第一个元素。
  • 执行时间为O(1)，也就是说，它是一个常量时间的操作。

深入理解

  • 时间复杂度：在计算机科学中，我们用大O表示法来描述算法的运行时间或空间需求与输入数据大小之间的关系。
  • 常量时间复杂度：当一个算法的运行时间不随着输入数据的增加而增加，我们称它具有常量时间复杂度。

中文总结

常量时间是算法复杂度分析中的一个概念，指的是一个操作的执行时间不会因为输入数据的大小而变化，即这个时间是一个常数。例如，访问数组的第一个元素，无论数组有多大，所需时间都是固定的，这就是一个常量时间的操作，用大O表示法表示为O(1)。

线性时间 (Linear Time)

定义与代码示例

  • 线性时间指的是算法执行时间与输入数据的大小成正比。
  • Java代码示例：
    • public static void printAllItems(int[] items) {
    • for (int item : items) {
    • System.out.println(item);
    • }
    • }

执行时间分析

  • 当使用一个循环遍历数组items中的每个元素并打印时，执行时间随数组大小线性增长。
  • 执行时间为O(n)，表示随着输入数组items的大小线性增加。

深入理解

  • 时间复杂度：O(n)的时间复杂度意味着如果输入数据翻倍，算法的执行时间也大致翻倍。
  • 线性时间复杂度的操作：通常涉及单层循环或单次遍历数据结构。

中文总结

线性时间是指执行时间与输入数据的大小成正比的情况。例如，在一个数组中打印每个元素，需要对数组进行一次遍历，执行时间会随着数组大小的增加而增加。这样的操作时间复杂度为O(n)，表示它是线性时间复杂度的算法或操作。

二次时间 (Quadratic Time)

定义与代码示例

  • 二次时间指的是算法执行时间与输入数据大小的平方成正比。
  • Java代码示例：
    • public static void printAllOrderedPairs(int[] items) {
    • for (int first : items) {
    • for (int second : items) {
    • System.out.println(first + ", " + second);
    • }
    • }
    • }

执行时间分析

  • 通过两个嵌套循环打印数组items中所有可能的有序对，使得执行时间与数组大小的平方成正比。
  • 执行时间为O(n^2)，表示随着输入数组items的大小增加，执行时间增长的速度是其大小的平方。

深入理解

  • 时间复杂度：O(n^2)的时间复杂度通常与嵌套循环相关，每增加一个循环层次，复杂度就会增加一个n的次方。
  • 二次时间复杂度的操作：涉及到数据集合中元素的成对组合，例如排序算法中的冒泡排序或插入排序。

中文总结

二次时间复杂度描述的是算法执行时间与输入数据大小的平方成正比的情况。当算法中包含双层循环，且每一层循环都需要遍历整个数据集时，就会出现这种复杂度，其表示为O(n^2)。例如，打印数组中所有元素的有序对就是一个二次时间复杂度的操作。

忽略常数 (Drop the Constants)

定义与代码示例

  • 忽略常数的原则指在大O表示法中，我们不考虑常数倍的差异，只关注最高次项。
  • Java代码示例：
    • public static void printAllItemsTwice(int[] items) {
    • for (int item : items) {
    • System.out.println(item);
    • }
    • for (int item : items) {
    • System.out.println(item);
    • }
    • }

执行时间分析

  • 即使printAllItemsTwice方法打印数组中的所有元素两次，执行时间仍然与数组大小成正比。
  • 执行时间为O(n)，即使操作数为2n，我们仍然将其视为O(n)，因为我们在分析时忽略了常数倍的差异。

深入理解

  • 时间复杂度简化：在大O分析中，复杂度为O(2n)的算法与O(n)的算法在本质上是等价的，因为当n足够大时，常数倍的差异变得不那么重要。
  • 复杂度的实际意义：这个原则有助于我们更好地理解算法的效率，关注那些随着输入规模增加而显著增加的操作。

中文总结

忽略常数原则是算法复杂度分析中的一个概念，它告诉我们在评估算法的效率时，可以忽视常数因子的影响。即使一个操作被执行了多次，只要这个次数是一个固定的常数，我们在大O表示法中仍然视其为同一级别的复杂度。例如，一个数组的元素被打印两遍，虽然操作数为2n，但复杂度仍然是O(n)。

使用最高次项 (Use the Most Significant Term)

定义与代码示例

  • 使用最高次项原则指在大O表示法中，我们只保留影响最大的项，其他低次项和常数项都可以忽略。
  • Java代码示例：
    • public static void printPairsThenItems(int[] items) {
    • for (int first : items) {
    • for (int second : items) {
    • System.out.println(first + ", " + second);
    • }
    • }
    • for (int item : items) {
    • System.out.println(item);
    • }
    • }

执行时间分析

  • 尽管printPairsThenItems方法先打印所有元素对，然后再打印所有单独的元素，执行时间仍然与输入数组大小的平方成正比。
  • 执行时间为O(n^2)，因为元素对的打印操作(n^2)远大于单独元素的打印操作(n)，后者在这里可以忽略。

深入理解

  • 时间复杂度简化：在大O表示法中，如果一个算法包含多项操作，我们只需考虑增长最快的项。例如，n^2 + n中n^2项增长得更快，所以算法的复杂度就是O(n^2)。
  • 对算法性能的影响：最高次项代表了算法在大规模输入下的性能，这有助于我们在不同算法间做出更有意义的比较。

中文总结

在算法复杂度分析中，最高次项原则告诉我们只需要关注那些随着输入规模增大而增长最快的操作。在这个例子中，虽然算法执行了n^2次打印元素对的操作和n次打印单个元素的操作，但是当我们谈论算法的时间复杂度时，我们只考虑n^2，即O(n^2)，因为它对算法性能的影响最大。

袋(集合)ADT实现的效率

操作及其时间复杂度

  • 添加元素 (add(newEntry))
    • 定长数组：O(1)
    • 链表：O(1)
  • 移除元素 (remove())
    • 定长数组：O(1)
    • 链表：O(1)
  • 移除特定元素 (remove(anEntry))
    • 定长数组：O(1), O(n), O(n)
    • 链表：O(1), O(n), O(n)
  • 清空集合 (clear())
    • 定长数组：O(n)
    • 链表：O(n)
  • 获取元素频率 (getFrequencyOf())
    • 定长数组：O(n)
    • 链表：O(n)
  • 判断包含元素 (contains())
    • 定长数组：O(1), O(n), O(n)
    • 链表：O(1), O(n), O(n)
  • 转换为字符串 (toString())
    • 定长数组：O(n)
    • 链表：O(n)
  • 获取当前大小 (getCurrentSize())
    • 定长数组：O(1)
    • 链表：O(1)

图表解释

  • 表格展示了两种袋(集合)ADT实现——定长数组和链表——的各种操作的时间复杂度。
  • 在两种实现中，大部分操作的时间复杂度相似，但是在移除特定元素和判断包含元素时，时间复杂度有多个可能的结果，这取决于具体实现和情况。

中文总结

这张图表比较了定长数组和链表实现袋ADT时各种操作的时间效率。大多数基本操作如添加、移除、获取大小都有常数时间复杂度O(1)。而对于需要遍历整个集合的操作，如清空集合、获取元素频率、转换为字符串，则时间复杂度为O(n)。特别地，在移除特定元素和判断包含元素时，复杂度可能是常数时间也可能是线性时间，这取决于特定元素在集合中的位置。
