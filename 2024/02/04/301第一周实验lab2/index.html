Lab 2 - Switch Settings (实验2 - 交换机设置)

Topology (拓扑结构)

  • 图中展示了一个简单的网络拓扑结构，包括两台PC（PC1和PC2）以及两台交换机（S1和S2）。
  • PC1通过交换机S1连接到PC2，PC2通过交换机S2连接回PC1，形成一个环形网络。

拓扑结构简单清晰，展示了两台PC通过两台交换机互联的基本网络环境。

Addressing Table (地址表)

  • 表格列出了设备名称、IP地址和子网掩码。
    • PC1的IP地址为192.168.1.1，子网掩码为255.255.255.0。
    • PC2的IP地址为192.168.1.2，子网掩码为255.255.255.0。

地址表为网络配置提供了必要的IP参数，确保了PCs之间的正确通信。

Objectives (目标)

  1. Verify the Default Switch Configuration (验证默认交换机配置)
    • 确认交换机出厂设置是否符合网络的基础运行要求。
  2. Configure a Basic Switch Configuration (配置基础交换机设置)
    • 设置交换机的基础参数，如名称、密码等。
  3. Configure a MOTD Banner (配置消息日志)
    • 设定交换机启动时显示的消息，用于法律声明或欢迎信息等。
  4. Configure the PCs (配置个人计算机)
    • 设置PC的IP地址，以符合地址表中的配置。
  5. Save Configuration Files to NVRAM (将配置文件保存到NVRAM)
    • 确保配置在交换机重启后依然有效，通过保存到非易失性存储器NVRAM实现。

目标部分概述了实验的主要步骤和目的，为用户提供了完成实验的明确指导。

总结：本实验指导涉及了网络拓扑结构的识别、网络设备的IP地址配置，以及交换机的基础设置。实验目标确保用户可以按照步骤操作，完成网络设置和配置的基本教学目的。

Background (背景)

在这个实验中，您将首先进行基本的交换机配置。然后，您将通过配置PC上的IP地址来实现基本的连通性，并使用各种show命令来验证配置，使用ping命令来验证PC之间的基本连通性。您还将学习如何通过加密和普通文本密码保护命令行界面（CLI）和控制台端口的访问权限。

这一部分为整个实验设定了背景，并概述了将要完成的任务。

Part 1: Verify the Default Switch Configuration (第一部分: 验证默认交换机配置)

Step 1: Enter privileged EXEC mode (步骤1: 进入特权EXEC模式)

  • 特权EXEC模式提供了访问所有交换机命令的权限。然而，由于特权命令可以配置操作参数，因此需要用密码来保护，以防止未授权使用。
  • 要进入特权EXEC模式，需要在CLI中输入 enable 命令。

通过特权EXEC模式，用户可以访问更高级的配置命令和设备状态信息，这有助于管理员控制网络设备。

Step 2: Examine the current switch configuration (步骤2: 检查当前交换机配置)

  • 使用 show running-config 命令来查看当前交换机的配置。
  • 根据显示结果回答以下问题：
    1. 交换机有多少 FastEthernet 接口？答案是24个。
    2. 交换机有多少 Gigabit Ethernet 接口？答案是2个。
    3. 哪个命令可以显示非易失性随机存取存储器（NVRAM）中的当前内容？答案是 show startup-configuration。
    4. 如果显示 startup-config is not present，意味着什么？这表示配置文件没有被保存到NVRAM中，当前只在RAM中。

这一部分指导用户如何查看和理解交换机的当前配置，并能够识别与端口和内存相关的关键信息。

总结：第一部分的内容向用户介绍了如何验证交换机的默认配置，包括如何进入特权模式，如何查看当前配置，以及如何理解与接口和存储相关的配置信息。

Part 2: 创建基础交换机配置 (Part 2: Create a Basic Switch Configuration)

Step 1: 为交换机命名 (Step 1: Assign a name to a switch.)

  • 配置交换机参数时，可能需要在不同配置模式之间切换。注意提示符如何随着您在交换机内导航而变化。
    • 进入配置模式：Switch> configure terminal
    • 设置主机名：Switch(config)# hostname S1
    • 退出配置模式：S1(config)# exit

中文总结：此步骤主要是为交换机设置一个名称，需要通过不同的命令进入配置模式，设置主机名后再退出。

Step 2: 保护控制台线路访问 (Step 2: Secure access to the console line.)

  • 为了保护控制台线路的访问，需要进入配置线路模式并设置控制台密码。
    • 进入控制台线路配置模式：S1> configure terminal → S1(config)# line console 0
    • 设置密码：S1(config-line)# password letmein
    • 启用登录要求：S1(config-line)# login

中文总结：此步骤通过设置密码和启用登录命令来保护控制台线路，以防未经授权的访问。

  • 退出至特权模式并验证控制台口令是否生效。
    • 退出配置线路模式：S1(config-line)# exit
    • 退出至特权模式：S1(config)# exit
    • 测试控制台口令：如果系统提示输入密码，说明设置成功。

中文总结：执行退出命令回到特权模式，以验证之前设置的控制台密码是否已生效。

  • 如果交换机在登录时没有提示输入密码，说明在第二步中没有正确配置登录参数。

中文总结：如果没有密码提示，则说明登录参数配置不正确，需要重新检查第二步的配置。

注意事项 (Note)

  • 如果交换机在登录时没有提示输入密码，那么就意味着在步骤2中没有正确配置登录参数。

中文总结：这是一个重要的验证步骤，以确保交换机的安全设置按预期工作。

总结：本部分内容涉及基础的交换机配置，包括为交换机命名和保护控制台线路访问。详细说明了每个步骤中需要执行的命令，以及如何验证配置是否正确。此外，还强调了检查和验证步骤的重要性，确保配置的安全性。

Step 3: 保护特权模式访问 (Step 3: Secure privileged mode access)

  • 设置启用密码为c1$c0，该密码用于保护访问特权模式。
    • 注意：0是数字零，不是大写的O。
    • 进入特权模式：S1> enable
    • 进入配置模式设置启用密码：S1(config)# enable password c1$c0
    • 退出配置模式：S1(config)# exit

中文总结：第三步骤是设置进入特权模式的密码，确保只有授权用户才能访问交换机的高级设置。

Step 4: 验证特权模式访问是否安全 (Step 4: Verify that privileged mode access is secure)

  • 执行退出命令，退出交换机。
  • 按键，并在提示时输入密码，验证特权模式密码是否设置成功。
  • 第一个密码是用于登录的，第二个密码是进入特权模式时需要的。
  • 使用show running-config命令来验证配置。

中文总结：第四步骤涉及验证是否已成功设置特权模式密码，以保护交换机的特权模式不被未授权访问。

Step 5: 配置加密密码以保护特权模式 (Step 5: Configure an encrypted password to secure access to privileged mode)

  • 使用enable secret命令来替换之前的启用密码，以加强安全性。
    • 进入特权模式：S1> enable
    • 进入配置模式设置加密的启用密码：S1(config)# enable secret itsasecret
    • 退出配置模式：S1(config)# exit

中文总结：第五步骤是将之前设置的密码更换为加密密码，提升了特权模式访问的安全级别。

总结：以上步骤详细介绍了如何为交换机设置特权模式密码，以及如何验证和加密这些密码以确保安全。通过这些步骤，可以确保只有授权人员才能访问和修改交换机的特权模式，从而保护网络的安全性。

Step 6: 验证启用密钥是否添加到配置文件 (Step 6: Verify that the enable secret password is added to the configuration file)

  • 使用show running-config命令来验证新配置的启用密钥。
    • 可以使用缩写命令：S1# show run
    • 检查启用密钥enable secret如何显示：它应以加密形式出现。

中文总结：第六步骤涉及验证启用密钥是否已经加入配置文件，并且其显示方式应为加密形式，以确保配置的安全性。

Step 7: 加密启用和控制台密码 (Step 7: Encrypt the enable and console passwords)

  • 之前步骤中虽然enable secret密码已加密，但启用和控制台密码仍是明文。现在将使用service password-encryption命令加密这些密码。
    • 进入配置模式并启用密码加密服务：S1# config t → S1(config)# service password-encryption
    • 退出配置模式：S1(config)# exit

中文总结：第七步骤是加密交换机中的启用和控制台密码，确保所有的密码都不以明文形式存储，提高安全性。

  • 如果在交换机上配置更多密码，它们将被自动加密，不再以明文显示。

中文总结：启用密码加密服务后，所有当前和将来设置的密码都将自动加密，增强了安全防护。

总结：以上步骤详细说明了如何验证启用密钥是否正确添加到配置文件，并介绍了如何使用服务命令来加密现有和未来的密码。这些步骤保证了网络设备的密码不会以明文形式存储，有效防止了密码泄露的风险。

Part 3: 配置MOTD横幅 (Part 3: Configure a MOTD Banner)

Step 1: 配置每日消息 (Step 1: Configure a message of the day (MOTD) banner)

  • 使用Cisco IOS命令设置MOTD，为任何登录交换机的用户显示消息。
    • 进入配置模式：S1(config)# banner motd "This is a secure system. Authorized Access Only!"
    • 退出配置模式：S1(config)# exit

中文总结：步骤一是配置登录时显示的每日消息横幅，用于提示或警告用户关于系统安全和访问权限的信息。

  • 横幅将在哪种情况下显示？当有人通过控制台端口进入交换机时显示。

中文总结：MOTD横幅在用户通过控制台登录时显示，起到提醒和警告的作用。

  • 为什么每个交换机都应该有MOTD横幅？横幅用于警告未授权用户，同时也可以传递给网络人员/技术员重要信息。

中文总结：MOTD横幅的目的是为了提醒用户注意安全，防止未经授权的访问，并向网络管理员传达紧急信息。

Part 4: 保存配置文件到NVRAM (Part 4: Save Configuration Files to NVRAM)

Step 1: 验证配置的准确性 (Step 1: Verify that the configuration is accurate using the show run command)

  • 使用show run命令验证配置是否正确。

Step 2: 保存配置文件 (Step 2: Save the configuration file)

  • 完成基础配置后，需要将运行配置文件备份到NVRAM以确保在重启或断电后配置不会丢失。
    • 保存配置文件到启动配置：S1# copy running-config startup-config

中文总结：步骤二是将当前的配置保存到NVRAM，确保配置的持久性。

  • 保存命令的最短缩写是什么？cop r s

Step 3: 检查启动配置文件 (Step 3: Examine the startup configuration file)

  • 使用show startup-config命令查看NVRAM中的内容。
  • 验证是否所有更改都记录在文件中？是的，它与运行配置相同。

中文总结：步骤三是查看和确认启动配置文件，确保所有更改都已经被正确保存。

总结：第三部分和第四部分介绍了如何为交换机配置MOTD横幅和保存配置文件到NVRAM。这些步骤确保了交换机在用户登录时显示适当的安全提示信息，并且在系统重启或断电后保持配置不变，增加了系统的稳定性和安全性。

Part 5: 配置S2交换机 (Part 5: Configure S2)

a. 为交换机命名

  • 设置交换机的主机名为S2：Switch(config)# hostname S2

b. 保护控制台访问

  • 使用密码letmein来保护控制台访问：S2(config-line)# password letmein

c. 配置启用密码和启用密钥

  • 设置启用密码为c1$c0和启用密钥为itsasecret：S2(config)# enable password c1$c0 和 S2(config)# enable secret itsasecret

d. 配置登录消息横幅

  • 设置MOTD横幅以警告未授权访问：S2(config)# banner motd # Authorized access only. Unauthorized access is prohibited and violators will be prosecuted to the full extent of the law. #

e. 加密所有明文密码

  • 使用服务命令加密所有明文密码：S2(config)# service password-encryption

f. 确认配置正确

  • 通过查看运行配置来确认配置正确：S2# show running-config

g. 保存配置文件

  • 以防电源断电，保存配置文件以免丢失：S2# copy running-config startup-config

中文总结：第五部分描述了如何对S2交换机进行配置，包括命名交换机、设置控制台密码、配置启用密码和启用密钥、配置MOTD横幅、加密所有明文密码、确认配置正确性以及保存配置文件以确保配置在电源断电时不丢失。这些步骤确保了交换机的安全性和配置的持久性。

总结：本部分提供了关于如何为S2交换机进行完整配置的详细步骤，从基础设置到安全加固，再到配置的持久化。每个步骤都强调了其在整个配置过程中的重要性和作用，以确保网络设备的稳定和安全运行。

Part 6: 配置个人计算机 (Part 6: Configure the PCs)

Step 1: 为两台电脑配置IP地址 (Step 1: Configure both PCs with IP addresses)

  • 点击PC1，然后选择桌面选项卡。
  • 点击IP配置，根据地址表输入PC1的IP地址为192.168.1.1，子网掩码为255.255.255.0。
  • 重复步骤a和b为PC2配置IP地址。

中文总结：第一步涉及为两台电脑配置IP地址和子网掩码，使它们能在网络上进行通信。

Step 2: 测试与交换机的连通性 (Step 2: Test connectivity to switches)

  • 点击PC1，关闭IP配置窗口，如果它仍然打开的话，在桌面选项卡中点击命令提示符。
  • 输入ping命令和PC2的IP地址，按回车：ping 192.168.1.2。

中文总结：第二步是测试配置的IP地址是否能够成功连接到网络中的其他设备，这是确保网络通信正常的重要步骤。

总结：第六部分指导了如何为两台个人计算机配置IP地址并测试它们与网络中其他设备的连通性。这确保了网络中的每台设备都能够相互通信，为网络的正常运作提供了基础。
