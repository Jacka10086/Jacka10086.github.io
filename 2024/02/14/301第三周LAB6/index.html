实验6：连接路由器和局域网

拓扑结构

  • 描述了包括两个路由器（R1和R2）以及四台PC（PC1到PC4）通过四个交换机（S1到S4）连接的网络拓扑结构。
  • 路由器之间通过序列连接（DCE）互连，并且分别连接到两个不同的局域网。

地址表

  • 列出了实验中所有设备的接口、IP地址、子网掩码和默认网关配置。
  • 例如，R1的G0/0接口配置了IP地址192.168.10.1，子网掩码为255.255.255.0，没有默认网关。

引言

  • 本实验中，您将使用各种show命令来显示路由器的当前状态。
  • 实验指导您通过使用配置路由器以太网接口的命令来验证和测试您的配置。
  • 注意，实验中的路由器部分配置可能未完成，附带的地址表用于辅助验证。
  • R1和R2之间的序列接口已使用EIGRP配置了WAN连接。EIGRP是一种路由协议。

总结

本实验详细介绍了一个多路由器和多局域网的网络拓扑结构，并提供了详细的地址表以供配置参考。本实验的目的是通过使用路由器的show命令来验证和测试网络配置，确保数据能够在不同的局域网之间正确路由。此外，实验强调了在两个路由器之间已经配置了EIGRP协议的重要性，这是一种用于自动路由决策的动态路由协议。

第一部分：显示路由器信息

步骤1：在R1上显示接口信息

  • 注意：点击设备后点击CLI标签以直接访问命令行接口。控制台密码是cisco，特权EXEC密码是class。
    • a. show interfaces命令显示了路由器上所有配置的接口信息。
    • b. 使用show interface serial 0/0/0命令显示序列0/0/0接口的信息。
    • c. 输入命令查看R1上的序列0/0/0接口统计信息，并回答以下问题：
      1. R1上序列0/0/0接口的IP地址是209.165.200.225/30。
      2. 该接口的带宽是1544 kbits/s。
    • d. 显示GigabitEthernet 0/0接口的统计信息，并回答以下问题：
      1. R1上GigabitEthernet 0/0接口没有配置IP地址。
      2. 该接口的MAC地址是0060.3E1C.D633。
      3. 该接口的带宽是1000000 kbits/s。

步骤2：在R1上显示接口列表摘要

  • a. 使用show ip interface brief命令，只显示当前接口状态的简要总结。
  • b. 在每台路由器上执行命令，并回答以下问题：
    1. R1和R2上有多少个序列接口？每台路由器有2个序列接口。
    2. R1和R2上有多少个以太网接口？R1有6个，R2有2个。
    3. R1上所有以太网接口是否相同？如果不同，请解释差异：不同，因为有Gigabit Ethernet接口和Fast Ethernet接口，Gigabit Ethernet接口支持最高1,000,000,000比特速率，而Fast Ethernet接口支持最高1,000,000比特速率。

步骤3：在R1上显示路由表

  • a. 使用show ip route命令显示路由表内容。
  • b. 在R1上执行命令，并回答以下问题：
    1. 当路由表中使用C代码时，应查找哪一条路由？直连路由。
    2. 哪条路由是指向209.165.200.224/30的？这是通过直连接口的路由。

总结

本部分介绍了如何在Cisco路由器上显示接口信息和路由表。步骤1涉及查看和理解各接口的详细统计信息。步骤2使用了一个简化的命令来快速查看接口状态。步骤3则聚焦于如何查看和解析路由表信息，特别是直连路由。这些步骤对于网络管理员来说是日常工作的重要部分，它们帮助管理员监控和故障排除网络设备。

第二部分：配置路由器接口（Part 2: Configure Router Interfaces）

步骤1：在R1上配置GigabitEthernet 0/0接口（Step 1: Configure the GigabitEthernet 0/0 interface on R1）

  • a. 输入以下命令来配置并激活R1的GigabitEthernet 0/0接口：
    • interface gigabitethernet 0/0：进入接口配置模式。
    • ip address 192.168.10.1 255.255.255.0：为接口分配IP地址及子网掩码。
    • no shutdown：启用接口，使其在线。
  • b. 为接口配置描述，帮助记录网络信息：
    • description LAN connection to S1：为接口设置描述，指明其连接到S1交换机。
  • c. 完成配置后，R1应能够ping通PC1：
    • 使用命令end退出到主配置模式，然后使用ping 192.168.10.1测试连通性。

步骤2：配置R1和R2的其余Gigabit Ethernet接口（Step 2: Configure the remaining Gigabit Ethernet Interfaces on R1 and R2）

  • 根据地址表信息完成R1和R2的接口配置：
    • 为每个接口分配IP地址并激活。
    • 为每个接口配置适当的描述。
    • 验证接口配置是否正确。

总结

本节内容介绍了如何在路由器上配置GigabitEthernet接口，包括分配IP地址、子网掩码，并通过no shutdown命令激活接口。同时，还强调了为接口配置描述的重要性，它能够帮助记录和标识网络连接的详细信息。步骤1展示了具体的配置命令和测试连通性的过程，而步骤2则提示了完成所有接口配置的重要步骤。

第三部分：验证配置（Part 3: Verify the Configuration）

步骤1：使用验证命令检查接口配置（Step 1: Use verification commands to check your interface configurations）

  • a. 在R1和R2上使用show ip interface brief命令快速验证接口配置正确且已激活。
    • 查询有多少个接口配置了IP地址并处于“up”状态。
    • 该命令输出中未显示子网掩码部分。

步骤2：测试网络的端到端连通性（Step 2: Test end-to-end connectivity across the network）

  • 应能从任何PC ping通网络上的其他PC。
  • 例如，从PC1尝试ping PC4可能会失败，因为R2的接口未配置到10.1.2.0/24网络。

进一步工作（Further work）

  • 根据上述地址表继续配置路由器的接口，并使用ping命令验证连通性。
  • 执行show ip route命令，在每个路由器上查看直连网络的数量以及路由表中的路由数量。
  • 如果有疑问，咨询实验指导老师。

总结

本部分内容指导如何验证路由器接口的配置是否正确，并测试网络的端到端连通性。步骤1中使用的show ip interface brief命令能够快速显示接口的状态和配置的IP地址，但不显示子网掩码信息。步骤2中强调了进行实际通信测试的重要性，以确保网络各部分可以正确通信。进一步工作提示了如何继续配置和验证网络设置，并推荐在有疑问时寻求实验指导老师的帮助。
