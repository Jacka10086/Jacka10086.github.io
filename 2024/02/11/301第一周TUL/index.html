操作系统(Operating Systems)

操作系统结构(Operating System Structure)

  • 硬件(Hardware): 包括底层电子元件。
  • 内核(Kernel): 负责硬件和软件之间的通信，并管理硬件资源满足软件需求。
  • 壳(Shell): 提供用户界面，允许用户请求特定任务。

操作系统由多层构成，最底层是硬件，内核位于中间层，负责管理资源，最外层是用户界面，包括命令行界面(CLI)和图形用户界面(GUI)。

操作系统的目的(Purpose of OS)

  • 个人电脑操作系统允许用户：
    • 使用鼠标选择和运行程序。
    • 输入文本和文本命令。
    • 在显示器上查看输出。
  • 网络操作系统（IOS）允许网络技术人员：
    • 使用键盘运行基于CLI的网络程序。
    • 使用键盘输入文本和文本命令。
    • 在显示器上查看输出。

操作系统使用户可以与计算机或网络设备交互，无论是通过PC操作系统还是通过网络设备的IOS，都提供了用户与系统交互的界面。

操作系统的升级(Operating System Upgrades)

  • 所有网络设备都附带默认的IOS。
  • 可以升级IOS版本或功能集。

网络设备的操作系统（如Cisco的IOS）可以根据需要进行升级，以获得新功能或改进性能。

总结(Summary)

操作系统是用户与计算机硬件之间交互的桥梁，它由多个组件构成，包括内核、壳，以及用户接口。操作系统不仅控制硬件资源，还提供了用户界面，使用户能够执行任务和配置设置。

操作系统的多层结构和功能使得用户能够有效地与计算机或网络设备进行交互和管理。

访问方法(Access Methods)

控制台(Console)

  • 使用控制台端口的优势在于，即使没有配置网络服务，设备也是可访问的，例如在执行网络设备的初始配置时。
  • 在执行初始配置时，运行终端仿真软件的计算机通过特殊电缆连接到设备的控制台端口。
  • 用于设置交换机或路由器的配置命令可以在连接的计算机上输入。

控制台访问是在设备初始设置或没有网络服务时进行配置的基本方式，通过物理连接实现。

Telnet

  • 最佳实践是使用SSH代替Telnet进行远程管理CLI连接。
  • Cisco IOS包括一个Telnet服务器和一个Telnet客户端，可以用来与其他设备建立Telnet会话。

Telnet是一种较老的网络协议，用于远程访问，但它不加密，因此不推荐用于安全环境。

SSH

  • SSH是远程管理的推荐方法，因为它提供了安全连接。
  • SSH提供加密的密码认证和会话数据的传输，确保用户ID、密码和管理会话的细节保持私密。
  • 大多数版本的Cisco IOS都包含一个SSH服务器和一个SSH客户端，可以用来与其他设备建立SSH会话。

SSH是现代网络管理的标准，提供了安全的远程访问方法，确保通信加密和身份验证。

总结(Summary)

访问方法包括物理访问（如控制台端口）和网络访问（如Telnet和SSH）。控制台访问用于没有网络连接的情况，而SSH是安全的远程网络访问推荐方法，Telnet因安全性问题而不推荐使用。

不同的访问方法适用于不同的场景，选择正确的访问方法对于确保网络设备安全和效率至关重要。

Cisco IOS 操作模式(Cisco IOS Modes of Operation)

  • 在开始配置Cisco设备之前，必须建立控制台连接。
  • 控制台连接后，网络技术人员需要通过IOS CLI的不同命令模式进行导航。
  • Cisco IOS模式使用分层结构，并且对于交换机和路由器来说非常相似。

进行设备配置时首先需要通过控制台端口与设备建立物理连接。一旦连接，技术人员可以通过命令行界面(CLI)在不同的模式之间导航，这些模式以分层结构组织，无论是配置交换机还是路由器操作都是类似的。

终端仿真程序(Terminal Emulation Programs)

Tera Term

  • Tera Term 是一个终端仿真程序，用于通过串行端口或网络连接到远程系统。
  • 该程序允许用户通过SSH、Telnet等协议进行网络通信。
  • 用户可以指定主机地址、端口和所需使用的服务协议。

终端仿真程序如Tera Term是网络管理员用于配置和管理网络设备的重要工具，它提供了一个用户界面，允许以安全的方式远程访问和控制网络设备。

主要命令模式(Primary Command Modes)

用户执行模式(User EXEC Mode)

  • 允许对路由器进行有限的检查。
  • 远程访问。
  • 仅允许基本监控命令，通常被称为查看模式。

用户执行模式提供基本的命令访问权限，通常用于查看配置和设备状态，但不允许进行配置更改。

特权执行模式(Privileged EXEC Mode)

  • 提供对路由器的详细检查。
  • 调试和测试。
  • 文件操作。
  • 远程访问。

特权执行模式提供更高级别的访问，允许执行监控、配置和管理命令。

配置命令模式(Configuration Command Modes)

全局配置模式(Global Configuration Mode)

  • 必须使用 configure terminal 命令进入此模式以配置设备。
  • 例如：Switch(config)#
  • CLI配置更改会影响整个设备的运作。
  • 从此模式，用户可以进入不同的子配置模式。

全局配置模式用于进行设备级别的配置更改，例如设置设备名称、密码等。

常用子配置模式(Common Sub-configuration Modes)

  • 行配置模式(Line Configuration Mode) - 用于配置控制台、SSH、Telnet或AUX访问。例如：Switch(config-line)#
  • 接口配置模式(Interface Configuration Mode) - 用于配置交换机端口或路由器网络接口。例如：Switch(config-if)#

子配置模式允许对特定的接口或线路进行详细配置，例如设置特定端口的安全性或协议设置。

总结(Summary)

Cisco设备的操作模式分为不同层级，每个层级提供了不同级别的配置和控制选项。用户执行模式用于基本监控，特权执行模式提供了完整的检查和配置功能，全局配置模式和子配置模式则用于特定的配置任务。

理解不同的命令模式对于有效地使用Cisco设备至关重要，每种模式都根据访问级别和配置类型设计。

在IOS模式之间导航(Navigate Between IOS Modes)

基本命令(Basic Commands)

  • enable 命令从用户执行模式(User EXEC Mode)进入特权执行模式(Privileged EXEC Mode)。
  • disable 命令从特权执行模式返回到用户执行模式。
  • exit 命令用于退出当前模式或结束会话。

配置命令(Configuration Commands)

  • configure terminal 进入全局配置模式。
  • interface vlan 1 从全局配置模式进入特定接口的配置模式。
  • exit 从任何配置子模式返回到上一级模式。
  • end 或 Ctrl+Z 从任何配置模式返回到特权执行模式。

用户可以通过这些命令在不同的IOS模式之间导航，进行设备的配置和管理。

示例(Examples)

  • 切换到全局配置模式：Switch> enable 然后 Switch# configure terminal
  • 配置特定接口：Switch(config)# interface vlan 1，完成后使用 exit 返回。
  • 配置线路：Switch(config)# line vty 0 4 进入虚拟终端线路配置，使用 interface FastEthernet 0/1 配置特定接口。

这些命令示例展示了如何从一个模式进入另一个模式，并在不同模式之间执行配置任务。

总结(Summary)

Cisco IOS提供了一系列的命令模式用于不同级别的设备配置和管理。了解如何在这些模式之间导航对于有效管理Cisco网络设备至关重要。

正确地在各个IOS模式之间导航是进行设备配置和维护的基础，每个模式都有其特定的用途和命令集。

IOS 帮助功能(IOS Help Features)

上下文相关帮助(Context-Sensitive Help)

  • 输入 ? 可以显示命令选项，列出以特定字符开始的命令或关键字。
  • IOS显示接下来可以输入哪些命令参数或变量，并对每个选项提供解释。

示例(Examples)

  • 输入 cl? 显示所有以 cl 开头的命令。
  • 输入 clock set ? 显示 clock set 命令后可以跟随的参数。
  • 输入 clock set 19:50:00 ? 显示设置特定时间后的日期选项。

这些帮助功能使得用户可以在不完全记住命令的情况下进行操作，通过提供命令的可能选项和参数。

帮助消息(Help Messages)

  • 当命令不完整时，IOS返回一条帮助消息，表明需要的关键字或参数没有输入。
  • 当输入的命令含糊不清时，IOS返回一条帮助消息，表明输入的字符不足以识别命令。
  • 当输入无效时，IOS返回错误指示器，指出无法识别的命令部分。

总结(Summary)

Cisco IOS 提供了上下文相关的帮助功能，帮助用户理解和使用命令行界面。通过使用 ? 来获取命令提示，用户可以更容易地学习和使用Cisco设备的各种功能。

IOS的帮助功能是一个强大的工具，它通过提供命令和参数的即时提示来辅助用户配置和管理Cisco设备。

热键和快捷方式(Hotkeys and Shortcuts)

  • Tab 键 - 自动完成部分输入的命令或关键字。
  • Ctrl+R - 重新显示一行。
  • Ctrl+A - 将光标移动到行的开头。
  • Ctrl+Z - 退出配置模式并返回到用户EXEC模式。
  • 下箭头(Down Arrow) - 允许用户向前滚动以查看以前的命令。
  • 上箭头(Up Arrow) - 允许用户向后滚动以查看以前的命令。
  • Ctrl+Shift+6 - 允许用户中断如 ping 或 traceroute 的IOS进程。
  • Ctrl+C - 终止当前命令并退出配置模式。

这些快捷方式能提高网络管理员在使用Cisco设备时的效率，使他们能够快速导航和操作CLI。

设备名称(Device Names)

选择主机名的指南(Guidelines to Choose a Hostname)

  • 主机名应该以字母开头。
  • 不包含空格。
  • 以字母或数字结尾。
  • 仅使用字母、数字和破折号。
  • 长度应少于64个字符。

配置设备名称(Configuring Device Names)

  • 主机名允许网络管理员通过网络或互联网识别设备。
  • 例如：Switch(config)# hostname SW-Floor-1 设置交换机的主机名为 SW-Floor-1。

设备名称是识别和管理网络中设备的重要部分，正确的命名约定有助于维护网络的清晰结构和组织。

总结(Summary)

热键和快捷方式提高了CLI的导航和操作效率，而适当的设备命名策略有助于网络管理员更有效地识别和管理网络设备。

掌握这些热键和快捷方式对于高效使用Cisco IOS命令行界面至关重要，而合理的设备命名则有助于网络的日常维护和管理。

配置密码(Configure Passwords)

特权EXEC模式密码(Privileged EXEC Password Example)

  • 通过 enable secret [password] 命令设置加密的特权EXEC模式密码。
  • 示例：Switch(config)# enable secret class

用户EXEC模式密码(User EXEC Password Example)

  • 通过 line console 0 进入控制台行配置，并通过 password [password] 和 login 命令设置用户EXEC模式密码。
  • 示例：Switch(config)# line console 0 接着 Switch(config-line)# password cisco 和 Switch(config-line)# login

VTY线路密码(VTY Line Password Example)

  • VTY线路允许通过Telnet访问Cisco设备。
  • 通过 line vty 0 15 进入VTY线路配置，并通过 password [password] 和 login 命令设置VTY线路密码。
  • 示例：Switch(config)# line vty 0 15 接着 Switch(config-line)# password cisco 和 Switch(config-line)# login

控制台端口安全(Console Port Security)

  • 控制台端口必须安全，以降低未经授权人员物理连接设备并获得设备访问权的风险。

总结(Summary)

通过适当的命令设置不同的密码，可以对Cisco设备的不同访问级别进行保护。这确保了只有拥有正确密码的人员才能访问设备的特定模式，是网络安全的关键部分。

在Cisco设备中设置密码是确保网络安全的基本步骤，这些密码通过不同的命令在设备上设置，并可提供不同级别的访问控制。

加密密码(Encrypt Passwords)

加密服务命令(Service Command)

  • 使用 service password-encryption 命令来加密配置文件中的明文密码。
  • 这个命令的目的是为了防止未授权人员在查看配置文件时看到密码。
  • 应用此命令后，即使移除密码加密服务，也不会撤销加密。

配置示例(Configuration Example)

  • 输入命令 Switch(config)# service password-encryption。
  • 退出全局配置模式并查看运行配置 Switch(config)# exit 然后 Switch# show running-config。
  • 结果显示密码已被加密，不再以明文显示。

总结(Summary)

在Cisco设备上加密密码是一种安全措施，用于保护设备配置中的敏感信息不被轻易访问。密码一旦加密，即使服务被禁用，加密也不会被撤销。

正确使用 service password-encryption 命令对提高网络安全性至关重要，它确保了配置文件中的密码不会以明文形式暴露。
‘## 横幅消息(Banner Messages)

法律过程的重要部分(Important Part of Legal Process)

  • 横幅消息在追究非法闯入设备者的法律责任时非常重要。

措辞建议(Word Choice)

  • 横幅消息中使用表明欢迎或邀请登录的措辞是不恰当的。

法律通知(Legal Notification)

  • 由于横幅消息会显示给所有连接的终端，所以通常用于法律通知。

配置实例(Configuration Example)

  • 输入命令 Sw1-Floor-1(config)#banner motd # This is a secure system. Authorized Access ONLY!!! # 来设置消息横幅。
  • 配置结果是每天的消息横幅。

总结(Summary)

横幅消息用于向所有尝试连接网络设备的人显示重要通知。在法律层面上，这些消息可以帮助防止未授权访问并在追究责任时提供支持。

设定适当的横幅消息是网络安全和法律合规的一部分，可以告知用户关于访问权限的法律声明，避免使用欢迎性质的语言。

保存运行配置文件(Save the Running Configuration File)

查看当前活动配置(Viewing Current Active Configuration)

  • 命令 show running-config 用于列出当前在RAM中活动的完整配置。

启动配置(Startup Configuration)

  • 存储在NVRAM中的文件，包含设备启动或重启时将会用到的所有命令。NVRAM在设备断电时不会丢失内容。

运行配置(Running Configuration)

  • 存储在RAM中的文件，反映了当前配置，修改会立即影响Cisco设备的操作。RAM在设备断电或重启时会丢失所有内容。

保存配置(Saving Configuration)

  • 通过命令 copy running-config startup-config，可以将活动配置复制到NVRAM。

了解这些配置文件的存储位置和它们的作用至关重要。运行配置是当前在设备上执行的配置，而启动配置是设备启动时将要加载的配置。正确地保存当前的运行配置至启动配置，可以确保在设备重启后所需的配置不会丢失。
