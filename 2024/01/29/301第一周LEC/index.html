The Internet: a “nuts and bolts” view（互联网：一个“螺丝钉”视角）

互联网由数十亿连接的计算设备组成：

  • hosts = end systems（主机=终端系统）
    • 在互联网的“边缘”运行网络应用程序

Packet switches（数据包交换设备）:

  • forward packets (chunks of data)（转发数据包（数据块））
    • routers, switches（路由器，交换机）

Communication links（通信链路）

  • fiber, copper, radio, satellite（光纤，铜线，无线电，卫星）
  • transmission rate: bandwidth（传输速率：带宽）

Networks（网络）

  • collection of devices, routers, links: managed by an organization（设备、路由器、链路的集合：由组织管理）

图示解释了互联网结构的基本组成部分，包括各种计算设备、数据包交换设备、通信链路和各类网络。这些元素共同构成了全球互联网的基础架构。主机或终端系统运行网络应用，通过路由器和交换机进行数据传输。通信链路则通过多种介质实现数据的物理传输，带宽决定了传输速率。最终，这些设备和链路形成了网络，由不同的组织进行管理，确保数据的有效传输。

Overview of Network Components（网络组件概览）

网络由多个重要组件构成，包括但不限于以下几类：

Devices（设备）

  • 包括个人计算设备，如电脑、手机和其他可以连接到局域网（LAN）的设备。

Internetwork（互联网络）

  • 设备通过路由器连接形成更大的网络，允许不同局域网的设备相互通信。

Media（媒介）

  • 网络设备通过各种传输媒介（如有线或无线）连接到互联网络。

Services（服务）

  • 网络提供多种服务，包括软件应用和各类网络过程与规则，如各种协议和配置规则。

总结：

这些幻灯片展示了网络组件的概览，强调了设备、互联网络、传输媒介以及网络服务之间的关系。设备是网络中用户直接交互的硬件，如计算机和移动设备。这些设备通过各种媒介连接到局域网，并通过路由器连接到更广泛的互联网络。最终，网络上运行的服务保障了数据的正确传输和处理。这些组件共同工作，形成了我们日常使用的网络环境。

The Internet: a “nuts and bolts” view（互联网：一个“螺丝钉”视角）

互联网被形象地描述为一个“网络中的网络”：

  • Internet: “network of networks”（互联网：网络中的网络）
    • Interconnected ISPs（相互连接的互联网服务提供商）

Protocols are everywhere（协议无处不在）

  • 控制消息的发送和接收
    • 例如：HTTP（Web），流媒体视频，Skype，TCP，IP，WiFi，4G，以太网

Internet standards（互联网标准）

  • RFC: Request for Comments（RFC：请求评论）
  • IETF: Internet Engineering Task Force（IETF：互联网工程任务组）

图示展示了不同类型的网络如何通过互联网服务提供商（ISP）连接在一起，以及如何通过各种协议（如HTTP，TCP，IP等）进行通信。此外，还指出了制定互联网标准的机构，如IETF，他们发布RFC文档以征求对新标准的评论和建议。这些标准和协议确保了不同网络设备之间能够有效地通信和协作。

总结：

这部分内容阐释了互联网的基本结构和工作原理。互联网由多个ISP构成的网络组成，这些网络通过遵守共同的协议来实现互联。这些协议涵盖了从物理层的以太网和WiFi到应用层的HTTP和流媒体等各个层面。互联网标准由像IETF这样的组织制定，确保全球网络的互操作性和技术的一致性。

The Internet: a “service” view（互联网：一个“服务”视角）

互联网不仅是一个由多个网络组成的基础设施，它还提供了多种服务给应用程序：

  • Infrastructure that provides services to applications（为应用程序提供服务的基础设施）:

    • 包括网页浏览、流媒体视频、多媒体会议、电子邮件、游戏、电子商务、社交媒体、互联家电等。

  • Provides programming interface to distributed applications（为分布式应用程序提供编程接口）:

    • “hooks” allowing sending/receiving apps to “connect” to, use Internet transport service（“钩子”，允许发送/接收应用程序“连接”到互联网传输服务）
    • 提供服务选项，类似于邮政服务

图示中展示了互联网如何通过服务提供商连接不同的家庭网络、企业网络以及数据中心网络，以及如何通过协议（如HTTP、Skype）提供服务。互联网作为一个服务平台，允许各种应用程序通过编程接口与之交互，使用其传输服务来进行数据的发送和接收，类似于邮政系统提供不同的邮寄选项。

总结：

本节内容从服务的角度对互联网进行了解读，强调了互联网作为基础设施为各种应用提供服务的能力。这些服务包括传统的网络应用如电子邮件和网页浏览，以及现代应用如流媒体和社交媒体。互联网还提供了编程接口，使得分布式应用能够高效地利用其传输服务。这种服务模型使得互联网成为了一个强大的平台，支持着现代数字化生活的各个方面。

Types of Networks（网络类型）

网络根据其覆盖范围和功能被分为不同的类型：

  • The two most common types of network infrastructures are（两种最常见的网络基础设施类型是）:
    • Local Area Network (LAN)（局域网）
    • Wide Area Network (WAN)（广域网）

Other types of networks include（其他网络类型包括）:

  • Metropolitan Area Network (MAN)（都会区域网）
  • Wireless LAN (WLAN)（无线局域网）
  • Storage Area Network (SAN)（存储区域网）

图示中展示了这些不同类型的网络的连接方式。局域网通常是在较小的地理区域内，如家庭办公室或企业中心，提供网络连接。广域网则连接更广泛的地理区域，如分支机构。其他类型的网络，如都会区域网，提供城市或大都市区域的网络连接，无线局域网提供无线连接到局域网的方式，而存储区域网则专注于连接数据存储设施。

总结：

本节内容介绍了不同类型的网络基础设施。局域网和广域网是两种基本类型，它们根据不同的需求和应用场景构建。都会区域网、无线局域网和存储区域网则是特定用途的网络类型，它们在现代网络架构中也扮演着重要角色，以适应不同的通信和存储需求。

Local Area Networks vs. Wide Area Networks（局域网与广域网）

Local Area Networks（局域网）

  • 局域网通常指在较小的地理区域（如办公室、学校或家庭）内的计算机网络。

Wide Area Networks（广域网）

  • 广域网是指连接距离较远、地理位置分散的局域网的网络，这种连接通常被称为WAN。

图示对比了局域网和广域网的结构。局域网由多个设备（如个人电脑、打印机等）通过交换机连接形成一个网络。而广域网则由多个这样的局域网通过路由器连接，以跨越更广的地理区域。WAN能够连接不同建筑物、城市甚至国家的网络，允许这些局域网之间的通信。

总结：

局域网和广域网是网络基础设施的两个基本类型，它们根据连接的地理范围和网络的规模进行分类。局域网用于较小的区域，而广域网则用于连接分散在较大地理范围内的网络。这两种网络类型对于构建覆盖从单个位置到全球范围内的通信网络都是必不可少的。

The Internet（互联网）

互联网通过广域网服务实现不同局域网之间的互联。

  • LANs use WAN services to interconnect（局域网使用广域网服务进行互联）:
    • 家庭局域网（Home LAN）
    • 分支局域网（Branch LAN）
    • 政府局域网（Government LAN）
    • 学校局域网（School LAN）
    • 企业局域网（Enterprise LAN）

图示展示了多个不同类型的局域网如何通过广域网服务连接起来，形成互联网。这些连接可以跨越不同的建筑物、城市甚至国家，允许在全球范围内的数据和信息交换。

总结：

这张幻灯片描述了互联网的结构，它是由多个局域网通过广域网服务相互连接而成的。这种结构使得无论是家庭用户、教育机构、企业还是政府机构，都能在全球范围内进行通信和数据交换，突显了互联网作为全球通信基础设施的核心作用。

Communication Fundamentals（通信基础）

在探讨通信的基础概念时，可以将其分为人类通信和计算机通信两大类：

Generic Communication Model（通用通信模型）:

  • 消息来源（Message Source）
  • 发送器（Transmitter）
  • 传输媒介（Transmission Medium）
  • 接收器（Receiver）
  • 消息目的地（Message Destination）

图示对比了人类通信和计算机通信。在人类通信中，信息的发送和接收通常是通过语言和肢体语言实现的。而在计算机通信中，消息从源头经过一个发送器（比如网络接口卡），通过某种传输媒介（比如有线或无线信号）传输，最终被接收器（另一台计算机的网络接口）接收，并送达至消息的最终目的地（通常是另一台计算机或网络设备）。

总结：

本幻灯片简洁地阐释了通信的基本组件，无论是人类之间的通信还是计算机之间的通信。它展示了消息是如何从一个源点传输到一个目的地的，无论是通过人类的交流方式还是通过计算机网络。这种模型是理解网络如何工作的基础，也是设计和实施有效通信策略的关键。

What’s a protocol?（什么是协议？）

协议是通信中必须遵守的规则和格式，无论是人类之间的交流还是计算机网络中的数据交换。

Human protocols（人类协议）:

  • “what’s the time?”（现在几点了？）
  • “I have a question”（我有个问题）
  • introductions（介绍）

这些是在特定情况下发送的特定消息，以及在收到消息或其他事件发生时采取的特定行动。

Network protocols（网络协议）:

  • computers (devices) rather than humans（计算机（设备）而非人类）
  • all communication activity in Internet governed by protocols（互联网中的所有通信活动都受协议的管理）

协议定义了网络实体间发送和接收的消息的格式、顺序，以及在消息传输、接收时采取的行动。

总结：

本页内容介绍了协议在人类交流和计算机网络中的角色。人类协议涉及日常交流中的惯例，而网络协议是计算机网络中确保信息正确传输和接收的一套标准化规则。这些规则定义了数据包的格式、发送和接收的顺序，以及在数据传输过程中所执行的操作。协议是网络通信能够顺畅进行的基础。

What’s a protocol?（什么是协议？）

协议是两个或多个通信实体之间交换消息的规则集合。这里比较了人类协议和计算机网络协议：

A human protocol and a computer network protocol（人类协议和计算机网络协议）:

Human Protocol（人类协议）:

  1. Person A says “Hi”
  2. Person B responds with “Hi”
  3. Person A asks “Got the time?”
  4. Person B replies with “2:00”

Network Protocol（网络协议）:

  1. Computer sends a TCP connection request to a server
  2. Server sends a TCP connection response to the computer
  3. Computer sends a GET request to the server for a webpage (http://gaia.cs.umass.edu/kurose_ross)
  4. Server sends the requested file to the computer

这个过程展示了在人类协议中，通信往往是基于礼貌和社交习惯，而计算机网络协议则是基于严格定义的格式和规则来交换数据。

Q: other human protocols?（其他人类协议？）

  • 这个问题暗示读者可以考虑其他的人类交流协议，如握手、告别时说“再见”等。

总结：

此幻灯片阐明了协议在人类交流和计算机网络中的作用。在人类交流中，协议通常是基于文化和社会习俗的非正式规则；在计算机网络中，协议则是确保数据准确无误传输的一系列具有特定格式和顺序的正式规则。理解这些基础是理解更复杂网络交互的关键。

Rule Establishment（规则建立）

在网络通信中，协议定义了数据传输的基本规则，包括以下几个关键方面：

  • Message Encoding（消息编码）

    • 定义消息如何转换成信号以便在网络中传输。

  • Message Formatting and Encapsulation（消息格式化和封装）

    • 规定了消息的结构以及如何将数据打包以进行网络传输。

  • Message Size（消息大小）

    • 确定消息或数据块的大小限制。

  • Message Timing（消息时序）

    • 涉及数据传输的时序问题，包括速率、同步等。

  • Message Delivery Options（消息传递选项）

    • 包括消息的传递方式，如单播、广播或多播。

协议作为通信的基础，确保了网络中的设备能够以一致的方式交换信息。

总结：

此幻灯片强调了在网络通信中建立规则的重要性。协议不仅仅是通信中的一组规则，它们是确保信息能够被正确编码、格式化、封装、定时和传递的基础。这些元素共同工作，形成了网络通信的基础框架，是实现有效和高效通信的关键。

Message Formatting and Encapsulation（消息格式化和封装）

消息格式化和封装是通信中将信息准备为可传输形式的过程。以下是个人信件和计算机网络消息封装的比较：

个人信件的元素包括：

  • 收件人标识符（An identifier of the recipient）
  • 问候语或称呼（A salutation or greeting）
  • 消息内容（The message content）
  • 结束语（A closing phrase）
  • 发件人标识符（An identifier of the sender）

计算机网络消息封装：

  • 目的地（Destination）- 物理/硬件地址
  • 来源（Source）- 物理/硬件地址
  • 开始标志（Start Flag）- 消息开始指示符
  • 收件人（Recipient）- 目的地标识符
  • 发件人（Sender）- 来源标识符
  • 封装数据（Encapsulated Data）- 数据内容（比特）
  • 结束帧（End of Frame）- 消息结束指示符

图中展示了一个个人信件的地址封装过程和计算机网络消息封装过程。在信件的例子中，我们可以看到信封上的地址标注和信件内容的布局。在计算机网络消息的例子中，我们看到了数据包的各个部分，包括帧的地址和数据本身。

总结：

这张幻灯片说明了消息在发送之前如何被组织和封装。无论是在传统的邮政系统中还是在计算机网络中，封装都是确保信息能够到达正确目的地并能够被正确理解的关键步骤。在网络通信中，封装包括了添加源地址、目的地址以及其他控制信息，这样接收方就可以正确地解析并处理消息。

Message Size（消息大小）

消息大小在人类交流和计算机通信中有着不同的应用和重要性。

Human Communication（人类交流）

  • 在人类交流中，我们可以连续不断地说话，无需担心消息的大小或长度。

Computer Communication（计算机通信）

  • 计算机通信中，源主机将长消息分解为满足最小和最大大小要求的多个片段或帧。
  • 每个帧还将拥有自己的寻址信息。
  • 在接收主机，这些片段被重建以便处理和解释。

图示显示了人类交流中的信息传递（例如，一个人对另一个人评论日落的美丽），而在计算机通信中，一个复杂的消息（如文件或数据流）需要被拆分成多个数据包，这些数据包在网络中传输，然后在目的地重新组装成原始消息。

总结：

此幻灯片展示了消息大小在不同类型的通信中的重要性。在人类交流中，信息的大小不是问题，但在计算机通信中，大型信息必须被分割成小块，以便它们可以在网络中有效传输。这是网络设计中一个关键的考虑因素，确保了网络的有效性和可靠性。

Message Timing（消息时序）

消息时序涉及通信过程中的时间规则，包括：

Rules of engagement（参与规则）:

  • Access Method（访问方法）
  • Flow Control（流量控制）
  • Response Timeout（响应超时）

图示中的人类交流展示了两个人问问题，等待对方的回复。这个过程暗示了消息时序的概念，例如一个人问问题后期望在一定时间内得到回答，而不是无限期地等待。

总结：

此幻灯片指出了在任何通信形式中，无论是人类之间还是计算机网络内，都需要遵循特定的时序规则。在计算机网络中，这些规则确保数据的有效流动，防止拥塞，并处理没有按时接收到响应的情况。在人类交流中，这些规则帮助我们维持对话的流畅和有序。

Protocol “layers” and reference models（协议“层”与参考模型）

网络的结构非常复杂，它由多个“部分”组成：

  • hosts（主机）
  • routers（路由器）
  • links of various media（不同媒介的连接）
  • applications（应用程序）
  • protocols（协议）
  • hardware, software（硬件，软件）

Question（问题）:

  • is there any hope of organizing structure of network?（有没有可能组织网络结构？）
  • …. or at least our discussion of networks?（或者至少是我们对网络的讨论？）

这张幻灯片提出了一个问题，即在网络的复杂性中是否有可能找到一种组织结构，或者至少对网络进行结构化的讨论。这通常通过引入层次化的协议和参考模型来实现，如著名的OSI七层模型和TCP/IP模型，这些模型将网络通信分解为不同的层，每一层负责不同的通信任务和功能。

总结：

网络的复杂性可以通过层次化的协议模型来管理和理解。这些模型为设计、讨论和理解网络通信提供了一个结构化的方法，使得网络的不同“部分”可以在一个统一的框架下工作，并为网络的研究、教育和工程实践提供了基础。

Example: organization of air travel（例子：航空旅行的组织）

航空旅行的组织可以通过层级服务的概念来理解，每一层都提供特定的服务，并依赖于其下一层的服务。

Layers（层）:

  • 每一层实现一个服务（each layer implements a service）
    • 通过其内部层的行动（via its own internal-layer actions）
    • 依赖于下面一层提供的服务（relying on services provided by layer below）

各层服务的描述（Describe in words the service provided in each layer above）:

  • 票务（购买、投诉）- 乘客购买机票和提出服务相关的投诉。
  • 行李（托运、领取）- 乘客的行李进行托运和在目的地领取。
  • 登机口（登机、卸载）- 乘客在登机口登机和飞机在登机口卸载乘客。
  • 跑道（起飞、着陆）- 飞机从跑道起飞和在跑道着陆。
  • 飞机路线 - 飞机在空中的路线规划。

总结：

这张幻灯片通过航空旅行的例子阐释了层级服务模型的概念。在航空旅行中，每个服务层都有明确的职责，从乘客购票开始，到行李处理，再到登机、飞行以及最终降落，每一层都建立在其下层提供的服务之上。这种层次化的服务模型也适用于网络协议的组织，帮助我们理解和管理复杂的网络系统。

Why layering?（为什么要分层？）

分层是处理复杂系统的一种有效方法，它有以下几个优点：

  • 显式结构允许识别，关系的复杂系统的部分（explicit structure allows identification, relationship of complex system’s pieces）

    • 为讨论而使用的分层参考模型（layered reference model）

  • 模块化使得系统的维护、更新变得容易（modularization eases maintenance, updating of system）

    • 层的服务实现的变化对系统的其余部分是透明的（change in layer’s service implementation: transparent to rest of system）
    • 例如，登机口程序的变化不会影响系统的其他部分（e.g., change in gate procedure doesn’t affect rest of system）

这张幻灯片解释了在设计和讨论复杂系统时，采用分层结构的原因。通过将系统划分为多个层，每个层负责不同的功能，可以更容易地识别和理解系统的各个部分如何相互作用。此外，这种模块化方法也使得修改和更新系统的某一部分，如改进一个服务的实现，不会影响到其他层，从而提高了系统的灵活性和可维护性。

总结：

分层是管理复杂系统的关键技术，它通过将系统划分为清晰定义的层来简化设计、实施和维护过程。这种方法不仅使得各个部分之间的交互更加清晰，而且还提供了更好的灵活性，因为单个层的更改不会对整个系统产生广泛的影响。

The OSI Reference Model（OSI参考模型）

OSI模型是网络通信中使用的一个基础分层模型，它定义了计算机网络通信的七个层次：

  1. Physical（物理层）
  2. Data Link（数据链路层）
  3. Network（网络层）
  4. Transport（传输层）
  5. Session（会话层）
  6. Presentation（表示层）
  7. Application（应用层）

每一层负责处理网络通信过程中的不同方面，从物理媒介的电信号传输到最终用户应用程序的数据处理。

总结：

OSI参考模型为理解和实施网络通信提供了一个框架。通过这个模型，网络通信被划分为七个独立的层次，每一层都对应特定的功能和协议。这使得网络设计和管理变得更加模块化，每一层可以独立于其他层进行开发和改进。

Internet protocol stack（互联网协议栈）

互联网协议栈是一个分层模型，用于描述网络通信中不同层次的协议和功能。

  • application（应用层）: 支持网络应用程序：IMAP, SMTP, HTTP
  • transport（传输层）: 进程到进程的数据传输：TCP, UDP
  • network（网络层）: 从源到目的地的数据报路由：IP, 路由协议
  • link（链路层）: 相邻网络元素之间的数据传输：Ethernet, 802.11 (WiFi), PPP
  • physical（物理层）: “在线”上的比特

TCP/IP Model（TCP/IP模型）:

  • Application（应用层）: 表示用户数据，并且包含编码和对话控制。
  • Transport（传输层）: 支持不同设备间在多样化网络中的通信。
  • Internet（互联网层）: 确定网络中的最佳路径。
  • Network Access（网络访问层）: 控制构成网络的硬件设备和媒体。

图示提供了TCP/IP模型的概览，该模型是互联网通信的基础，它将不同的网络功能和协议组织成一个分层结构。

总结：

互联网协议栈提供了一个明确的框架来理解和实施网络通信。它通过明确的层次结构，使得网络设计、故障诊断和性能优化变得可行。每一层都专注于特定的通信任务，并为上层提供服务，这种分层也有助于标准化通信协议的开发和实现。

The Benefits of Using a Layered Model（使用分层模型的好处）

使用分层模型对于理解和设计网络系统是非常有益的。图中展示了两个主要的网络模型：OSI模型和TCP/IP模型。

  • OSI Model（OSI模型）

    • Application（应用层）
    • Presentation（表示层）
    • Session（会话层）
    • Transport（传输层）
    • Network（网络层）
    • Data Link（数据链路层）
    • Physical（物理层）

  • TCP/IP Protocol Suite（TCP/IP协议套件）

    • 对应于OSI模型的各层，举例包括HTTP, DNS, DHCP, FTP（应用层协议），TCP, UDP（传输层协议），IPv4, IPv6, ICMPv4, ICMPv6（网络层协议），PPP, Frame Relay, Ethernet（数据链路层技术）。

  • TCP/IP Model（TCP/IP模型）

    • Application（应用层）
    • Transport（传输层）
    • Internet（互联网层）
    • Network Access（网络访问层）

重要的是要理解，网络模型仅仅是网络操作的表示，并不是实际的网络本身。分层模型提供了一种方法，可以独立地开发和理解网络的每个部分，并确保整个网络的协调工作。

总结：

分层模型的好处在于它提供了一种清晰的方式来组织和理解网络的复杂性。每一层负责特定的功能，并且可以独立于其他层进行创新和改进。模型也促进了跨不同硬件和软件平台的互操作性，以及不同网络技术之间的通信。

ISO/OSI reference model（ISO/OSI参考模型）

在ISO/OSI模型中，有两层在互联网协议栈中没有直接对应的层：

  • presentation（表示层）: 允许应用程序解释数据的意义，例如，加密、压缩、机器特定的约定。
  • session（会话层）: 同步、检查点、数据交换的恢复。

互联网协议栈“缺失”了这些层！

  • 这些服务，如果需要，必须在应用程序中实现（these services, if needed, must be implemented in application）
  • 需要吗？（needed?）

这张幻灯片指出，在实际的互联网协议栈中，并不直接包括表示层和会话层，而这些功能通常被集成到应用程序层中。它引发了一个问题，即这些层在网络通信中是否真的必要，以及它们的功能是否必须由应用程序来承担。

总结：

ISO/OSI参考模型提供了一个分层的网络通信框架，但并不是所有层都在互联网协议栈中直接实现。表示层和会话层的功能通常被集成到应用程序中，这表明在不同的网络模型中，分层的功能可能会有所不同。这种灵活性允许根据实际需要在不同层次上实现特定的网络功能。

Network Protocols（网络协议）

网络协议定义了网络通信的多个方面：

  • The role of protocols（协议的作用）: 确定通信规则和标准。
  • How the message is formatted or structured（消息的格式或结构）: 规定数据包的格式化方式。
  • The process by which networking devices share information about pathways（网络设备共享路径信息的过程）: 确定设备如何交换路由和位置信息。
  • How and when error and system messages are passed between devices（错误和系统消息如何以及何时在设备间传递）: 规定错误处理和系统通信的机制。
  • The setup and termination of data transfer sessions（数据传输会话的建立和终止）: 描述会话的开始和结束过程。

Protocol Interaction（协议交互）

协议交互描述了在Web服务器和Web客户端之间的通信中不同协议是如何协同工作的。

  • 在应用层，使用Hypertext Transfer Protocol (HTTP)。
  • 传输层使用Transmission Control Protocol (TCP)。
  • 互联网层使用Internet Protocol (IP)。
  • 网络访问层常用Ethernet。

这个过程展示了一个简单的HTTP请求和响应过程，如何通过网络的不同层次进行传输，并最终到达用户。

总结：

网络协议是网络通信的基石，它们定义了数据如何在网络上被格式化、传输、处理和接收。协议交互展示了不同层次的协议是如何共同作用，以完成从一个设备到另一个设备的数据传输的。理解这些协议及其交互是网络工程和故障排除的关键。

TCP/IP Protocol Suite（TCP/IP 协议套件）

TCP/IP协议套件是一组用于互联网和相似网络的通信协议。这一套件按功能分为不同的层级。

Application Layer（应用层）

  • Name System（命名系统）: DNS
  • Host Config（主机配置）: BOOTP, DHCP
  • Email（电子邮件）: SMTP, POP, IMAP
  • File Transfer（文件传输）: FTP, TFTP
  • Web（网络）: HTTP

Transport Layer（传输层）

  • UDP
  • TCP

Internet Layer（互联网层）

  • IP
  • NAT
  • ICMP
  • Routing Protocols（路由协议）: OSPF, EIGRP

Network Access Layer（网络访问层）

  • ARP
  • PPP
  • Ethernet
  • Interface Drivers（接口驱动）

这些协议层负责不同的网络功能，从物理连接到应用程序之间的数据传输。应用层包括处理数据请求和响应的高级协议。传输层确保数据可靠性。互联网层处理数据包的路由。网络访问层管理与物理网络的接口。

总结：

TCP/IP协议套件是网络通信的基础，它将通信过程分层处理，以确保数据有效、安全、高效地从一个设备传输到另一个设备。每一层都包含特定的协议，这些协议定义了在该层执行的特定网络任务。

TCP/IP Communication Process（TCP/IP通信过程）

Protocol Operation（协议操作）

  • 在TCP/IP通信过程中，数据会经历多层封装。首先，在应用层，数据被HTTP协议封装，然后在传输层被TCP协议进一步封装成段（segment）。在网络层，TCP段被IP协议封装成数据包（packet）。最后，在网络接入层，IP数据包被以太网协议封装成帧（frame）。

Protocol Encapsulation Terms（协议封装术语）

  • Data: 用户数据
  • HTTP Data: HTTP协议数据
  • TCP Segment: TCP协议段
  • IP Packet: IP协议包
  • Ethernet Frame: 以太网帧

从下至上的数据流动如下：

  1. 以太网帧传输IP数据包
  2. IP数据包包含TCP段
  3. TCP段包含HTTP数据
  4. HTTP数据包含最终的用户数据

图中示意

  • 图中显示了一个Web服务器和Web客户端之间的通信。数据从服务器出发，通过上述提到的层次进行封装，最后到达客户端，并在客户端进行解封装以获取原始数据。

总结：

TCP/IP通信过程涉及到多层的数据封装，确保信息能够在网络中安全有效地传输。每一层都有其特定的功能，协同工作以完成数据的发送和接收。

Encapsulation（封装）

在网络通信中，封装是指数据在从源到目的地的传输过程中，通过每一层网络模型添加相应的头部信息。

  • 源（Source）: 数据包在发送前由应用层发起，然后依次通过传输层、网络层、链路层，最终到达物理层进行传输。

    • 应用层（Application）: 处理高级应用程序协议，如HTTP。
    • 传输层（Transport）: 例如TCP/UDP，负责端到端的通信。
    • 网络层（Network）: 处理数据包的路由，例如IP。
    • 链路层（Link）: 在局域网内部分配数据包。
    • 物理层（Physical）: 实际的物理传输媒介。

  • 交换机（Switch）: 在链路层工作，转发帧到正确的端口。

  • 路由器（Router）: 在网络层工作，根据IP地址决定数据包的最佳路径。

  • 目的地（Destination）: 数据包在到达目的地后，会逆向通过物理层、链路层、网络层、传输层，最后到应用层。

总结

封装过程涉及到消息在各层次添加头部信息，以确保数据包能在复杂的网络中正确地从源点传输到目的地，并且在到达目的地时能被正确地解封装和处理。
